Description

一种低空飞行器智能运维信息物理系统
技术领域
本发明属于航空电子设备技术领域，具体涉及一种低空飞行器智能运维信息物理系统。
背景技术
随着低空经济纳入国家战略新兴产业，无人机、eVTOL等低空飞行器的规模化应用对飞行安全与运维效率提出了更高要求。同时，新一代人工智能呈现出深度学习、人机融合、群体智能等新特征，为制造业革命性的升级提供了历史性机遇。智能制造是新一代人工智能技术与先进制造技术的深度融合，将重塑设计、制造、服务等产品全生命周期的各环节及其集成。其中，“远程运维服务”等智能服务是新一代智能制造新模式的关键要素之一。随着智能制造战略的不断推进，信息物理系统(Cyber-Physics-System，CPS)的推广应用，制造业所产生的数据将呈现爆炸式的增长，但数据本身不会带来价值，需要将其转换成信息才能驱动“智能服务”产生价值，其中健康监测与智能维护技术是实现工业大数据转换的重要技术之一。
在大型运输类飞机领域的实践已经证明飞行数据(包括飞行工况、性能及机载设备的状态数据)的采集与分析，为飞行事故调查与运行安全、飞行器健康监测以及飞行与维修保障调度等提供重要依据，大大提高了现代民用航空器的安全性与运营效率。而对于飞行数据采集中，飞行数据记录系统发挥重要作用。传统民航领域的飞行数据记录系统存在体积大、功耗高、集成度低等问题，难以适配低空飞行器的轻量化需求。而目前在低空领域，对于无人机数据采集记录仪的研究，集中在大容量的数据存储技术和高速采集技术上，对于数据的卸载大多还使用USB口进行数据卸载，如此则需人工介入设备物理连接，操作耗时，且无法满足低空飞行中的实时监控需求。
发明内容
发明目的：本发明所要解决的技术问题是针对现有技术的不足，提供一种低空飞行器智能运维信息物理系统，包括低空飞行器机载数据管理设备和空地一体实时飞行数据分析系统；
所述低空飞行器机载数据管理设备包括主控单元、电源管理模块、数据采集单元、存储单元、无线传输单元；
所述主控单元支持客户化编程计算，支持对飞行数据报文与触发逻辑的定义；所述数据采集单元、存储单元和无线传输单元均由主控单元控制；
所述数据采集单元用于对低空飞行器关键系统部件的数据进行实时采集，采集的数据由主控单元处理后在存储单元存储，并能通过无线传输单元实现对数据的实时传输到地面监控站，支持自定义数据传输规则，记录的飞行参数能够根据实际需求选择。
所述主控单元以ARM微处理器为核心，搭载嵌入式Linux操作系统并集成实时补丁，实现对数据采集单元、存储单元和无线传输单元的控制；
通过客户化编程计算，对采集数据的基于数据通信规范进行解码，如ARINC429、CAN通信规范，进一步面向飞行安全与故障监控需求，定义客户化监控报文及报文触发计算逻辑。
所述电源管理模块支持宽电压输入(DC 6-18V)，配备自恢复保险丝与稳压电路，确保系统在电压波动环境下的稳定运行，并具备过充、过放保护机制，同时对外提供5V/3.3V电压。
所述数据采集单元具备如下信号采集接口：ARINC429、RS485、RS232、CAN，USB、以太网，支持模拟量信号和离散量信号采集；采集的数据由ARM微处理器处理后以通用数据格式文件存储(如CSV，TXT等，或特定规范的二进制数据文件，如航空717规范、767规范等)，并设置有文件管理功能；
所述文件管理功能包括对飞行数据的有序存储、索引构建，便于后期快速检索与分析；
所述文件管理功能支持数据分块、循环覆盖策略，确保在存储空间有限的情况下，持续记录最新飞行数据。
所述存储单元使用非易失存储器作为存储介质，并设置多级存储，包括嵌入式存储芯片和外部存储卡，具备高速读写能力、大容量存储空间和优秀的耐冲击、抗振动性能，确保在飞行过程中安全、完整地记录大量飞行数据。对于数据的存储策略优先推荐：先将采集数据存储到嵌入式存储芯片中，当存储芯片满时，自动将存储数据移动到外部存储卡中，同时芯片内有文件管理功能，能够自动进行数据的覆盖，而旧的数据记录在存储卡中，当存储卡满时，人工更换存储卡，以实现数据的长久保存。
所述无线传输单元通过4G模块进行实时数据传输，4G传输协议栈基于PPP拨号建立IP连接，支持IPv4/v6双栈连接，能够搭载两种以上网络协议；当检测到网络连接可用时，自动将飞行数据通过4G模块传输至地面监控站；
所述无线传输单元支持断点续传、数据加密功能，保证数据传输的安全与完整性。
所述空地一体实时飞行数据分析系统提供飞行数据译码功能、实时故障监控与健康评估功能、安全风险预警与管理功能、机队飞行调度与自主化维修保障功能；
所述飞行数据译码功能包括ARINC717、767译码参数库的新建、修改或删除功能，用户能够自定义添加参数，并支持二次新参数定义和计算；所述飞行数据译码功能支持数据自动化译码，数据保存为CSV、TXT格式数据，方便后续分析使用，同时也支持历史航段飞行数据的回放和查看等。
所述实时故障监控与健康评估功能包括：利用从机载系统上的传感器获得的飞行数据对飞行器进行状态监控和健康评估，支持客户化定义典型系统、部件多层级健康监控参数谱，覆盖飞行器关键系统，如电源、电机、电驱、旋翼等，基于空地数据实时监控飞行器关键系统性能的下降，实现飞行器健康状态的综合评估，为提前机队飞行任务调度、维修及控制措施、降低故障风险提供信息。
所述安全风险预警功能包括：基于报文的超限事件分析与基于全航段数据的日常运行测量；所述超限事件分析基于飞行数据，通过设定超限事件触发的逻辑进行处理，所述逻辑针对不同的低空飞行器运行安全风险设定不同等级监控项目，设置监控项目的超限阈值，基于实时的飞行数据作比较，辅助发现安全隐患、识别飞行安全趋势，及早采取管控措施；
所述日常运行测量是指：基于飞行时序数据统计特征、性能基线偏差值等，建立先兆指标监控项目和标准，基于全航段数据分析先兆指标数据，对事件级别以下的运行安全风险等进行预警。
所述机队飞行调度与自主化维修保障功能包括：面向机队飞行任务需求，考虑飞行器综合健康状态差异与不同的飞行任务对于飞行器的健康状态要求不同，以机队任务完成率利用率和综合成本为目标，实现不同飞行任务下飞行器指派，同时在满足机队飞行任务调度约束下，基于飞行器健康状态实现机队维护计划排班。
本发明提供了一种面向低空飞行器尤其适用于民用无人机、电动垂直起降飞行器(eVTOL)等低空飞行器的低成本、小型化的飞行数据采集、存储、管理与空地实时传输设备，在地面数据分析系统对飞行数据进行自动化分析计算，实现数据驱动的空地一体化低空载运工具故障预测与健康监测模型设计，考虑低空载运工具的健康状态调度机队飞行任务与维修计划，实现对低空载飞行器运工具机群的自主化持续适航与维修保障技术设计，结合健康管理的无人机队维修调度设计。
本发明提供了一种低空飞行器智能运维信息物理系统，其中轻量化、高可靠性的低空飞行器机载数据管理设备是其核心硬件设备，相较于传统民机数据记录仪有小型化、低成本的优势，相较于一般无人机数据记录仪有着能够空地实时记录、实时传输的优势，且具备客户化编程计算能力。其空地一体实时飞行数据分析系统是核心软件功能，实现基于飞行数据的低空飞行器实时故障监控与健康评估、安全风险预警与管理以及机队飞行调度与自主化维修保障等功能。
有益效果：本发明设有多种数据采集设备接口，可以与多种设备连接实现数据的实时存储和传输的功能并且结构紧凑，并且可根据实际型号需求裁剪数据接口配置，确保满足数据采集要求前提保障重量轻，体积小，适用于民用无人机、电动垂直起降飞行器(eVTOL)等低空飞行器的飞行数据采集、存储、管理与空地实时传输系统设计。
本发明具有数据预处理、客户化编程计算以及数据管理能力，采用多级存储和文件管理实现对数据的长时间存储，确保不具备空地通讯网络的情况下数据可以自动缓存，待网络恢复后自动恢复数据下传。对数据的查询。
本发明能实时进行空地数据传输，减少人工对数据的卸载，方便地面监控站对飞行器进行实时健康监测与安全风险预警，保障飞行安全、提高运营效率。
本发明能够实现实时故障监控与健康评估，覆盖飞行器全寿命周期、关键机载系统与部件，基于多层级健康监控特征谱实现整机实时故障监控与综合健康评估。
本发明能够实现实时安全风险预警与管理功能，通过超限事件监控与日常运行测量数据分析，降低飞行器任务执行中发生运行不正常事件甚至安全事故的概率。
本发明能够实现基于飞行器健康状态的飞行任务调度与自主化维修保障，优化了飞行器飞行任务指派与维修计划排班调度方案，有效节省了人工检查、维修排故的时间，提高了飞行器机队利用率、降低维护成本。

具体实施方式
如图1所示，本发明提供了一种低空飞行器智能运维信息物理系统，包括：轻量化、高可靠性的低空飞行器机载数据管理设备和空地一体实时飞行数据分析系统。
在本实施例中，通过在民用低空飞行器中装载数据管理设备可以采集飞行器上的飞行数据，经过主控单元初步处理采集的数据并将其传输到无线传输单元中，无线传输单元通过天线发送数据到地面监控站中，在地面监控站中实时运行飞行数据分析系统，该系统可以对数据管理设备空地实时传输的飞行数据进行译码、从而利用飞行数据进行实时故障监控与健康评估、安全风险预警与管理；除此之外该系统可以基于低空载运工具个体的健康评估，结合其计划维修需求，进一步进行机队的健康管理与调度保障等功能。
在本实施例中，数据管理设备的原理框图如图2所示，包括主控单元、电源管理模块、数据采集单元、存储单元、无线传输单元。所述主控单元采用ARM嵌入式芯片，集成RAM和Flash，搭载嵌入式Linux操作系统，通过相应的接口电路与外部进行数据交互，实现数据的采集、存储、传输控制、文件管理和导出等功能，完成用户需求功能的实现。所述数据采集单元包括各种接口和接口电路，可以实现对数字量信号包括但不限于ARINC429、RS485、RS232、CAN，模拟量信号以及离散量信号的采集；所述无线传输单元通过4G模块和天线实时进行空地数据传输，具有上行速率可达50Mbps，时延低于100ms，支持高带宽实时传输的优点；所述电源供电/稳压电路，配备自恢复保险丝与稳压电路，为设备支持宽电压输入(DC6-18V)，确保系统在电压波动环境下的稳定运行；所述存储单元包括嵌入式flash芯片以及外部扩展存储卡SD卡，可以实现对数据的存储。
所述主控单元主要以ARM cortex-A架构的微处理器和外围电路组成，微处理器具体型号是MCIMX6Y2CVM08AB，具有800MHz的主频；同时主控单元集成512MB的DDR3L RAM，具体型号为NT5CC256M16EP-EK，以及8GB的EMMC Flash，具体型号为KLM8G1GET。
数据采集单元包括对RS485,RS232,CAN的接口电路以及外扩ARINC429的协议转换模块。具体而言通过电平转换芯片实现对不同协议数据的采集，具体使用的芯片型号包括SP3232、SP3485、TJA1050。对ARINC429数据的采集，通过四川科洛威尔公司设计的内嵌式CLV-8063模块，将ARINC429转换为串口数据进而被采集。
存储单元除了flash存储还包括外部扩展存储卡SD卡，具体是microSD卡(TF卡)，通过TF卡接口电路可搭载TF卡实现存储扩展。本实施例中使用的为sandisk的32GB通用高速存储卡。电源管理模块主要为电压的稳压电路，具体使用两个DC-DC稳压芯片以及自恢复保险丝实现对电源的管理，可以支持宽电压输入(DC 6-18V)。无线传输模块通过4G模块进行实时数据传输，具体4G模块为上海移远公司的EC20R2.1 Mini PCIe-C模块通过MiniPCIe接口接入设备，基于PPP拨号建立IP连接，支持IPv4/v6双栈连接，可搭载多种网络协议。
在本实施例中，数据管理设备的软件实现基于嵌入式Linux操作系统构建多线程任务调度架构，软件架构如图3所示。本示例包括硬件平台，操作系统，软件功能，硬件平台主要由上述硬件功能架构组成；操作系统包括内存管理、进程管理、设备驱动程序等，为底层硬件和上层软件功能提供接口适配，传输服务，管理服务等功能，接口适配能方便软件代码在不同操作系统上移植，提高软件功能的实用型，可移植性，传输服务为不同软件功能提供数据传输，同时隔离软件应用层和内核层空间，提高安全和稳定性，管理服务使系统在运行时最大限度的利用计算资源；软件功能基于实践应用功能开发，实现设备功能，满足用户需求。
在一些实施例中，所述数据管理设备软件的工作流程，包括设备接口和存储硬件初始化，初始化文件系统，连接云服务器，启动采集、存储、传输线程、运行结束流程。
在本示例中，所述线程包括数据采集线程、存储线程和网络传输线程；对于数据采集线程，设置多级环形缓冲区以提高数据吞吐量并且设置缓冲区剩余空间小于256B时切换缓冲区，防止数据溢出丢失；对于存储线程，将采集数据存储到存储模块中，存储模块格式化为FAT32格式，格式兼容PC端，数据采用CSV格式文件存储，设置最大单文件容量阈值和文件数量最大阈值，当超过阈值时触发文件轮转机制自动生成新文件并删除最旧文件，实现数据的覆盖；对于网络传输线程主要通过4G模块使用TCP/IP协议连接云服务器并拥有TCP断线重连功能，连接成功后，自动将采集的数据传输到云服务器中；在运行退出流程中，收到结束信号，关闭线程并安全退出程序。
在一些实施例中，对于数据管理设备的数据卸载，包括但不限于通过高速USB接口将数据有线传输到PC机中，在采集过程中实时传输采集数据到云服务器中，再由地面监控站下载，以及通过人工更换存储卡，再通过读卡器对数据进行读取。
在一些实施方式中，可以设计数据处理的方式，进一步完善对数据的管理，对于传输数据也可以经过数据初步处理，比如卡尔曼滤波、异常值剔除等方式对数据进行清洗，实现对传输数据的精细化管理，从而更好的实现对低空飞行器的实时监测，故障预警等。
进一步的，对本设备基于软件平台的功能进行描述，如图4所示，将软件平台功能分为飞行数据译码、故障监控与健康评估、安全风险预警、机队飞行调度与自主化维修保障功能。在本实施例中，对于上述地面数据分析系统的飞行数据译码功能，可以支持在航空中广泛使用的包括ARINC717、767规范进行自动化译码。具体功能包括译码参数库的新建、修改或删除等功能，也支持用户自定义添加参数，并支持二次新参数定义和计算。译码后数据保存为CSV、TXT等格式数据方便后续分析使用，同时也支持历史航段飞行数据的回放和查看等。具体而言，当数据管理设备将采集的数据以ARINC717/767进行二进制存储，传输时，地面监控站可以实时接收并根据译码参数库对数据进行译码，转化后的工程值数据以CSV、TXT等格式保存。上述译码参数库可以根据ARINC717、767规范建立，同时也可以用户自定义。
在本实施例中，对故障监控与健康评估功能描述，功能模块经历以下几个流程：关键系统部件导入、特征参数定义、参数谱构建、算法选择、飞参数据导入。在一些实施例中，通过脚本设置，定义低空飞行器的电源、电机、电驱、旋翼等关键部件，为不同部件添加不同特征参数定义，如电源部件的电流，电压，电源电量等定义。再对低空飞行器的健康监控特征谱构建，将不同特征分为一级、二级、三级特征数据，将基于传感器收集数据定义为一级特征数据，将基于一个或者多个一级特征进行脚本处理生成的数据定义为二级特征数据，如将不同任务阶段电源电流时序统计特征将其定义为二级特征数据，基于此更高级的特征数据也可被定义，由此生成多层级的健康监控参数谱。对特征数据进行机器学习处理，采用的算法模型不限于向量回归、随机森林回归，长短期记忆网络模型对低空飞行器进行关联分析、异常检测、趋势预测分析，利用森林回归模型通过引入多个特征数据进行关联分析，分析特征数据的重要程度，利用向量回归算法模型，捕捉特征谱中实际测量值与基线值的偏差值序列捕获异常值，实现异常检测。在实例中，经过上述流程后，实现对低空飞行器关键系统或部件的故障监控和健康评估。
进一步的，对本设备的安全风险预警功能描述，功能基于事件触发的逻辑与日常运行测量进行处理，基于事件触发的逻辑：事件在非预期飞行状态时产生，被触发的事件遵循触发逻辑，该逻辑将一个或多个参数的当前值与一些定义的阈值或一些其他变量/参数进行比较，一旦超过这些值，就会触发一个具有相应级别的事件。在实例中，将低空飞行器飞行数据的状态值与设定的参数阈值相比较，当飞行状态值超过逻辑设定的阈值时即发生“超限事件”，超限事件是指监控参数超过规定的门限，且超限状态持续达到规定时间长度的现象，发出安全风险预警。阈值的设置基于不同事件中不同参数的“超限裕度”，超限裕度基于超限事件以及稳定阈值的概念即超限裕度可以衡量飞行参数离超限事件发生的远近程度，当超限裕度越大时，距离发生超限事件越远。基于不同事件中的不同飞行参数不同，超限裕度与阈值的设置也应视情况而定。由于对于同一事件在任务执行不同阶段的触发阈值不同，为了帮助分析的标准化，设定不同的飞行阶段，飞行数据流被分成飞行阶段可以简化事件脚本逻辑中的触发条件，实例中，将低空飞行器的飞行阶段划分为：起飞，垂直爬升、巡航、着陆阶段。有两种类型的事件：单个事件监控一个或几个参数的偏差，并根据阈值定义触发严重性级别、使用多个单个事件的组合事件，以识别特定的状况或风险。在实例中，不同飞行阶段设定事件有爬升阶段初始爬升速度小、巡航阶段超过最大速度等。针对不同事件如eVTOL爬升阶段初始爬升速度小事件的阈值设定30节(55km/h)。日常运行测量基于飞行时序数据统计特征、性能基线偏差值等，对全航段做先兆指标数据分析，对事件级别以下的运行安全风险等进行预警。
进一步的，对本设备的机队飞行调度与自主化维修保障功能描述，本设备所配套的“机队飞行调度与自主化维修保障功能”以健康状态感知驱动的智能决策为核心，实现从飞行任务调度到维修保障活动规划的全流程耦合协同，最低限度提高机队飞行任务完成率于可利用率。系统融合多源数据融合建模、实时PHM算法与任务调度优化机制，构建面向低空无人系统的智能保障平台。在飞行任务调度阶段，系统根据任务类型(如投递/侦察)、负载权重、环境干扰等级等参数，结合飞行器的健康评估指标(Health Index,HI)与剩余使用寿命(RUL)，通过基于阈值驱动的优先级匹配机制和基于约束规划的多目标调度算法(改进型NSGA-II)，实时决策低空飞行器任务分配方案，提升任务完成率与编队协同性。在维修保障活动规划阶段，结合随机退化建模与健康值回归预测(采用BiLSTM/贝叶斯退化模型)，基于历史与实时数据生成健康退化曲线。通过构建潜在失效-功能失效(P-F)曲线并引入Lead-time预测窗口，动态触发维修决策，区分计划维修与故障后紧急维修。系统同时对预测误差(MAE)与虚警/漏警率(FPR/FNR)进行动态评估，确保维修策略在鲁棒性与经济性之间取得最优平衡。
