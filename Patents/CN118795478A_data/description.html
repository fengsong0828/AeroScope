Description

基于低空无人飞行器的雷达感知方法
技术领域
本申请涉及无线感知技术领域，具体涉及雷达感知技术领域，尤其涉及基于低空无人飞行器的雷达感知方法。
背景技术
随着低空无人飞行器技术的快速发展和广泛应用，雷达感知方法在低空无人飞行器的导航、避障、目标跟踪等方面发挥着至关重要的作用。然而，随着无人飞行器应用场景的多样化和复杂化，特别是在城市、山区等复杂环境中，对雷达感知方法的要求也日益提高。传统的低空无人飞行器雷达感知方法往往面临着控制精度低、响应速度慢等问题。在复杂的飞行环境中，这些问题可能导致无人飞行器无法准确感知周围环境，进而影响到其飞行安全、任务执行效率和自主性。
发明内容
本申请通过提供了基于低空无人飞行器的雷达感知方法，旨在解决了低空无人飞行器在复杂环境中对目标感知物体的无法准确感知和定位的技术问题。
鉴于上述问题，本申请提供了基于低空无人飞行器的雷达感知方法。
本申请提供了基于低空无人飞行器的雷达感知方法，所述方法包括：在待感知区域启动目标无人飞行器，基于目标无人飞行器发送第一雷达发送信号；基于所述第一雷达发送信号生成第一雷达接收信号，确定目标状态信息；根据所述目标状态信息生成第二雷达发送信号，确定多个目标参数；根据所述多个目标参数对目标感知物体进行实时跟踪，生成目标实时定位信息；连接远程终端，将所述目标实时定位信息进行可视化，传输至所述远程终端进行处理。
本申请中提供的一个或多个技术方案，至少具有如下技术效果或优点：
上述基于低空无人飞行器的雷达感知方法，该方法在需要进行感知的区域，启动目标无人飞行器。目标无人飞行器会发送第一雷达发送信号，用于探测周围环境。随后，接收这些信号从环境中反射回来的部分，形成第一雷达接收信号。通过分析这些接收信号，判断目标的状态信息。之后，基于获取的目标状态信息，生成第二雷达发送信号。这些信号用来进一步探测并确定多个目标参数。然后，对目标感知物体进行实时跟踪。通过不断发送和接收雷达信号，实时更新目标的位置信息，形成目标实时定位信息。最后，将目标实时定位信息发送到远程终端。在远程终端上，这些信息会以可视化的形式展示，方便操作人员进行处理。整个流程实现了无人飞行器对目标物体的精确感知和实时跟踪，提高了无人机在执行任务时的效率和准确性。

实施例
如图1所示，本申请提供了基于低空无人飞行器的雷达感知方法，所述方法包括：
在待感知区域启动目标无人飞行器，基于目标无人飞行器发送第一雷达发送信号。
在低空飞行时，无人飞行器需要更加精确地感知周边环境，以确保飞行的安全性和稳定性。传统的雷达感知方法由于无法实时准确地获取这些信息，难以满足现代低空无人飞行器的高精度、高可靠性和高自主性需求。
在本申请实施例中，在待感知的区域，系统终端启动目标无人飞行器。目标无人飞行器会利用其搭载的雷达探测单元，生成并发送第一雷达发送信号，这些信号以电磁波的形式向周围环境扩散，以感知周围的物体和环境。通过发送这些信号，无人飞行器可以对目标区域的初步探测和感知。
进一步，如图2所示，本申请提供了在待感知区域启动目标无人飞行器，基于目标无人飞行器发送第一雷达发送信号，方法包括：
基于目标无人飞行器的飞行性能数据、飞行安全数据，确定目标无人机的最大飞行高度信息；基于目标无人飞行器内布设的雷达探测单元提取雷达性能数据、雷达探测需求数据；根据所述雷达性能数据、所述雷达探测需求数据确定最大探测距离信息、水平传播广度信息；按照所述最大飞行高度信息、所述最大探测距离信息、所述水平传播广度信息生成所述第一雷达发送信号。
优选的，在规划目标无人飞行器的飞行任务时，系统终端首先根据无人飞行器的飞行性能数据和飞行安全数据来确定其最大飞行高度。这个最大飞行高度是确保无人机能够安全、稳定飞行且不超出其性能极限的关键参数。随后，利用无人机内部布设的雷达探测单元来提取雷达的性能数据和探测需求数据。这些性能数据包括雷达的探测能力、精度、稳定性等方面的信息，以及任务中需要雷达达到的具体探测要求。之后，基于提取的雷达性能数据和探测需求数据，系统终端确定雷达的最大探测距离和水平传播广度。最大探测距离指的是雷达能够探测到的最远距离，而水平传播广度则描述了雷达信号在水平方向上能够覆盖的范围。然后，根据最大飞行高度、最大探测距离和水平传播广度这些信息，系统终端生成第一雷达发送信号。这个信号将被无人飞行器进行发送，用于在飞行过程中进行环境感知和目标探测，确保无人机能够安全、高效地完成任务。
基于所述第一雷达发送信号生成第一雷达接收信号，确定目标状态信息。
在一个实施例中，当目标无人飞行器发送出第一雷达发送信号后，系统终端同时也会等待并接收返回的雷达信号，即第一雷达接收信号。这些接收到的信号是第一雷达发送信号在接触到周围环境中的物体，如建筑物、山脉、飞机或其他目标，后反射回来的。通过分析这些第一雷达接收信号，无人飞行器可以提取出关于反射物体的目标特征参数，并根据这些目标特征参数与预设阈值的关系，判断出目标特征参数所属的类型。这些类型将共同组成目标状态信息，它们为无人飞行器提供了关于其周围环境的详细感知，为后续的飞行和任务执行提供重要依据。
进一步，本申请提供了基于所述第一雷达发送信号生成第一雷达接收信号，确定目标状态信息，方法包括：
通过目标无人飞行器发送所述第一雷达发送信号；基于待感知区域的环境信息，提取所述第一雷达发送信号探测反射的第一雷达接收信号；对所述第一雷达接收信号进行环境滤波，提取目标特征参数。
优选的，目标无人飞行器在执行任务时，系统终端首先发送第一雷达发送信号以探测其周围的待感知区域。这些信号在碰到环境中的物体后会发生反射，形成第一雷达接收信号，这些信号将会被系统终端进行收集。这些收集到的信号包含了关于反射物体的丰富信息。为了从这些接收信号中获取需要的特征数据，无人飞行器会进行环境滤波处理。环境滤波是一个去除噪声和干扰，提高信号质量的过程，它有助于更准确地识别和分析反射信号。经过环境滤波后，系统终端会提取目标的特征参数。这些特征参数包括目标到无人飞行器的距离、目标相对于无人飞行器的移动速度，以及目标相对于无人飞行器的方向等。这些参数对于无人飞行器了解其周围环境和做出后续决策至关重要，为后续任务执行提供了关键信息支持。
当所述目标特征参数小于等于预设阈值，则获得静态目标信息，添加至所述目标状态信息；当所述目标特征参数大于所述预设阈值，则获得动态目标信息，添加至所述目标状态信息。
优选的，当获得目标特征参数后，系统终端将这些参数与预设的阈值进行比较。这一步骤是为了判断探测到的目标是静态的还是动态的。这个预设阈值是基于实际应用场景的需求、雷达探测单元的性能以及历史数据分析来设定的。例如，在无人飞行器进行航拍或环境监测时，如果探测到的目标速度非常缓慢或几乎不动，那么可以设定一个较低的阈值来判断其为静态目标。而在需要跟踪快速移动目标的场景中，如军事侦察或空中交通管理，会设定一个较大的阈值。如果提取到的目标特征参数小于或等于预设的阈值，那么系统终端会判断这个目标是静态的，可能是一个固定的物体，如建筑物或山丘等。随后，系统终端将这个静态目标的信息添加到目标状态信息中，以便后续参考或处理。相反，如果目标特征参数大于预设的阈值，系统终端则会判断这个目标是动态的，可能是另一个飞行器、车辆或生物等。同样地，也会将这个动态目标的信息添加到目标状态信息中，以便进一步跟踪、分析，为后续的飞行和任务执行提供重要参考。
根据所述目标状态信息生成第二雷达发送信号，确定多个目标参数。
在一个实施例中，在获取了目标状态信息后，系统终端根据这些信息生成第二雷达发送信号。这一步骤是为了进一步精细化探测和定位已经识别出的目标。具体来说，系统终端首先根据目标状态信息中记载的信息类型，判断进行二次发射探测所需要的信号种类，并生成第二雷达发送信号。随后，根据这个第二雷达发送信号，系统终端进行发送探测，从而生成更为精准的第二雷达接收信号。通过对这个接收信号进行解析，可以获得多个目标参数，如更精确的距离、速度、角度等，为后续的决策和操作提供有力支持。
进一步，本申请提供了根据所述目标状态信息生成第二雷达发送信号，确定多个目标参数，方法包括：
当所述目标状态信息为所述静态目标信息时，生成脉冲雷达信号；将所述脉冲雷达信号作为所述第二雷达发送信号进行发送探测，获得第二雷达接收信号。
可选的，当目标状态信息所记载的信息为静态目标信息时，系统终端会生成脉冲雷达信号作为第二雷达发送信号。脉冲雷达信号的特点在于它发射一个短而高功率的脉冲，并等待这个脉冲从目标上反射回来的回波。这种信号模式适用于远距离探测，因为脉冲宽度越短，雷达探测单元的距离分辨率就越高，从而能够更准确地测量目标到雷达的距离。通过发送这种脉冲雷达信号，系统终端可以获取到第二雷达接收信号，该信号包含了目标反射回来的脉冲信息。通过对接收信号的处理和分析，无人飞行器可以进一步确定静态目标的精确位置、距离等参数，为后续的导航、避障或监测任务提供重要依据。
基于所述第二雷达接收信号进行解析，获得静态目标距离、静态目标方位角、静态目标俯仰角；将所述静态目标距离、所述静态目标方位角、所述静态目标俯仰角添加至所述多个目标参数。
可选的，在利用脉冲雷达信号进行静态目标探测的过程中，系统终端首先发送脉冲雷达信号，并接收从目标反射回来的第二雷达接收信号。再对这些接收到的信号进行解析以获取关于静态目标的详细参数，包括静态目标距离、静态目标方位角、静态目标俯仰角。其中，静态目标距离是通过测量脉冲雷达信号从发射到接收的时间差来计算的。由于雷达信号传播的速度是已知的，即光速，因此系统终端可以通过时间差来精确地计算出目标到雷达的距离。静态目标方位角和静态目标俯仰角则依赖于相位差测向法、波束形成法等技术。这些方法通过分析接收到的信号中不同方向上的相位差异或波束模式，来计算出目标信号的到达角，从而确定目标的精确方位角和俯仰角。一旦获得了这些参数，系统终端就会将这些静态目标距离、静态目标方位角和静态目标俯仰角添加到多个目标参数中，以便后续的数据处理、分析和决策制定。
进一步，本申请提供了根据所述目标状态信息生成第二雷达发送信号，确定多个目标参数，方法包括：
当所述目标状态信息为所述动态目标信息时，生成连续波雷达信号；将所述连续波雷达信号作为所述第二雷达发送信号进行发送探测，获得第二雷达接收信号。
可选的，当目标状态信息所记载的信息为动态目标信息时，系统终端会生成连续波雷达信号作为第二雷达发送信号。与脉冲雷达信号不同，连续波雷达信号是持续发射的，不存在明显的脉冲或间隔。这种信号模式适用于需要持续追踪动态目标的情况。通过发送这种连续波雷达信号，系统终端会接收从目标反射回来的信号，即第二雷达接收信号。这些接收信号包含了关于动态目标的位置、距离等信息。
基于所述第二雷达接收信号进行解析，获得动态目标距离、动态目标速度、动态目标轨迹；将所述动态目标距离、所述动态目标速度、所述动态目标轨迹添加至所述多个目标参数。
可选的，当探测到动态目标时，系统终端会发送连续波雷达信号进行探测，并接收反射回来的第二雷达接收信号。随后，对这些接收信号进行解析，以获取关于动态目标的关键参数，包括动态目标距离、动态目标速度、动态目标轨迹。对于动态目标距离，系统终端通过测量连续波雷达信号与雷达回波信号之间的相位差，并结合信号的频率，来计算动态目标与雷达系统之间的距离。这种方法能够准确地反映目标在某一瞬间的位置。对于动态目标速度，系统终端连续测量不同时间点的距离。通过比较这些时间点上的距离数据，计算出目标相对于雷达的速度。这种速度测量对于跟踪快速移动的目标尤为重要。对于动态目标轨迹，系统终端会结合连续多个时间点的到达角数据。这些到达角数据的获取与前述相同。通过综合分析这些到达角数据，可以确定目标的运动方向，并据此绘制出目标的运动轨迹。一旦获得了这些参数，系统终端会将这些信息添加到多个目标参数中。这些参数对于后续的目标识别等任务具有重要意义。
根据所述多个目标参数对目标感知物体进行实时跟踪，生成目标实时定位信息。
在一个实施例中，在获取了多个目标参数后，系统终端利用这些信息对目标感知物体进行实时跟踪。这个实时跟踪是通过构建的长短时记忆网络对目标感知物体的运动模式进行分析实现的。通过实时跟踪，系统终端能够生成目标实时定位信息，这些信息反映了目标物体在当前时刻的确切位置和运动状态。这个实时定位信息可以帮助系统终端做出快速而准确的决策，以确保任务的顺利执行。
