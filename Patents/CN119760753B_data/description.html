Description

低空智能飞行器数据安全保护方法、系统、介质及服务器
技术领域
本申请涉及保护数据安全的技术领域，尤其是涉及低空智能飞行器数据安全保护方法、系统、介质及服务器。
背景技术
随着低空智能飞行器技术的迅猛发展，这类飞行器在航拍、农业监测、紧急救援、货物运输乃至休闲娱乐等多个领域展现出了前所未有的广泛应用潜力，极大地提升了工作效率和生活质量。然而，在充分享受这些高科技产物所带来的便捷与效益的同时，一个不容忽视的问题——数据安全问题，正逐渐浮出水面，成为制约低空智能飞行器进一步发展的关键因素之一。
由于低空智能飞行器在执行任务时，往往搭载着广角高清摄像头、传感器等先进设备，能够实时捕捉并传输大量的地面图像、视频及其他敏感信息。这些信息若被不当使用或泄露，可能对个人、企业乃至社会稳定构成严重威胁。特别是在一些特殊时期或敏感区域，为了维护公共安全、保护重要设施或应对特定事件，会设立有禁飞区域。
虽然低空智能飞行器在接收到禁飞指令后会遵守规定，不再进入这些被明确划定的禁飞区域，但是即使低空智能飞行器本身不越界，低空智能飞行器在接近禁飞边界时仍有可能拍摄到边缘区域甚至是禁飞区域内禁止拍摄的内容，使得低空智能飞行器在远离目标区域时就能捕捉到清晰的图像。一旦这些包含禁飞区域的图片被非法获取、传播或恶意篡改，后果将不堪设想。
发明内容
为了提高低空智能飞行器的内部数据的安全性，本申请提供了低空智能飞行器数据安全保护方法、系统、介质及服务器。
第一方面，本申请提供了低空智能飞行器数据安全保护方法，采用如下的技术方案：
低空智能飞行器数据安全保护方法，包括以下步骤：
数据采集：获取低空智能飞行器存储的内部数据；
第一判断：判断内部数据是否包含有图像数据，若是，则执行建系的步骤；若否，则执行第二存储的步骤；
建系：以图像数据中任意一个像素点作为坐标原点建立坐标系；
第一获取：获取图像数据中每个像素点的坐标值和像素值；
建栈：建立与像素值的类别数量相同的堆栈；
第一存储：将像素值相同的像素点的坐标值压入堆栈，记录每个堆栈对应的像素值；
第二存储：将内部数据存储在预设的数据库中；
用户验证：验证用户身份是否合法，若是，则执行反馈的步骤；若否，则执行告警的步骤；
反馈：将每个堆栈对应的像素值和/或内部数据反馈给用户；
告警：发出告警信号。
本申请首先获取了低空智能飞行器存储的内部数据，之后判断内部数据是否存在图像数据，如果存在，则建立坐标系，以获取每个像素点的坐标值和像素值，之后建立与像素值的类别数量相同的堆栈，进而方便地存储和管理具有相同像素值的像素点的坐标值，有助于对这些像素点进行快速访问和处理，堆栈的使用也提高了图像数据处理的灵活性和效率。随后，本申请将与像素值相同的坐标值按照预设顺序压入堆栈，并记录每个堆栈对应的像素值，反之，则对内部数据进行存储，在用户身份验证通过后，才会向该用户反馈正确的数据，通过采用上述技术方案，本申请实现了对像素点的分类存储和管理，进而实现了对图像数据的隐藏，提高了低空智能飞行器内部图像数据的安全性。

第四计算：计算图像数据中所有像素值的方差；
第二判断：判断所述方差是否等于零，若是，则执行建栈的步骤；若否，则执行图像分割的步骤；
图像分割：采用区域增长算法对图像数据进行分割，获得多个分割块；
第三判断：基于八邻域像素值判断分割块是否存在噪声点，若是，则将噪声点处的像素值修改为所述噪声点的八邻域像素值的平均值；若否，则执行图像更新的步骤；
图像更新：分别将每个分割块更新为图像数据，并执行第四计算的步骤。
本申请首先计算了图像数据中所有像素值的方差，方差为零表示图像数据中的像素值都相同或均匀变化，即图像为单色图像或均匀图像。随后，本申请通过判断方差是否等于零，以识别当前的图像是否为单色图像或均匀图像，当方差不为零时，表明当前图像为不均匀图像，即可能存在噪声点，需要进一步进行图像分割，以判断是否的确存在噪声点。随后，本申请采用区域增长算法对图像数据进行分割，以获得多个具有相似像素值的分割块，进而对图像中的不同区域进行单独处理和分析，之后基于八邻域像素值判断当前分割块是否存在噪声点，若是，则将噪声点处的像素值修改为其八邻域像素点像素值的平均值，从而平滑图像。通过图像分割和噪声处理，本申请能够减少图像中的噪声和干扰，提高图像的质量和可用性。
可选地，在执行图像分割的步骤之后，执行第三判断的步骤之前，还包括：
第四判断：判断分割块内像素点的数量是否为一，若是，则执行第三判断的步骤；若否，则执行图像更新的步骤。
本申请通过判断分割块内像素点的数量是否为一，若是，则意味着该分割块为单色块，并直接执行第三判断的步骤，以减少图像分割的步骤的执行次数，从而提高整体处理效率，本申请通过先筛选掉只包含一个像素点的分割块，可以优化计算资源的分配，使得更多的资源可以集中在处理较大的、包含更多有用信息的分割块上，进而提高处理效率。
可选地，在执行第一判断的步骤之后，执行第二存储的步骤之前，还包括：
第二获取：获取内部数据的聚类结果，基于聚类结果获取n个聚类中心；
建库：构建n个数据子库，将第i个数据子库记为第i数据子库；
第五计算：依次计算第j个聚类中心与剩余的聚类中心的欧几里得距离，获得n-1个欧几里得距离；计算n-1个欧几里得距离的和，记为第二数据；
第一排序：将n个第二数据按照预设顺序进行排列，将排列后的结果记为第二数据序列；
关联：将第二数据序列中的第二数据与拥有相同序号的数据子库进行关联，获得关联结果；
第三存储：根据关联结果将内部数据存储至对应的数据子库中。
本申请首先获取了内部数据的聚类结果，并基于聚类结果获取了n个聚类中心，随后，本申请构建了n个数据子库，便于分类存储低空智能飞行器的内部数据。随后，本申请计算了各聚类中心之间的欧几里得距离，并且分别计算了每个聚类中心到剩余聚类中心的欧几里得距离的和，记为第二数据，将n个第二数据按照预设顺序进行排列。随后，本申请将第二数据序列中的第二数据与拥有相同序号的数据子库进行关联，实现了内部数据与数据子库的对应，便于后续的内部数据存储和访问。随后，本申请根据关联结果将内部数据存储至对应的数据子库中，实现了数据的分类存储，进而有助于加快数据检索速度，提高数据处理效率，本申请通过动态细粒度的数据存储方式，增强了数据的安全性。
可选地，在执行第三存储的步骤之后，所述方法还包括：
第一加密：采用对称加密算法对第i数据子库内存储的内部数据进行加密，获得第i加密数据，采用第i加密数据更新第i数据子库内存储的内部数据；
第二加密：采用非对称加密算法对第i加密数据与第i+1数据子库内存储的内部数据进行加密，获得第i+1加密数据，采用第i+1加密数据更新第i+1数据子库内存储的内部数据；
第五判断：判断i+1是否等于n，若否，则执行迭代的步骤；
迭代：将第i+1数据子库作为新的第i数据子库，将第i+1加密数据作为新的第i加密数据，执行第二加密的步骤，直至满足预设的停止条件。
本申请采用对称加密算法对第i数据子库内存储的内部数据进行加密，能够提高第i数据子库内存储的内部数据的安全性，使得加密后的内部数据即使被非法获取，也难以被解密和篡改，从而保护了内部数据的隐私和完整性。随后，本申请采用非对称加密算法对第i加密数据与第i+1数据子库内存储的内部数据进行加密，进一步增强了数据的安全性，使用不同的加密算法可以增加数据被破解的难度，提高数据保护的层次性。随后，通过执行第五判断的步骤，本申请可以检查是否所有的内部数据都已经完成了加密处理。如果还有未加密的内部数据，则通过迭代的步骤继续执行加密过程，直到所有内部数据都被加密为止，进而减少数据泄露的风险。通过采用上述方案，本申请进一步增强了低空智能飞行器内部数据的安全性，本申请采用的多层次、多算法的加密方式提高了数据保护的强度，使得内部数据即使在被非法获取的情况下也难以被解密和篡改。
可选地，在执行关联的步骤之后，执行第一存储的步骤之前，还包括：
数据筛选：识别内部数据中的敏感数据与非敏感数据；
第二排序：将各个簇内的敏感数据按照采集时间先后顺序进行排序，获得n个敏感数据序列；
计算距离：分别计算第a个敏感数据序列与剩余敏感数据序列的距离矩阵，根据距离矩阵计算累计距离矩阵；
确定序列：基于累计距离矩阵，利用动态路径规划算法获取与第a个敏感数据序列相似度最大的敏感数据序列，记为相似数据序列；
结合：将第a个敏感数据序列的敏感数据与相似数据序列所对应的非敏感数据结合，获得结合后的数据，将结合后的数据更新为内部数据。
本申请首先识别了内部数据中的敏感数据与非敏感数据，随后，将各个簇内的敏感数据按照时间先后顺序进行排序，以揭示数据的时序特征和变化趋势，便于后续的数据分析和挖掘。随后，本申请通过计算第a个敏感数据序列与剩余敏感数据序列的距离矩阵，并根据距离矩阵计算累计距离矩阵，进而量化第a个敏感数据序列与剩余敏感数据序列之间的相似性。随后，本申请根据累计距离矩阵获取与第a个敏感数据序列相似度最大的敏感数据序列（即相似数据序列），之后将第a个敏感数据序列的敏感数据与相似数据序列所对应的非敏感数据结合，并更新为内部数据，有助于在保护敏感数据的同时，更好的利用其他数据序列的非敏感数据隐藏第a个数据序列的敏感数据，进一步提高了数据安全。

具体实施方式
以下结合图1至图5对本申请作进一步详细说明。
实施例1：本实施例公开了低空智能飞行器数据安全保护方法，参照图1，所述方法包括：S1数据采集、S2第一判断、S3图像存储、S4第二存储和S5合法验证，首先采集内部数据，之后判断内部数据是否含图像数据。若含图像数据，则选取图像中任一像素点作为原点建系，获取各像素的坐标与像素值，并建立与像素值类别数相等的堆栈存储相同像素值的坐标信息；若不含图像数据，则直接将内部数据存储于预设数据库。此外，本实施例还包含用户身份验证环节，合法用户可获得数据反馈，非法用户则触发告警信号，本实施例包括如下步骤：
S1数据采集，从飞行器中获取存储的内部数据，所述内部数据包括飞行参数、传感器读数、设备状态信息以及采集的图像数据等。
S2第一判断，判断所述内部数据是否包含有图像数据，若是，则执行S3图像存储；反之，则执行S4第二存储。
S3图像存储，包括S31建系、S32第一获取、S33建栈和S34第一存储。
S31建系，选择图像数据中的一个任意像素点作为坐标原点。这个坐标原点可以是图像的左上角、中心或其他位置的任意一个像素点。
确定坐标轴的方向，本实施例中选择水平方向为横轴，垂直方向为纵轴，形成直角坐标系。
在直角坐标系中，每个像素点都有一个唯一的坐标值，用于标识其在图像数据中的位置。
在其他实施例中，也可以选择其他方向作为横轴和纵轴，只要能够确定图像数据中每个像素点有唯一的坐标值即可。
S32第一获取，遍历图像数据中的每个像素点，获取每个像素点的坐标值和像素值。
S33建栈，统计图像数据中不同像素值的种类数量，对于每种不同的像素值，建立一个对应的堆栈，该堆栈用于存储具有相同像素值的像素点的坐标值。每个堆栈中存储的元素表示具有该像素值的像素点在图像中的位置。

S81第二获取，获取S61聚类中对内部数据的聚类结果，基于聚类结果获取n个聚类中心。

S83第五计算，对于每个聚类中心，计算它与剩余聚类中心之间的欧几里得距离，获得n-1个欧几里得距离（因为每个聚类中心都与其他n-1个距离中心有距离）。
将这些欧几里得距离的数值相加，得到一个总和，将所述总和记为第二数据，本过程将对每个聚类中心重复进行，最终得到n个第二数据。

S86第三存储，由于每个数据子库都与一个指定的第二数据相关联，该第二数据对应一个指定的聚类中心，该聚类中心又对应一种指定的内部数据，因此，通过关联结果能够将内部数据存储到对应的数据子库中。

假设一个低空智能飞行器（以下简称为飞行器）记录了大量的飞行数据，包括飞行高度、速度、温度、压力等参数，以及各个传感器的读数和设备状态信息。
S1数据采集，从飞行器中采集了1000条内部数据记录，每条记录都包含了飞行参数（如高度、速度）、传感器读数（如温度、压力）和设备状态信息（如发动机状态）。
经S61聚类采用K-Means聚类算法对这些内部数据进行处理（设定n=5，即将内部数据划分为5个簇），得到了5个聚类中心，每个聚类中心都代表了一组具有相似特征的内部数据的平均值。
S81第二获取，获取内部数据的聚类结果，获得5个聚类中心。
S82建库，根据K-Means聚类得到的5个聚类中心，构建了5个对应的数据子库。
每个数据子库将用于存储与相应聚类中心相关联的数据点，为了方便管理和引用，将第i个数据子库记为第i数据子库（i=1, 2, 3, 4, 5）。
S83第一计算，对于每个聚类中心（例如第一个聚类中心），计算它与剩余4个聚类中心之间的欧几里得距离，得到4个欧几里得距离的值，之后将这4个欧几里得距离的值相加，得到一个总和，将所述总和记为第二数据。
本步骤对5个聚类中心都进行了计算，最终得到了5个第二数据。
S84第一排序，将得到的5个第二数据按照从小到大的顺序进行排列，得到了一个有序的数据序列，即第二数据序列。
S85关联，将第二数据序列中的每个第二数据与拥有相同序号的数据子库进行关联。例如，第二数据序列为：高度数据、速度数据、温度数据、压力数据、发动机状态数据；高度数据所对应的第二数据与第一数据子库进行了关联，速度数据所对应的第二数据与第二数据子库进行了关联，温度数据所对应的第二数据与第三数据子库进行了关联，压力数据所对应的第二数据与第四数据子库进行了关联，发动机状态数据所对应的第二数据与第五数据子库进行了关联。
S86第三存储，根据关联结果，将原始的内部数据存储到对应的数据子库中。例如，高度数据存储在第一数据子库，速度数据数据存储在第二数据子库，温度数据数据存储在第三数据子库，压力数据数据存储在第四数据子库，发动机状态数据数据存储在第五数据子库。
本实施例涉及从飞行器中获取内部数据（如飞行参数、传感器读数、设备状态信息等），采用K-Means聚类算法处理这些内部数据，得到n个聚类中心，构建n个数据子库，每个数据子库与相应的聚类中心相关联。然后，计算每个聚类中心与其他聚类中心的欧几里得距离，并计算这些欧几里得距离的和，得到n个第二数据，之后将第二数据进行排序，形成第二数据序列。根据第二数据序列中的第二数据所处的位置与数据子库的序号进行关联，最后根据关联结果将内部数据存储到对应的数据子库中，实现数据的动态化存储和高效管理，进而提高低空智能飞行器的内部数据的安全性。

S91数据筛选，对从飞行器的内部数据进行深入分析，以识别出其中的敏感数据和非敏感数据。
对敏感数据的识别方法可以是基于数据特性和用途的识别方法、基于预设的规则的识别方法或基于机器学习模型的识别方法。
例如，基于数据特性和用途的识别过程为：首先，对飞行器内部数据进行详细分类，如飞行参数、人员信息、系统状态数据等。之后根据数据的特性和用途，初步判断哪些数据可能属于敏感数据，哪些属于非敏感数据。之后分析数据的具体内容，如：是否包含个人隐私信息（如姓名、身份证号、联系方式等）、飞行器运营数据（如飞行计划等）或飞行器安全数据（如故障记录、维修记录等）。最后，根据数据的敏感程度，将其划分为敏感数据和非敏感数据。
例如，基于机器学习算法的识别过程为：利用已有的敏感数据和非敏感数据样本训练一个监督学习模型。通过训练后的监督学习模型对内部数据进行识别，判断其是否包含有敏感数据。
例如，基于规则的识别过程为：
首先，制定敏感数据识别规则：根据飞行器内部数据的特性和用途，制定一套敏感数据识别规则，所述识别规则可以包括数据的内容、格式、存储位置等方面的要求。

S93计算距离，对于第a个敏感数据序列，本步骤将计算它与剩余敏感数据序列之间的距离矩阵，距离矩阵中的每个元素代表两个敏感数据序列对应样本之间的某种距离度量（如欧几里得距离、曼哈顿距离等）。所述对应样本可以是在各个敏感数据序列中处于相同序号的样本，也可以是，各个敏感数据序列中采集时间相同的样本。
基于单个距离矩阵，本步骤会进一步计算累计距离矩阵，累计距离矩阵反映了敏感数据序列之间的整体相似性或差异性。
S94确定序列，根据累计距离矩阵，本步骤能够通过动态路径规划算法找出与第a个敏感数据序列相似度最大的敏感数据序列，记为相似数据序列。
S95结合，将第a个敏感数据序列的敏感数据与相似数据序列所对应的非敏感数据进行合并处理，以获得伪装后的第a个数据序列，伪装后的第a个敏感数据序列将被更新为内部数据，之后执行S86第三存储将这些内部数据存储至对应的数据子库中。
S96第一加密，采用对称加密算法对第i数据子库内存储的内部数据进行加密处理，获得第i加密数据与i次加密后的密钥，采用第i加密数据更新第i数据子库内存储的内部数据。
S97第二加密，在S96第一加密的基础上，采用非对称加密算法对第i加密数据与第i+1数据子库内存储的内部数据进行加密处理，获得第i+1加密数据与i+1次加密后的密钥，采用第i+1加密数据更新第i+1数据子库内存储的内部数据。
S98第五判断，判断i+1是否等于n，若是，则不进行后续处理；若否，则说明还有未加密的内部数据，需要执行S99迭代，继续加密。
S99迭代，将第i+1数据子库作为新的第i数据子库，并执行S96第一加密。这个过程将持续进行，直到满足预设的停止条件（如所有数据子库都已加密、达到最大迭代次数等）。
在用户调用相关数据时，首先会验证用户身份，之后从最后一个数据子库开始逆向迭代使用相应解密算法和密钥进行解密，直至所有数据子库解密完成，随后进行数据重组与验证，识别并保护敏感数据。
