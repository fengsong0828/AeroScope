Description

一种基于低空无人飞行器自组网的测控通信方法和系统
技术领域
本发明涉及测控通信领域，具体为一种基于低空无人飞行器自组网的测控通信方法和系统。
背景技术
低空无人飞行器指的是飞行高度在平流层及以下的无人飞行器，包括多用途无人机和四旋翼飞行器等。低空无人飞行器具有控制简单、机动性强、图像清晰、效费比高等优势，对于场地勘察、应急搜寻和通信保障等具有重要的价值。
但是，单个无人飞行器受飞行距离、功能单一等因素影响，在广域条件下勘察和数据中继等任务中所发挥的作用有限。因此，通过低空无人飞行器编组和协同形成集群优势是无人机领域未来的重要发展方向，而飞行器分组协同网络是一个动态性很强的网络，随着部分飞行器的加入或离开网络，网络的拓扑结构会发生快速变化，现有技术中的测控通信系统和方法均无法满足其应用要求。
发明内容
针对现有技术中存在的问题，本发明提供一种作用覆盖范围广，能够协同配合，实现动态稳定的基于低空无人飞行器自组网的测控通信方法和系统。
本发明是通过以下技术方案来实现：
本发明一种基于低空无人飞行器自组网的测控通信方法，包括，
将不同高度不同区域的低空无人飞行器进行自组网分簇；
以簇为单位将低空飞行器分别与空基通信平台和地面通信终端进行联通；
定期对低空无人飞行器自组网内的分簇进行更新和维护。
优选的，在低空无人飞行器之间、低空无人飞行器和空基通信平台之间、低空无人飞行器和地面通信终端之间均采用统一的通信体制结构和协议；统一的通信体制结构和协议包括UDP/IP五层体系结构和统一的调制体制及约定频点；传输层采用UDP协议，应用层协议格式由低空无人飞行器所承担的具体任务来决定。
优选的，采用AdHoc网络中的最大连接度算法对低空无人飞行器自组网进行分簇，地面通信站将通信服务需求送达附近的簇，在该簇内进行处理或对外转发。
进一步，最大连接度分簇算法的步骤如下，
1)将低空无人飞行器作为自组网中的节点并给予相应的ID编号，标记所有节点为普通节点；
2)当一个普通节点发现它是邻居普通节点中节点度最大的节点时，它成为簇首，并标记自己为簇首；当节点度相同时选择ID编号较小的节点为簇首，其中邻居节点中的两个节点间距离满足互相通信需求；
3)簇首的邻居标记为簇首的簇内节点；
4)重复步骤2)和3)，直到网络中不存在普通节点；
5)每隔一个周期T后进行步骤1)～4)。
进一步，对外转发时，通过对其他簇或空基通信平台进行转发，采用整体链路性能最佳优先路由算法选择转发路径；包括如下步骤，
a.将通信任务数据分为两类，一类是包括测量信息和控制指令的非图像数据，一类是包括音频、视频和图像的图像数据；
b.采用最短路径路由算法，计算源端到目的端的多条备选路径；
c.计算各备选路径的误码率和带宽平均值；
d.对于非图像数据，选择最低误码率的路径转发，对于图像数据，选取具有最高带宽的路径转发。
进一步，对外转发时，通过对其他簇或空基通信平台进行转发，根据自组网结构采用基于开放式最短路径优先协议、基于簇的多路径动态源路由算法、基于簇的QoS多路径路由算法或最大可靠性多路径路由算法选择转发路径。
优选的，根据应用环境选择最小ID启发式算法、节点权重启发式算法或自适应按需加权算法对低空无人飞行器自组网进行分簇。
本发明一种基于低空无人飞行器自组网的测控通信系统，以上任意一中方案中所述的测控通信方法，其包括空基通信平台和多个低空无人飞行器；空基通信平台用于与低空飞行器、地面通信终端和其他空基通信平台进行通信；低空飞行器上搭载任务载荷，用于与空基通信平台、地面通信终端和其他低空无人飞行器进行通信。
优选的，低空无人飞行器之间、低空无人飞行器和空基通信平台之间、低空无人飞行器和地面通信站之间均采用的统一的通信体制结构；统一的通信体制结构包括UDP/IP五层体系结构、统一的约定频点和调制体制。
进一步，调制体制采用正交频分复用调制体制。
与现有技术相比，本发明具有以下有益的技术效果：
本发明将空基通信平台和各低空无人飞行器组织起来，形成两层复合立体信息网，既能完成某些广域大范围勘察和通信中继任务，又能完成局域范围内的各项任务，既能完成空中通信中继，又能完成空中与地面的通信，因此本发明所述系统具备多功能性；通过多个低空无人飞行器进行自组网，避免了各飞行器负载不均衡引起的通信拥塞，按簇作为通信和信息处理的单元，提高了系统整体效率，使其具备高效性；通过定期对簇的更新和维护，使其实现移动自组网的组织方式，分簇数和簇内结构是动态调整的，具有高度的冗余备份性能，避免了低空飞行器的变化对组网带来的影响，使个别低空无人飞行器的损坏都不会对系统造成大的影响，可靠性高，稳定性强。
进一步的，通过采用包括统一的调制体制和约定频点的统一的通信体制结构，能够优化低空飞行器之间的通信和协同，提供通信效率和数据吞吐能力。
进一步的，利用最大连接度分簇算法进行自组网分簇，能够减少簇数量，从而减少分组投递的延迟，保证了通信传输的效率。采用整体链路性能最佳优先路由算法不仅能够兼顾通过中继节点所引起的时延，也就是路径长度，而且能够满足不同的通信任务对链路的不同需求。
进一步的，通过提供其他不同方法的自组网分簇，以及多种选择转发路径的方法，从而满足和适应各种实际的使用情况，使用范围广。

具体实施方式
下面结合具体的实施例对本发明做进一步的详细说明，所述是对本发明的解释而不是限定。
本发明将空基通信平台和多个低空飞行器进行联通，在组织和通信规则下形成由高到低的两层复合立体信息网，空基通信平台形成高层信息枢纽，部署在不同高度不同区域的低空无人飞行器通过定期的分簇完成自组网，从而形成动态稳定的低层信息网，以簇为基本单位在自组网内完成场地勘察、信息中继和信息处理等任务；不仅能够对低空飞行器通过分簇进行有效的整合组织，而且能够利用定期的对分簇进行更新和维护，满足了低空飞行器位置的动态变化。
本发明中的测控通信系统，如图1所示，由空基通信平台和多个低空无人飞行器组成。空基通信平台采用高空浮空器，空基通信平台承担来自低空飞行器、地面通信站和其他空基通信平台的通信任务需求。低空飞行器采用搭载测控、通信和对地观测等任务载荷的各种形式的低空低速飞行器，低空无人飞行器能够承担的任务包括：与地面终端通信、与其他低空无人飞行器通信、对地进行观测、搜寻和勘察等，低空无人飞行器的实体可以是多用途无人机或四旋翼飞行器等。
在系统中采用了统一的通信体制结构和协议。低空无人飞行器之间、低空无人飞行器和空基通信平台之间、低空无人飞行器和地面通信站之间均采用的统一的通信体制结构：UDP/IP五层体系结构，如图2所示；并且采用统一的调制体制和约定频点。
如图2所示，UDP/IP五层体系结构中的传输层选择UDP协议，能够非常好的适用于链路拓扑结构变化剧烈，不利于建立固定链路的情况，提高了通信效率。应用层协议格式则根据低空无人飞行器所承担的具体任务来决定，不同的任务有不同的数据格式，如定位数据格式、图像数据格式、光学数据格式等。
采用统一的调制体制和约定频点有利于飞行器之间的通信和协同。统一的调制体制指的是飞行器和用户统一采用的调制体制，本优选实例中采用正交频分复用调制(OFDM)。各飞行器除了搭载各自的任务载荷外，还装有数据接收天线和发送天线，既要执行各自的任务，又要执行通信中继或接受简单的数据处理任务。
系统中的低空无人飞行器进行自组网，如图1所示，根据低空无人飞行器组网特点，采用AdHoc网络中的最大连接度算法对低空无人飞行器自组网进行分簇，地面通信站将通信服务需求送达附近的簇，在簇内进行处理或对外转发，对其他簇或空基通信平台进行转发时采用整体链路性能最佳优先路由算法。
其中，最大连接度分簇算法的原理是：在一群相邻节点之间，选择连接度高的节点作为簇首，其一跳邻居节点成为该簇的成员节点。算法的具体步骤是：1)将低空无人飞行器作为自组网中的节点并给予相应的ID编号，标记所有节点为普通节点；2)当一个普通节点发现它是邻居普通节点中节点度最大的节点时，它成为簇首，并标记自己为簇首，当节点度相同时选择ID编号较小的节点为簇首，其中邻居节点的定义是两个节点间的距离满足互相通信需求；3)簇首的邻居标记为簇首的簇内节点；4)重复步骤2)和3)，直到网络中不存在普通节点；5)为了避免低空无人飞行器的数量变化为组网带来影响，每隔一个周期T后需要进行步骤1)～4)。该算法的特点是簇数目较少，减少了分组投递时延。分簇算法还能够根据应用环境选择：最小ID启发式算法、节点权重启发式算法或自适应按需加权算法。
其中，整体链路性能最佳优先路由算法兼顾了通过中继节点所引起的时延和不同的通信任务，也就是路径长度和不同的通信任务对链路的不同需求。首先，将通信任务数据分为两类：一类为非图像数据，包括测量信息和控制指令，此类数据的数据量小，在B(字节)数量级，对链路的带宽和传输速率要求不高，但要求低误码率和高实时性；一类为图像数据，包括音频、视频和图像等，此类数据的数据量大，在MB(兆字节)数量级，对链路的带宽和传输速率要求较高，但对误码率和时延要求不高。然后通过如下的步骤搜索最优路径，先采用较为成熟的最短路径路由算法，计算源端到目的端的多条备选路径；再计算几条备选路径的误码率和带宽平均值；最后对于非图像数据，选择低误码率的路径，对于图像数据，选取具有高带宽的路径。路由算法还能够根据自组网结构可选择：基于开放式最短路径优先协议(OSPF)、基于簇的多路径动态源路由算法、基于簇的QoS多路径路由算法或最大可靠性多路径路由算法。
本发明的系统由多个低空无人飞行器和空基通信平台组成，系统生成过程就是低空无人飞行器自组网过程，自组网算法采用最大连接度算法，分簇数为N，并且处在动态变化中。其中低空无人飞行器实现的自组织网络是一种移动通信网络，具有无中心、自组织、多跳路由、分布控制、节点移动等特点，能够被快速地部署和重新配置来满足特定应用的需要，提高了网络构成的灵活性。本发明通过将不同高度不同区域的低空无人飞行器进行自组网，建立分簇组网规则和路由算法，采用统一的通信协议和体制，结合空基通信平台形成一套双层综合测控通信系统，协同完成地质勘察、应急搜寻和通信保障等任务。
其具体实施过程是：(1)将低空无人飞行器作为自组网中的节点并给予相应的ID编号，标记所有节点为普通节点；(2)当一个普通节点发现它是邻居普通节点中节点度最大的节点时，它成为簇首，并标记自己为簇首，当节点度相同时选择ID编号较小的节点为簇首，其中邻居节点的定义是两个节点间的距离满足互相通信需求；(3)簇首的邻居标记为簇首的簇内节点；(4)重复步骤(2)和(3)，直到网络中不存在普通节点。系统形成后还需定期地进行成员更新和维护，需要对可用的所有低空无人飞行器进行重新自组网，即重新进行步骤(1)～(4)。
该系统内每个簇的簇首负责：(1)进行任务分配，根据簇内各成员的功能和负载情况分配本簇收到的任务申请，避免节点拥塞；(2)与其他簇首或空基通信平台进行通信，计算簇与簇之间或簇与空基通信平台之间的通信路由算法。低空无人飞行器之间、低空无人飞行器和空基通信平台之间、低空无人飞行器和地面通信站之间均采用的统一的通信体制结构：UDP/IP五层体系结构，采用统一的约定频点和正交频分复用调制(OFDM)体制。
利用本发明所述的系统和方法在执行不同任务时，如以下两个实施例。
实施例一：某项长距离通信中继任务。
某集团业务需要分属两个不同区域的两个节点群共同完成，两个区域间通信需要长距离的无线数据中继，中继路径需要通过3个行政区，分别为L1～L3，经过3个低空无人飞行器分簇，命名为C1～C3。如图1所示，L1行政区中，某数据终端将某数据序列通过地面路由发送到通信站1，通信站1通过无线发射天线发送到簇C1中某个具有通信中继功能的飞行器节点，如低空无人飞行器3，该节点将通信中继需求传递给簇C1的簇首，如低空无人飞行器1，簇首根据整体链路性能最佳优先路由算法计算出多条备选路径后，根据所传递数据类型优选得到数据传递的路由序列，根据路由序列该数据将经过某空基通信平台、簇C3的传递到位于L3的通信站2，通信站2通过地面路由将数据发送到控制中心，控制中心经过挑帧解码得到所需的测量数据序列。簇C3中具体承担中继任务的探测器由簇C3的簇首根据负载情况指定较闲的探测器执行，如簇C3中的簇首为低空无人飞行器8，将此任务指定到低空无人飞行器11进行执行。
从而通过基于低空无人飞行器自组网的测控通信系统完成了远程数据中继任务。
实施例二：某项广域地质勘察任务。
对于某项大范围地质勘察任务，单个特定的飞行器监视范围小且代价高，需要多个低空无人飞行器共同完成，由于低空无人飞行器在不断的运动，目标区域上方的飞行器数量在不断变化，需要采用本发明所述的方法和系统来完成。在某个时刻，假设目标区域由3个分簇覆盖，分别命名为C1～C3，各分簇所获取的光学图像数据流分别为P1～P3。簇C1和C2的簇首根据整体链路性能最佳优先路由算法计算出多条备选路径后，根据所传递数据类型优选得到数据传递的路由序列，P1和P2被按各自的路由序列传递到簇C3，由簇C3簇首将三个数据流发送任务分解给多个探测器共同完成，地面通信站收到数据流后转发到指挥中心处理。指挥中心经过时标对齐、空间配准、图像配准后可得到目标区域的实时监控图像，由于P1～P3数据流是在不同高度获取的，因此地质勘察任务可以在目标区域全局和局部展开。
