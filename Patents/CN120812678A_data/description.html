Description

一种面向低空飞行器与地面网络深度融合的多模通信模块及 其组网与动态切换方法
技术领域
本申请涉及低空飞行器通信技术领域，尤其涉及一种面向低空飞行器与地面网络深度融合的多模通信模块及其组网与动态切换方法。
背景技术
随着低空领域的开放和低空飞行器数量的不断增加，低空飞行器与地面网络之间的通信需求也日益增长。传统的通信方式往往存在通信不稳定、覆盖范围有限等问题，无法满足低空飞行器在复杂环境下的通信需求。
现有的低空飞行器通信系统大多采用单一的通信模式，如卫星通信、移动通信网络等，这些通信模式在特定的环境下可能表现出良好的通信性能，但在复杂多变的低空环境中，其通信稳定性和覆盖范围往往受到限制。
上述内容仅用于辅助理解本申请的技术方案，并不代表承认上述内容是现有技术。
发明内容
本申请的主要目的在于提供一种面向低空飞行器与地面网络深度融合的多模通信模块及其组网与动态切换方法，旨在提升低空飞行器在复杂环境下的通信稳定性、以及实现智能化资源调配和快速切换。
为实现上述目的，本申请提出一种面向低空飞行器与地面网络深度融合的多模通信模块组网与动态切换方法，所述多模通信模块设置于所述低空飞行器，所述多模通信模块包括定位单元、多模射频单元以及FPGA硬触发单元，所述的方法包括：
通过所述定位单元获取所述低空飞行器的定位数据，并通过所述多模射频单元采集链路质量数据；
基于所述定位数据进行动态阈值计算，得到自适应切换阈值；
将所述链路质量数据与定位数据输入预切换预测模型，输出网络盲区预测数据；
基于所述网络盲区预测数据生成预分配资源指令，并发送至地面专网及Mesh自组网节点；
当所述链路质量数据低于所述自适应切换阈值或网络盲区预测数据满足预设切换条件时，基于所述FPGA硬触发单元触发FPGA硬切换指令；
执行所述FPGA硬切换指令，以调用所述预分配资源指令完成双模链路切换，并输出对应的通信状态反馈数据。
在一实施例中，所述基于所述定位数据进行动态阈值计算，得到自适应切换阈值的步骤包括：
采集电磁环境扫描数据，并解析出频段干扰强度值；
解析所述定位数据中的高度值和速度值，结合预设场景分类规则生成环境类型标识；
基于所述环境类型标识调用对应的基线阈值模板，得到对应的初始切换阈值；
融合频段干扰强度值与初始切换阈值，以输出自适应切换阈值。
在一实施例中，所述将所述链路质量数据与定位数据输入预切换预测模型，输出网络盲区预测数据的步骤包括：
将所述定位数据与预设数字高程地图进行三维坐标匹配，输出地形衰减系数；
将所述链路质量数据分解为信号强度值数据和丢包率数据；
基于地形衰减系数对信号强度值数据进行衰减补偿，生成修正信号强度数据；
当所述修正信号强度数据低于预设第一门限或丢包率数据高于预设第二门限时，生成网络盲区预测数据。
在一实施例中，当所述修正信号强度数据低于预设第一门限或丢包率数据高于预设第二门限时，生成网络盲区预测数据的步骤包括：
对修正信号强度数据进行滑动窗口均值滤波，得到稳定信号强度数据；
计算丢包率数据在预设时间窗口内的丢包变化率数据；
若稳定信号强度数据在连续的预设第一数量的采样周期内均低于预设第一门限且丢包变化率数据高于预设第二门限，生成网络盲区预测数据。
在一实施例中，所述基于所述网络盲区预测数据生成预分配资源指令，并发送至地面专网及Mesh自组网节点的步骤包括：
解析网络盲区预测数据中的目标网络类型标识，生成候选节点列表数据；
向所述候选节点列表数据中的节点发送资源预留请求，接收各节点的可用带宽数据；
基于所述可用带宽数据和预设业务优先级选择最优节点，生成预分配资源指令，并发送至地面专网及Mesh自组网节点。
在一实施例中，所述当所述链路质量数据低于所述自适应切换阈值或网络盲区预测数据满足预设切换条件时，基于所述FPGA硬触发单元触发FPGA硬切换指令的步骤包括：
实时比对链路质量数据与自适应切换阈值，当链路质量数据小于自适应切换阈值时生成第一触发信号；
当网络盲区预测数据指示所述低空飞行器将在未来第一预设时长内进入盲区时，生成第二触发信号；
通过所述FPGA硬触发单元的FPGA硬件逻辑电路对第一触发信号或第二触发信号进行优先级仲裁，输出FPGA硬切换指令。
在一实施例中，所述通过所述FPGA硬触发单元的FPGA硬件逻辑电路对第一触发信号或第二触发信号进行优先级仲裁，输出FPGA硬切换指令的步骤包括：
当第一触发信号与第二触发信号同时存在时，优先响应第一触发信号；
通过FPGA硬触发单元内置的硬件计时器记录第二触发信号的持续时长，当第二触发信号持续超过第二预设时长时提升其优先级；
基于优先级仲裁的仲裁结果生成包含目标网络ID和加密策略标识的FPGA硬切换指令。
在一实施例中，所述执行所述FPGA硬切换指令，以调用所述预分配资源指令完成双模链路切换，并输出对应的通信状态反馈数据的步骤包括：
解析所述FPGA硬切换指令中的加密策略标识，激活对应加密引擎；
根据预分配资源指令建立目标链路物理层连接，并保留源链路会话层状态；
在数据传输中断前将缓存数据通过DMA方式拷贝至目标链路缓冲区；
完成双链路并行传输后关闭源链路，并输出通信状态反馈数据。
在一实施例中，所述根据预分配资源指令建立目标链路物理层连接，并保留源链路会话层状态的步骤包括：
解析预分配资源指令中的目标网络节点信息，生成物理层连接参数组；
根据物理层连接参数组配置多模射频前端的硬件寄存器，建立目标链路物理层连接；
创建双协议栈镜像表，实时同步源链路的TCP/UDP会话状态数据；
通过所述FPGA硬触发单元控制多模射频单元，以完成通道切换；
检测目标链路信号强度，当连续预设第二数量的采样点的信号强度大于预设信号强度时激活目标链路。
此外，为实现上述目的，本申请还提出一种面向低空飞行器与地面网络深度融合的多模通信模块，所述多模通信模块设置于所述低空飞行器，所述多模通信模块包括定位单元、多模射频单元以及FPGA硬触发单元，所述多模通信模块使用了所述的面向低空飞行器与地面网络深度融合的多模通信模块组网与动态切换方法的步骤。
本申请提供的一种面向低空飞行器与地面网络深度融合的多模通信模块组网与动态切换方法及多模通信模块，通过动态阈值计算、预切换预测模型和FPGA硬触发机制，实现了基于飞行器实时状态和环境变化的智能通信资源调配，在链路质量下降或预测进入盲区时快速触发双模切换，有效解决了传统通信方式在复杂低空环境中稳定性差、切换效率低的问题，能够提升通信可靠性和保障数据传输连续性。

在现有技术中，低空飞行器通信系统普遍采用单一通信模式，例如卫星通信或移动通信网络。这类通信方式在特定场景下虽能维持基本通信需求，但在复杂多变的低空环境中存在明显缺陷。例如在山区、城市建筑群或电磁干扰密集区域，飞行器可能频繁遭遇信号衰减、网络覆盖盲区等问题，导致通信链路中断或数据传输延迟。传统方法缺乏动态调整机制，无法根据实时环境变化切换通信模式，难以保障低空飞行器在复杂空域中的持续通信能力。
为了解决上述问题，需要设计一种能够适应动态环境的通信切换机制。首先考虑如何准确评估当前通信链路质量，传统方法仅依赖信号强度单一指标，无法反映地形遮挡或电磁干扰的综合影响。进一步思考发现，引入定位数据可预测飞行器即将进入的盲区区域，但现有系统未将位置信息与链路质量进行联合分析。此外，软件触发的切换方式存在延迟过高的问题，难以满足低空飞行器高速移动场景下的实时性要求。基于这些分析，解决方案需要集成多维度数据评估、网络盲区预测以及硬件级快速触发机制。

步骤S100，通过所述定位单元获取所述低空飞行器的定位数据，并通过所述多模射频单元采集链路质量数据；
步骤S200，基于所述定位数据进行动态阈值计算，得到自适应切换阈值；
步骤S300，将所述链路质量数据与定位数据输入预切换预测模型，输出网络盲区预测数据；
步骤S400，基于所述网络盲区预测数据生成预分配资源指令，并发送至地面专网及Mesh自组网节点；

步骤S600，执行所述FPGA硬切换指令，以调用所述预分配资源指令完成双模链路切换，并输出对应的通信状态反馈数据。
本实施例中，定位数据是指通过定位单元的卫星导航系统或惯性导航系统获取的三维坐标、高度及运动轨迹信息，具体可以采用GPS模块与IMU传感器融合实现，用于分析飞行器所处环境的地形特征与电磁干扰分布。链路质量数据是指通信链路的信号强度、误码率及传输延迟参数，具体可以通过多模射频单元的射频前端芯片的物理层测量电路采集，用于实时评估当前通信网络的稳定性。动态阈值计算是指根据飞行器高度、速度及环境干扰强度动态调整的切换触发条件，具体可以采用加权算法融合电磁扫描数据与预设场景分类规则，解决固定阈值无法适应复杂环境的问题。预切换预测模型是指基于数字高程地图与历史链路数据的机器学习模型，具体可以采用卷积神经网络对地形衰减系数进行补偿计算，提前识别可能出现的网络盲区。FPGA硬触发单元是指基于可编程逻辑器件构建的硬件级切换控制器，具体可以通过硬件描述语言设计优先级仲裁电路，实现微秒级切换响应速度，避免软件中断处理带来的延迟。
本实施例中，飞行器在飞行过程中持续采集定位信息与链路质量参数。定位数据输入动态阈值计算模块，结合飞行高度、运动速度及实时扫描的电磁干扰强度，生成随环境变化的自适应切换阈值。同时，链路质量数据与定位坐标输入预切换预测模型，通过三维地形匹配修正信号衰减值，当修正后的信号强度持续低于门限或丢包率异常升高时，判定即将进入网络盲区。地面网络节点根据预测结果提前分配通信资源，建立备用链路通道。当实时链路质量低于动态阈值或预测模型触发切换条件时，FPGA硬件电路直接控制射频单元切换通信模式，同步调用预分配资源完成双模链路无缝切换，并通过状态反馈机制验证切换有效性。
本实施例中，通过融合多维度环境参数实现阈值动态调整，结合地形匹配的预测模型提前识别盲区，采用硬件级触发机制将降低切换延迟，显著提升高速移动场景下的通信连续性。如此，本申请有效解决了低空复杂环境下通信链路不稳定的问题。动态阈值机制适应不同高度与速度下的信号衰减特性，预测模型提前识别网络盲区，以延长资源预分配时间窗口。此外，多模协同工作模式还能够提升通信覆盖率，满足低空飞行器在城区、山区等复杂场景下的可靠通信需求。
在一种可行的实施方式中，所述基于所述定位数据进行动态阈值计算，得到自适应切换阈值的步骤包括：采集电磁环境扫描数据，并解析出频段干扰强度值；解析所述定位数据中的高度值和速度值，结合预设场景分类规则生成环境类型标识；基于所述环境类型标识调用对应的基线阈值模板，得到对应的初始切换阈值；融合频段干扰强度值与初始切换阈值，以输出自适应切换阈值。

本实施例中，在低空飞行器运行过程中，通过多模射频单元实时采集当前所处环境的电磁频谱数据，利用信号处理算法提取特定通信频段的干扰强度分布。同时解析飞行器定位数据中的高度参数和运动速度参数，结合预设的飞行场景分类规则，例如当高度低于100米且速度小于20米/秒时判定为城区复杂环境，生成对应的环境类型标识。根据该标识调用存储在本地数据库中的对应阈值模板，例如城区环境对应的初始切换阈值为-85dBm。进一步将实时测量的频段干扰强度值通过加权算法与初始阈值进行融合计算，例如当检测到2.4GHz频段存在持续干扰时，将切换阈值动态调整为-80dBm，从而形成适应实际电磁环境变化的自适应切换阈值。

在一种可行的实施方式中，所述根据预分配资源指令建立目标链路物理层连接，并保留源链路会话层状态的步骤包括：解析预分配资源指令中的目标网络节点信息，生成物理层连接参数组；根据物理层连接参数组配置多模射频前端的硬件寄存器，建立目标链路物理层连接；创建双协议栈镜像表，实时同步源链路的TCP/UDP会话状态数据；通过所述FPGA硬触发单元控制多模射频单元，以完成通道切换；检测目标链路信号强度，当连续预设第二数量的采样点的信号强度大于预设信号强度时激活目标链路。
本实施例中，物理层连接参数组是指包含目标网络节点通信频率、调制方式及功率等级的配置参数集合，具体可采用XML格式的配置文件进行存储，通过解析预分配资源指令中的节点IP地址与端口号动态生成，用于快速配置射频前端硬件寄存器。双协议栈镜像表是指同时存储源链路与目标链路传输层协议状态的数据结构，具体可通过FPGA内部的双端口RAM实现实时同步，确保会话层状态在切换过程中不丢失。预设数量的采样点信号强度阈值是指用于判断目标链路稳定性的触发条件，例如可设置为连续3个采样周期内信号强度大于-80dBm，通过硬件计数器实现快速检测。
本实施例中，在接收到预分配资源指令后，目标网络节点的IP地址与通信参数被提取并转换为物理层连接参数组，例如包含载波频率2.4GHz、QPSK调制方式及20dBm发射功率。多模射频前端的硬件寄存器根据该参数组进行配置，例如通过SPI接口写入频率合成器寄存器，完成目标链路物理层连接初始化。双协议栈镜像表在FPGA内部创建，实时记录源链路的TCP窗口大小、序列号及UDP端口状态，并通过硬件逻辑实现数据同步。当FPGA硬触发单元检测到目标链路信号强度在连续3个采样周期内达到-75dBm时，立即激活目标链路数据传输通道。
在一些具体实施方式中，硬件寄存器配置过程可采用DMA传输方式将物理层连接参数组直接写入射频前端芯片，例如使用STM32系列处理器的DMA控制器实现零延迟配置。双协议栈镜像表的同步机制可通过硬件状态机实现，例如在FPGA中设计专用状态机实时捕获源链路的ACK报文并更新镜像表数据。信号强度检测模块可集成RSSI检测电路，例如采用AD8362芯片实现信号强度模拟量到数字量的快速转换。
本实施例中，通过双协议栈镜像表实时同步会话状态，可使传输层协议在链路切换后直接沿用原有会话参数，例如TCP序列号保持连续递增，避免因会话重建产生的数据包重传。物理层参数的硬件级快速配置有效降低了链路切换时延，避免了因软件协议栈处理产生的性能瓶颈。目标链路信号强度的多周期检测机制则提升了链路激活判断的准确性，防止因瞬时信号波动导致的误切换操作。
