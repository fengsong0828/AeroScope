Description

一种基于低空飞行器的应急通信方法
技术领域
本发明涉及通信技术领域，尤其涉及一种基于低空飞行器的应急通信方法。
背景技术
低空飞行器在应急通信领域具有关键作用，特别是在自然灾害、突发事件等场景中，能够快速部署并提供通信支持，确保救援行动高效展开。低空飞行器的灵活机动性和覆盖范围使其成为地面通信受限时的理想解决方案。然而，现有应急通信方法在应对低空飞行器复杂运行环境时存在显著局限。传统方案通常依赖固定参数的信号调制与传输方式，难以适应飞行器在不同速度和环境下的动态需求，尤其是在快速机动或复杂地形中，通信链路容易中断或性能下降。这导致救援信息传递延迟，甚至影响生命救援的及时性。核心挑战在于如何实现通信系统对飞行器速度变化的实时适应性。
低空飞行器在应急场景中常需快速切换飞行状态，例如从悬停到高速机动，这要求通信系统能够在极短时间内调整信号调制参数以维持稳定连接。然而，速度变化不仅影响信号传输的稳定性，还对传输功率的动态分配提出更高要求。如果系统无法快速响应速度变化，信号质量会下降，例如在高速飞行中可能出现数据丢包，救援指令无法及时传递。进一步地，快速调整调制参数需要高效的信号处理能力，而现有技术在处理速度与资源消耗之间难以平衡。例如，在一次山地救援中，飞行器需在10秒内从静止加速至50米每秒，同时保持与地面指挥中心的实时视频传输，若调制参数切换滞后，通信中断将导致救援指令丢失。因此，如何设计一种能够根据低空飞行器速度变化实时调整信号调制参数和传输功率的通信方法，以确保在高速机动场景下仍能维持稳定数据链路，成为本研究的关键问题。
发明内容
针对飞行器在复杂地形中因速度变化和环境干扰导致通信链路不稳定、数据丢包率高、指令传递时延大的业务场景问题，本发明提出了一种基于低空飞行器的应急通信方法。
一种基于低空飞行器的应急通信方法，包括：
获取飞行器的实时速度数据和环境参数，确定飞行器速度变化和复杂地形适应需求；
根据飞行器速度变化，采用自适应调制编码算法调整信号调制参数，得到优化的调制方案，从所述优化的调制方案中提取信号调制参数，通过功率控制算法动态分配传输功率，确定传输功率分配方案，并计算信号调制参数和传输功率分配，得到更新后的通信配置；
采用更新后的通信配置，执行实时视频传输任务，判断数据丢包率是否满足救援指令传递要求，获得优化的信号处理方案；
根据优化的信号处理方案，重新配置通信链路，降低参数调整时延，确定稳定的数据传输通道，从稳定的数据传输通道中提取实时视频数据，获取压缩后的救援指令数据；
通过压缩后的救援指令数据，执行指令传递任务，判断通信链路稳定性是否满足快速机动需求，进而判断最终的通信链路稳定性。
与现有技术相比，本发明具有如下优点和技术效果：
本发明通过惯性测量单元与全球定位系统融合获取飞行器速度与环境参数，采用自适应调制编码算法动态调整信号调制参数，结合功率控制算法优化传输功率分配，确保通信链路稳定性。当链路稳定性或丢包率未达标时，本发明利用快速傅里叶变换分析频域特征，重新配置调制参数与功率分配，降低调整时延，构建稳定数据传输通道，最终通过数据压缩算法生成高效救援指令数据。本发明显著提升了复杂地形下通信链路的稳定性和指令传递效率，满足快速机动需求，为救援任务提供可靠支持。

本实施例提出了一种基于低空飞行器的应急通信方法，如图1，包括：
获取飞行器的实时速度数据和环境参数，确定飞行器速度变化和复杂地形适应需求；

采用更新后的通信配置，执行实时视频传输任务，判断数据丢包率是否满足救援指令传递要求，获得优化的信号处理方案；
根据优化的信号处理方案，重新配置通信链路，降低参数调整时延，确定稳定的数据传输通道，从稳定的数据传输通道中提取实时视频数据，获取压缩后的救援指令数据；
通过压缩后的救援指令数据，执行指令传递任务，判断通信链路稳定性是否满足快速机动需求，进而判断最终的通信链路稳定性。
进一步地，获取飞行器的实时速度数据和环境参数，确定飞行器速度变化和复杂地形适应需求，包括：
从飞行器传感器获取实时速度数据和环境参数，采用惯性测量单元和全球定位系统进行数据采集，得到原始传感器数据集，采用卡尔曼滤波算法对原始传感器数据集进行融合处理，结合惯性测量单元的加速度数据和全球定位系统的位置数据，得到平滑速度估计值；
通过平滑速度估计值计算飞行器速度变化率，采用差分方法对连续时间点的数据进行比较，确定速度变化趋势；
根据速度变化趋势和环境参数中的地形高度数据，采用地形跟随算法计算飞行器与复杂地形的相对高度差，得到地形适应性参数，若地形适应性参数超过预设适应性阈值，则通过动态规划算法调整飞行器的速度和高度。
具体地，在无人机执行复杂地形巡航任务时，惯性测量单元(IMU)和全球定位系统(GPS)协同采集实时数据。IMU提供加速度和角速度数据，例如每秒10次采样，得到加速度值如2.5m/s2；GPS提供经纬度和高度数据，例如每秒1次采样，精度为±2米。这些原始数据存在噪声，如IMU的漂移误差或GPS的定位抖动。卡尔曼滤波算法融合两类数据，通过预测和更新步骤，综合加速度和位置信息，生成平滑速度估计值。
例如，无人机在某时刻的估计速度为15m/s，较原始数据噪声降低约30％，显著提升速度数据的稳定性。具体而言，平滑速度估计值用于计算速度变化率。采用差分方法，比较连续时间点(如0.1秒间隔)的速度数据，得出变化率。例如，速度从15m/s增加到16m/s，变化率为10m/s2，表明加速趋势。结合环境参数中的地形高度数据，如通过激光雷达获取的地形高度每秒更新，精度为±0.1米，地形跟随算法计算无人机与地形的相对高度差。无人机飞行高度为50米，地形高度为30米，相对高度差为20米。地形适应性参数基于此高度差和速度变化率生成，若参数超过阈值(如高度差小于15米)，触发调整。
进一步地，采用自适应调制编码算法调整信号调制参数，得到优化的调制方案，包括：
根据飞行器速度变化，获取实时速度数据，采用自适应调制编码算法计算调制参数，得到初步调制方案，通过初步调制方案，结合环境适应属性，分析通信质量数据，确定调制参数调整方向；
若通信质量数据低于预设通信质量阈值，则采用正交幅度调制方法调整信号调制，得到优化后的调制参数，根据优化后的调制参数，生成对应的编码方案，获得实时调整后的信号输出；
通过实时调整后的信号输出，采集通信链路反馈数据，判断信号传输稳定性；若信号传输稳定性未达到预设稳定性阈值，则采用最小均方误差算法更新调制参数，获得所述优化的调制方案。
具体地，在飞行器执行复杂地形巡航任务时，实时速度数据的获取与调制参数的优化是确保通信链路稳定的关键。飞行器通过惯性测量单元和全球定位系统协同工作，获取实时速度数据。例如，惯性测量单元每秒采样20次，提供加速度数据，如3.2m/s2；全球定位系统每秒采样2次，提供位置数据，精度为±1.5米。这些数据为自适应调制编码算法提供基础输入。算法根据速度变化，动态计算调制参数。例如，飞行器速度从10m/s增至12m/s，算法初步选择16-QAM调制方案，以平衡数据速率和抗噪能力。
在一种可能的实现方式中，结合环境适应属性，如地形高度变化或电磁干扰强度，分析通信质量数据。通信质量通过误码率和信号强度评估，例如误码率阈值为0.01，信号强度阈值为-80dBm。若误码率达到0.015或信号强度降至-85dBm，表明通信质量低于预期，需调整调制参数。
通过优化后的调制参数，生成对应的编码方案，如采用低密度奇偶校验码，确保数据传输的可靠性。实时调整后的信号输出通过通信链路传输，采集反馈数据，如信号传输延迟和丢包率。假设延迟为50ms，丢包率为0.5％，低于预设阈值，表明信号传输稳定。若延迟增至80ms或丢包率升至1％，则采用最小均方误差算法更新调制参数。
进一步地，从所述优化的调制方案中提取信号调制参数，通过功率控制算法动态分配传输功率，确定传输功率分配方案，包括：
从所述优化的调制方案中提取信号调制参数，采用功率控制算法计算初始功率分配，得到初步功率分配方案，根据初步功率分配方案，结合环境适应属性，采集实时通信质量数据，判断功率分配调整方向；
根据所述功率分配调整方向，生成对应的信号编码指令，获得实时调整后的信号输出，采集实时调整后的信号输出反馈数据，判断信号传输稳定性；
若信号传输稳定性未达到预设稳定性阈值，则采用最小均方误差算法更新功率分配参数，确定所述传输功率分配方案。
具体地，在飞行器通信领域，优化信号调制参数后，功率控制是确保通信链路高效稳定的关键环节。信号调制参数的提取为功率分配提供基础，功率控制算法通过分析实时速度和环境数据，计算初始功率分配方案。例如，飞行器以15m/s速度巡航，惯性测量单元提供加速度数据2.5m/s2，全球定位系统提供位置精度±1.2m，算法据此分配初始发射功率为20dBm，以满足基本通信需求。结合环境适应属性，如地形起伏或天气干扰，采集实时通信质量数据是判断功率调整方向的基础。
若通信质量未达标，采用正交幅度调制方法调整功率分配。具体而言，算法将功率从20dBm提升至22dBm，降低误码率至0.009，增强信号鲁棒性。调整后的功率分配方案生成信号编码指令，例如采用卷积码，生成实时信号输出。实时调整后的信号输出需通过反馈数据验证稳定性。例如，采集到的传输延迟为55ms，丢包率为0.4％，符合预设阈值60ms和0.5％。若反馈数据表明延迟增至75ms，丢包率升至0.8％，则采用最小均方误差算法更新功率分配参数。
进一步地，计算信号调制参数和传输功率分配，得到更新后的通信配置，包括：
通过所述传输功率分配方案，结合实时环境响应数据，计算传输功率分配比例，确定初步功率配置，基于所述初步功率配置的通信质量采用最大似然估计算法更新功率分配参数，得到调整后的功率分配方案；
根据调整后的功率分配方案，生成对应的信号编码指令，获得实时调整后的信号输出，采集实时调整后的信号输出反馈数据，判断信号传输稳定性，得到所述更新后的通信配置。
具体地，基于环境响应参数，算法将载波频率划分为多个子载波，每个子载波带宽为200kHz，调制方式选择16-QAM，以适应高海拔区域的信号衰减特性。优化后的调制方案生成后，结合实时环境响应数据计算传输功率分配比例。
一种实施例中，算法根据信号衰减率和多径效应，初步分配发射功率为18dBm，其中70％功率分配给主要子载波，30％分配给辅助子载波，以确保信号覆盖范围。若初步功率配置的通信质量未达标，例如信号强度为-85dBm，低于预设阈值-80dBm，则采用最大似然估计算法更新功率分配参数。具体而言，算法分析实时反馈数据，如误码率0.013，结合环境响应参数，重新计算功率分配比例，将发射功率调整至20dBm，主要子载波功率占比提升至75％。调整后的功率分配方案生成信号编码指令，例如采用LDPC码，生成实时调整后的信号输出。采集实时调整后的信号输出反馈数据是验证通信稳定性的关键。反馈数据表明传输延迟为50ms，丢包率为0.3％，符合预设阈值55ms和0.4％。若反馈数据显示延迟增至70ms，丢包率升至0.6％，则通过迭代分析更新通信配置，生成优化的信号调制指令。
进一步地，采用更新后的通信配置，执行实时视频传输任务，判断数据丢包率是否满足救援指令传递要求，获得优化的信号处理方案，包括：
若数据丢包率超过预设丢包率阈值，则采集实时网络状态数据，分析网络拥塞特性，确定网络状态参数，根据网络状态参数，采用正交频分复用方法调整信号带宽分配，得到优化后的带宽配置；
通过优化后的带宽配置，结合实时网络状态数据，计算数据包重传比例，确定重传策略，若重传策略下的传输质量低于预设阈值，则采用最小二乘法优化重传参数，得到调整后的重传方案；
根据调整后的重传方案，生成对应的数据编码指令，获得实时调整后的数据输出，采集实时调整后的数据输出反馈，分析传输稳定性，获得优化的信号处理方案。
具体地，在飞行器通信领域，数据丢包率是衡量通信质量的重要指标。若丢包率超过预设阈值，需通过动态调整网络参数以优化传输性能。
例如，采集实时网络状态数据是优化通信的基础。飞行器在高海拔巡航时，网络状态数据包括带宽利用率70％、网络延迟40ms、信号干扰强度0.03mV/m。这些数据反映了网络环境的动态变化，为后续分析提供依据。分析网络拥塞特性时，可通过数据处理确定关键参数，如丢包率0.5％、抖动10ms。这些参数帮助识别网络瓶颈，为带宽分配提供依据。在一种可能的实现方式中，采用正交频分复用方法调整信号带宽分配。基于上述网络状态参数，算法将总带宽划分为多个子信道，例如总带宽10MHz划分为50个子信道，每个子信道带宽200kHz。针对高丢包率场景，优先为视频数据分配60％带宽，其余分配给控制信号，以降低视频传输中断风险。
结合实时网络状态数据计算数据包重传比例。假设反馈显示当前丢包率0.5％，算法根据网络延迟和抖动，初步确定重传比例为30％，即每100个数据包重传30个。重传策略中，优先重传关键视频帧数据，确保视频流畅性。
在一种可能的实现方式中，若传输质量未达标，例如视频帧率低于25fps，预设阈值为30fps，则采用最小二乘法优化重传参数。算法分析实时反馈数据，如丢包率0.6％、延迟45ms，调整重传比例至35％，并优先重传高优先级数据包。调整后的重传方案生成数据编码指令，例如使用FEC编码，添加冗余数据以提高抗干扰能力。例如，采集实时调整后的数据输出反馈，分析传输稳定性。反馈数据表明调整后延迟降至38ms，丢包率降至0.3％，接近预设阈值40ms和0.4％。若反馈显示延迟升至50ms，算法通过迭代分析更新配置，生成优化指令。
进一步地，根据优化的信号处理方案，重新配置通信链路，降低参数调整时延，确定稳定的数据传输通道，包括：
对通信链路丢包率采用快速傅里叶变换获取信号频谱分布，确定初始的信号处理参数，根据初始的信号处理参数，采用卡尔曼滤波方法优化通信链路配置，得到优化的链路参数；
通过优化的链路参数，分析实时网络传输时延，确定低时延的传输通道，若传输时延高于预设传输时延阈值，则通过最小均方误差算法调整数据包调度策略，得到优化的调度参数；
根据优化的调度参数，生成数据包传输优先级指令，获得实时调整后的传输序列，通过实时调整后的传输序列，采集网络拥塞状态数据，判断拥塞程度，确定稳定的传输配置，根据稳定的传输配置，生成最终的数据传输指令，获得实时稳定的数据输出。
具体地，通过优化后的链路参数，分析实时网络传输时延。采集数据显示带宽利用率80％、时延38ms、丢包率0.6％。若预设时延阈值为35ms，当前时延超标，系统需确定低时延传输通道。通过分析网络负载，优先选择带宽占用较低的通道，如调整至次要信道，降低时延至34ms。
在一种实施例中，若传输时延仍高于传输时延阈值，通过最小均方误差算法优化数据包调度策略。算法根据实时网络状态，如丢包率0.55％、时延40ms，调整调度参数，设置关键视频帧的传输优先级为高，普通数据包为中，优化后的调度参数减少冗余传输。
基于优化调度参数，系统生成数据包传输优先级指令。示例性地，指令规定每100个数据包中，50个为高优先级视频帧，30个为中优先级控制数据，20个为低优先级日志数据。这种优先级序列确保视频流连续性，优化传输效率。
例如，通过实时调整后的传输序列，采集网络拥塞状态数据。数据表明拥塞程度为中，带宽利用率升至85％，时延稳定在33ms。系统据此判断当前配置接近稳定，微调带宽分配比例至70％用于视频数据，保留30％用于控制信号。
进一步地，从稳定的数据传输通道中提取实时视频数据，获取压缩后的救援指令数据，包括：
从稳定的数据传输通道中获取实时视频数据，采用Huffman编码进行压缩，得到压缩后的指令数据包，通过压缩后的指令数据包，解析数据包内容，获取压缩后的救援指令数据；
根据压缩后的救援指令数据，采用优先级队列算法设置数据包优先级，得到优先级排序后的传输序列，若优先级排序后的传输序列中存在高优先级数据包，则通过带宽分配机制调整传输通道状态，获取优化后的通道配置；
根据优化后的通道配置，采用滑动窗口协议控制实时数据流，得到拥塞控制后的数据流，通过拥塞控制后的数据流，采集网络拥塞状态数据，判断拥塞程度，确定最终的传输指令，根据最终的传输指令，生成实时稳定的救援数据输出。
进一步地，通过压缩后的救援指令数据，执行指令传递任务，判断通信链路稳定性是否满足快速机动需求，包括：
通过预设的压缩算法对救援指令数据进行解压，得到解压后的指令数据集，若解压后的指令数据集完整性通过验证，则将指令数据集传输至目标节点，获取传输过程中的链路状态数据；
根据链路状态数据，采用预设的链路质量评估模型，判断链路稳定性参数，若链路稳定性参数满足预设的快速机动阈值，则通过实时传输协议推送指令数据集，得到传输完成时间；
根据传输完成时间，计算指令传递效率和机动响应速度，确定是否满足快速机动需求，若指令传递效率低于预设传输效率阈值，则采用动态路由优化算法调整传输路径，获取优化后的链路状态数据；
根据优化后的链路状态数据，重新执行链路质量评估模型，判断最终的通信链路稳定性。
具体地，在救援指令数据处理场景中，针对预设的压缩算法解压救援指令数据，可采用Huffman解码算法对压缩后的数据包进行解压。Huffman解码通过构建字符频率树，将压缩数据还原为原始指令数据集。示例性地，假设救援指令数据包含坐标信息“X:100，Y:200，动作:前进”，压缩后数据包大小为50字节，解压后还原为200字节的完整指令集。解压过程需验证数据完整性，可通过校验和机制比较解压前后数据一致性，确保指令无丢失或篡改。
在一种可能的实现方式中，解压后的指令数据集传输至目标节点时，可通过无线通信链路将数据发送至救援机器人。传输过程中，链路状态数据如信号强度、丢包率等被实时采集。例如，信号强度为-60dBm，丢包率为2％，表明链路质量较高。采集这些数据有助于后续链路评估。具体而言，链路质量评估模型可基于加权评分机制，结合信号强度、延迟和丢包率，计算链路稳定性参数。示例性地，假设模型设定信号强度权重为0.5，延迟权重为0.3，丢包率权重为0.2，计算得稳定性参数为85，超过预设快速机动阈值80，表明链路适合快速指令传输。这种评估方式确保指令传输的可靠性。
