Description

基于多模态监测的低空飞行器全景侦测方法及系统
技术领域
本申请涉及数据处理技术领域，尤其涉及一种基于多模态监测的低空飞行器全景侦测方法及系统。
背景技术
随着航空技术的不断发展以及低空飞行器（如小型民用无人机、低空飞行的通用航空飞机等）应用的日益广泛，对低空飞行器的有效侦测变得至关重要。在航空管理、安全监控等诸多领域，准确掌握低空飞行器的相关信息，包括其位置、状态以及与周围环境的关系等，是保障空域安全、实现有效管理的基础。
传统的飞行器侦测技术往往存在诸多局限性。例如，仅依靠单一模态的传感器进行侦测，若采用可见光图像传感器，虽然能够获取飞行器外观以及周围环境的彩色图像，提供丰富的颜色和纹理信息，但在一些特殊情况下，如低光照条件或者飞行器与背景颜色相近时，其侦测效果会大打折扣。而红外热成像探测器虽然能够捕捉物体的热辐射信息，对热源敏感，有助于在夜间或者复杂气象条件下发现飞行器，但它所提供的图像主要基于温度差异，缺乏颜色和细节纹理信息，单独使用时难以全面准确地理解场景信息。
在对低空飞行器的侦测分析过程中，如何准确理解侦测图像中的场景信息也是一个巨大挑战。传统的分析方法往往缺乏对图像中多个对象之间关系的深入挖掘。

步骤S100：在多模态融合图像中确定若干个关键对象，多模态融合图像是通过低空飞行器的红外热成像探测器和高清彩色可见光图像传感器各自探测得到的图像融合得到的。
本申请实施例中，多模态融合图像是通过低空飞行器的红外热成像探测器和高清彩色可见光图像传感器各自探测得到的图像融合而成的。例如，红外热成像探测器能够捕捉到物体的热辐射信息，形成热成像图像。在这个图像中，不同温度的物体呈现出不同的灰度或色彩（例如高温物体可能显示为白色或亮色调，低温物体显示为黑色或暗色调）。而高清彩色可见光图像传感器则能够捕捉到物体的可见光反射信息，形成日常所熟悉的彩色图像，包含丰富的颜色和纹理信息。将这两种图像进行融合，就得到了多模态融合图像。这种融合后的图像综合了两种不同模态图像的优势，能够提供更全面、更丰富的信息。
计算机系统在处理这个多模态融合图像时，需要确定其中的关键对象。关键对象是指在多模态融合图像中具有重要意义、对后续的侦测和分析工作有着关键影响的对象。确定关键对象的方法可以基于多种技术手段。
一种技术手段是基于图像的特征分析。计算机系统可以对多模态融合图像进行特征提取，例如提取图像中的边缘特征、纹理特征、形状特征等。以边缘特征为例，边缘通常是物体与背景或者不同物体之间的分界线，通过检测边缘可以确定物体的大致轮廓。假设在多模态融合图像中有一个疑似建筑物的物体，计算机系统通过边缘检测算法（如Canny边缘检测算法），可以确定这个建筑物的边缘轮廓，这就是一种特征提取的过程。
对于纹理特征，不同的物体往往具有不同的纹理。比如植被可能具有不规则的、细腻的纹理，而人造的金属结构可能具有相对规则的纹理。计算机系统可以通过纹理分析算法（如灰度共生矩阵法）来分析图像中的纹理特征，从而区分不同的物体，进而确定可能的关键对象。形状特征也是确定关键对象的重要依据。如果图像中有一些具有特定形状的物体，例如圆形的储油罐或者矩形的建筑物，计算机系统可以利用形状识别算法（如霍夫变换用于检测圆形或矩形等几何形状）来识别这些具有特定形状的物体，将其作为关键对象的候选。
除了基于图像特征分析之外，计算机系统还可以结合先验知识来确定关键对象。例如，在低空飞行器的侦测场景下，某些类型的物体是比较常见且重要的，如其他飞行器、地面上的重要设施（如机场跑道、通讯基站等）、特殊地形（如山脉、河流等）。计算机系统可以预先存储这些常见且重要物体的相关信息（如形状、纹理、热成像特征等），然后在多模态融合图像中搜索与之匹配的物体，将其确定为关键对象。
计算机系统确定关键对象的过程是一个综合考量多种因素的过程，需要充分利用多模态融合图像中丰富的信息以及预先获取的知识，从而准确地找出那些对全景侦测有着关键意义的对象，为后续的侦测工作奠定基础。这一步骤在整个低空飞行器全景侦测方法中起着至关重要的作用，它直接影响到后续步骤中与图像知识库的匹配、侦测对象二元组的构建以及最终的侦查解析结果的准确性。
步骤S200：获取用以指示若干个侦测对象之间的对象关系的图像知识库，在若干个侦测对象中获取若干个关键对象分别匹配的侦测对象，作为匹配侦测对象。
本申请实施例中，图像知识库（例如是一个视觉知识图谱）是一种存储了各种侦测对象之间关系的知识集合。它可以包含不同侦测对象的属性信息、对象之间的空间关系（如相邻、包含等关系）、语义关系（如类别关系，一个侦测对象属于某个特定的类别，而不同类别之间可能存在上下级或者关联关系）等。例如在低空飞行器侦测场景下，图像知识库中可能存储着飞行器与机场跑道之间的关系（空间上，飞行器在跑道附近或者在跑道上起降；语义上，飞行器依赖跑道进行起降操作），还可能存储着通讯基站与周围环境（如建筑物、植被等）的关系（空间上，基站可能位于建筑物顶部，周围有植被环绕；语义上，基站为一定范围内的区域提供通讯服务，其覆盖范围与周围的地形、建筑物分布等有关）等关系信息。
计算机系统在执行步骤S200时，从这个图像知识库中获取相关信息。对于侦测对象的概念，在这个场景下，侦测对象是指在低空飞行器侦测范围内可能出现的各种实体，包括飞行器、地面的各种设施（如前所述的机场跑道、通讯基站等）、自然地理元素（如山脉、河流等）以及其他可能影响飞行或者被飞行影响的物体。
在获取用以指示若干个侦测对象之间的对象关系的图像知识库后，计算机系统要在若干个侦测对象中为之前确定的关键对象找到匹配的侦测对象。这里的匹配过程是基于多种因素和技术手段的。
一种技术手段是基于特征相似性的匹配。计算机系统可以为每个侦测对象和关键对象提取特征向量。例如，对于一个关键对象为疑似飞行器的物体，计算机系统可以提取其在多模态融合图像中的形状特征向量（如长、宽、高比例等反映形状的参数组成的向量）、热成像特征向量（如不同部位的温度分布特征转化为向量形式）以及可见光图像中的颜色和纹理特征向量（如颜色的直方图特征、纹理的粗糙度等特征转化为向量）。对于侦测对象，同样进行这样的特征向量提取。然后，计算机系统通过计算特征向量之间的距离（如欧几里得距离公式）来衡量它们之间的相似性。距离越小，表示两者越相似，越有可能是匹配的侦测对象。
另一种技术手段是基于语义关系的匹配。如果图像知识库中存储了语义信息，计算机系统可以利用这些信息进行匹配。例如，如果关键对象被初步判断为是一种交通工具类型的物体，计算机系统会在侦测对象中寻找同样被标记为交通工具类型并且具有相似功能或者使用场景的物体作为匹配侦测对象。例如，飞行器与其他飞行类的交通工具（如直升机、热气球等）在语义上有一定的关联，如果关键对象是飞行器，那么在侦测对象中的直升机就可能因为这种语义关系被认为是一个匹配侦测对象的候选。
此外，还可以基于空间关系进行匹配。假设在多模态融合图像中，关键对象是一个靠近山脉的疑似飞行器物体。计算机系统在图像知识库中查找与山脉有空间关系（如靠近山脉的物体）的侦测对象，并且这些侦测对象具有与飞行器相似的特征（如飞行轨迹、速度范围等，如果这些信息可从图像中分析或者知识库中获取），那么这些侦测对象就可能被确定为与关键对象匹配的对象。
步骤S300：在若干个匹配侦测对象中，逐一探索获取包括作为源侦测对象的匹配侦测对象以及作为终止侦测对象的匹配侦测对象的侦测对象二元组；源侦测对象与终止侦测对象不同。
本申请实施例中，源侦测对象是作为起始点或者关系发起者的匹配侦测对象，而终止侦测对象则是与源侦测对象存在某种潜在关系并且在关系链中处于终点位置的匹配侦测对象。例如，在低空飞行器的侦测场景中，如果一个匹配侦测对象是飞行器，另一个匹配侦测对象是机场跑道，那么飞行器可以被看作源侦测对象，机场跑道就可以被看作终止侦测对象，因为飞行器与跑道之间存在着降落、起飞等关联关系，从飞行器到跑道有一个关系的指向。
计算机系统在执行这一步骤时，一种可行的技术手段是基于关系矩阵的探索。假设计算机系统为每个匹配侦测对象建立了一个关系矩阵，矩阵的行和列分别代表不同的匹配侦测对象。在这个矩阵中，每个元素表示两个匹配侦测对象之间可能存在的关系权重。例如，如果匹配侦测对象A（如飞行器）和匹配侦测对象B（如机场跑道）之间存在较高的关联可能性（如基于前面步骤确定的特征相似性、语义关系或者空间关系等），那么在关系矩阵中对应的元素值就会较高。
计算机系统通过遍历这个关系矩阵来寻找可能的侦测对象二元组。从关系矩阵的第一行第一列开始，计算机系统将每个元素对应的两个匹配侦测对象作为一个可能的侦测对象二元组进行考虑。如果元素值满足一定的阈值条件（这个阈值可以根据实际情况进行设定，例如根据历史数据或者经验确定一个表示较强关系的数值），则将这两个匹配侦测对象确定为一个侦测对象二元组，其中一个作为源侦测对象，另一个作为终止侦测对象。例如，在关系矩阵中，如果飞行器与机场跑道对应的元素值高于设定的阈值，那么就可以将（飞行器，机场跑道）确定为一个侦测对象二元组。
另一种技术手段是基于图搜索算法。可以将所有的匹配侦测对象看作是图中的节点，而它们之间的关系看作是图中的边。计算机系统从一个匹配侦测对象（源节点，即源侦测对象）开始，使用深度优先搜索或者广度优先搜索算法遍历图中的其他节点（匹配侦测对象）。当搜索到一个与源节点不同且满足一定关系条件的节点（终止节点，即终止侦测对象）时，就可以确定一个侦测对象二元组。例如，以飞行器为源节点，在图中进行深度优先搜索，当搜索到机场跑道这个节点时，就可以确定（飞行器，机场跑道）为一个侦测对象二元组。
在实际的低空飞行器侦测场景中，还有很多情况需要考虑。例如，当有多个飞行器同时存在于多模态融合图像中时，计算机系统需要对每个飞行器分别作为源侦测对象进行探索。假设存在飞行器A、飞行器B和机场跑道C、通讯基站D等匹配侦测对象。对于飞行器A，计算机系统在探索过程中可能会发现与机场跑道C存在降落关系，从而确定（飞行器A，机场跑道C）为一个侦测对象二元组；同时，飞行器A可能与通讯基站D存在某种信号交互或者空间位置关系（如飞行器A在通讯基站D的信号覆盖范围内飞行），那么也可以确定（飞行器A，通讯基站D）为一个侦测对象二元组。对于飞行器B，同样需要进行这样的探索过程，可能会确定与机场跑道C或者其他匹配侦测对象之间的侦测对象二元组。
此外，计算机系统在确定侦测对象二元组时，还需要考虑到数据的准确性和完整性。由于多模态融合图像可能存在噪声、部分信息缺失或者不准确的情况，在基于关系矩阵或者图搜索算法时，可能会出现一些误判。例如，由于图像噪声的影响，可能会导致某个匹配侦测对象的特征被错误识别，从而在关系矩阵中的关系权重被错误计算，或者在图搜索中被错误地连接到其他节点。为了避免这种情况，计算机系统可以采用数据验证和纠错机制。例如，对于基于关系矩阵确定的侦测对象二元组，可以再次根据多模态融合图像中的原始信息进行验证，检查两个匹配侦测对象之间是否真的存在合理的关系。如果发现存在疑问，可以重新评估关系矩阵中的元素值或者重新进行图搜索过程。
步骤S400：对图像知识库进行游走，得到用以建立源侦测对象和终止侦测对象联系的空间关系，生成包括源侦测对象、空间关系和终止侦测对象的关联链路。
本申请实施例中，计算机系统进行游走操作时，一种可能的技术手段是基于规则的搜索。例如，知识库中的关系可能被标记上不同的类型标签，如空间关系标签（包括相邻、包含、远离等）、功能关系标签（如飞行器依赖跑道起降、通讯基站为一定区域提供通讯服务等）等。计算机系统从源侦测对象开始，按照预定义的规则，查找与源侦测对象存在直接关联关系的对象以及对应的关系类型。

然而，在这个过程中，存在着诸多复杂的情况和挑战。首先，由于多模态融合图像的特性，不同模态之间的信息融合可能会带来一些不确定性。

其次，图像中的噪声和干扰因素会影响匹配和同现频率的计算。噪声可能导致特征的误识别，使得原本不相似的特征被错误地判断为相似，或者相反。例如，由于图像传感器的噪声，飞行器机翼的边缘在图像中可能出现一些不规则的波动，这可能会导致在与跑道边缘特征匹配时产生错误的结果。为了应对这种情况，计算机系统可以在进行匹配之前对图像进行预处理，如采用滤波算法来降低噪声的影响。对于图像中的干扰因素，如阴影（在可见光图像中）或者热成像中的热干扰（如附近热源的干扰），也需要进行特殊处理。例如，对于阴影部分的特征，可以通过分析周围正常光照区域的特征以及物体的几何形状来进行修正；对于热干扰，可以根据热源的分布规律和热传导原理来排除其对目标特征的影响。
此外，不同的匹配算法在准确性和效率方面存在差异，并且对于不同类型的特征可能有不同的适用性。例如，SIFT算法在处理具有明显纹理和形状变化的特征时效果较好，但对于一些较为平滑、纹理不明显的特征可能表现不佳。计算机系统可能需要根据具体的特征类型和要求选择合适的匹配算法，或者采用多种算法相结合的方式来提高匹配的准确性。例如，对于飞行器的发动机部分这种纹理相对复杂且具有温度特征的局部特征，可以同时使用SIFT算法和基于热成像特征的匹配算法（如根据温度梯度方向等特征进行匹配），然后综合两者的结果来确定与跑道对象局部特征的匹配情况和对象同现频率。
