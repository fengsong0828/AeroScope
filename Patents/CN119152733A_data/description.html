Description

一种低空空域的飞行器管制方法、系统及终端
技术领域
本发明涉及空域管理技术领域，尤其涉及一种低空空域的飞行器管制方法、系统、终端及计算机可读存储介质。
背景技术
随着民用无人驾驶航空器的广泛应用和普及，以及产业技术的迅速发展，当前低空安全风险日益累积，低空安全问题逐渐凸显，成为未来低空产业发展的重要风险点。为了保障低空空域的安全稳定，提高低空经济的持续发展，规范化的空域管制是必不可少的。
然而，目前成熟的低空空域管制系统由低空监视系统和飞行计划和报备制度组成，在管控力度上只能明确飞机的坐标和大概航线，实时性弱，规划性不强，对于维度的信息描述少，导致无法形成一个规范的空域管制规则及系统。
因此，现有技术还有待于改进和发展。
发明内容
本发明的主要目的在于提供一种低空空域的飞行器管制方法、系统、终端及计算机可读存储介质，旨在解决现有技术中对于低空空域的飞行器管控力度不够，规划性不强，低空空域的安全稳定无法得到保障的问题。
为实现上述目的，本发明提供一种低空空域的飞行器管制方法，所述一种低空空域的飞行器管制方法包括如下步骤：
当检测到目标飞行器进入低空空域时，获取所述目标飞行器的飞行信息、测量信息和探测信息，对所述飞行信息、所述测量信息和所述探测信息进行预处理，得到预处理后的目标数据；
基于预设的融合算法对所述目标数据进行分析和融合，生成定位信息，根据所述定位信息得到所述目标飞行器的网格区域，获取所述目标飞行器的类型；
若所述类型的目标飞行器符合所述网格区域的飞行标准，则获取所述网格区域的历史飞行数据，将所述历史飞行数据输入到预设的低空飞行器轨迹预测模型中进行预测，得到所述目标飞行器的未来飞行轨迹；
获取所述网格区域内其他飞行器的飞行轨迹甬道，若所述未来飞行轨迹与其他飞行器的飞行轨迹甬道不存在冲突，则根据所述未来飞行轨迹和所述网格区域生成目标飞行轨迹甬道，控制所述目标飞行器按照所述目标飞行轨迹甬道飞行，并将所述目标飞行轨迹甬道设置为锁定状态。

本发明较佳实施例所述的低空空域的飞行器管制方法，如图1和图2所示，所述低空空域的飞行器管制方法包括以下步骤：
步骤S10、当检测到目标飞行器进入低空空域时，获取所述目标飞行器的飞行信息、测量信息和探测信息，对所述飞行信息、所述测量信息和所述探测信息进行预处理，得到预处理后的目标数据。
当低空空域内检测到存在飞行器时，将所述飞行器作为目标飞行器开始获取所述目标飞行器的飞行信息、测量信息和探测信息。
具体而言，定期获取所述目标飞行器的传感器和导航系统发送来的飞行信息，所述飞行信息包括：飞行器位置信息、飞行器速度信息、飞行器方向信息。
可以理解的是，飞机上的传感器和导航系统可以提供自身的位置、速度、方向等数据。这些数据通常以定期回报的形式发送给地面控制站系统。
定期获取所述目标飞行器接收到的信标信号，通过信号传播模型计算出所述目标飞行器与信标之间的相对位置关系，根据所述相对位置关系，通过几何计算或优化算法确定出测量信息。
在本实施例中，在飞行区域内布置的信标（导航信号标记）可以发送无线信号，飞机接收到这些信号后可以测量出与信标的距离或角度信息，并作为回报数据发送给地面控制站。具体而言，利用飞机接收到的信标信号，结合信号传播模型（如到达时间差、到达角度等），计算出飞机与信标之间的相对位置关系。然后，结合信标的已知位置，通过几何计算或优化算法确定飞机的位置。
定期获取基站通过雷达反射和无线电信号传播技术进行主动探测得到的目标飞行器的探测信息。在飞行区域内布置的信标可以发送无线信号，飞机接收到这些信号后可以测量出与信标的距离或角度信息，并作为回报数据发送给地面控制站。
可以理解的是，本实施例中，所述基站可以为5G-A通感基站（5G-Advanced，是基于5G网络在功能上和覆盖上的演进和增强，是支撑互联网产业3D化，云化，万物互联智能化，通信感知一体化，智能制造柔性化等产业数字化升级的关键信息化技术），5G-A通感基站可以主动探测飞行器的位置，并将探测结果发送给地面控制站。具体而言，通感基站通过主动探测获得飞行器的位置信息。这通常是通过雷达反射和无线电信号传播等物理原理实现的。将通感基站探测到的位置信息作为另一个独立的位置估计。
可见，本发明采用三重定位技术，通过结合三种不同的定位源，进行三重定位，能够提供更准确的位置信息，每种定位源都有其独特的优势，通过融合它们的数据，可以相互补充，减少单一方法带来的误差，三重定位技术能够提供高精度的飞行器位置信息。通过结合飞机回报信息、信标回报信息和通感基站探测，可以实现对飞行器的精确跟踪和定位。这有助于减少定位误差，提高空域管制的准确性。
进一步地，所述对所述飞行信息、所述测量信息和所述探测信息进行预处理，得到预处理后的目标数据，具体包括：
分别对所述飞行信息、所述测量信息和所述探测信息进行清理，去除所述飞行信息、所述测量信息和所述探测信息中的缺失值和异常值，得到有效信息；对所述有效信息进行归一化处理和标准化处理，得到预处理后的目标数据。
在本实施例中，为了保证获取到的所述飞行信息、所述测量信息和所述探测信息的质量的后续处理的精度，首先对所述飞行信息、所述测量信息和所述探测信息进行清理，清理主要包括去除所述飞行信息、所述测量信息和所述探测信息中的缺失值和异常值，以从中提取处有效信息；进一步地，对所述有效信息进行格式化处理，包括对所述有效信息进行归一化处理和标准化处理，得到预处理后的目标数据，即预处理后的飞行信息、预处理后的测量信息和预处理后的探测信息，以规范数据的形式，方便后续处理。
步骤S20、基于预设的融合算法对所述目标数据进行分析和融合，生成定位信息，根据所述定位信息得到所述目标飞行器的网格区域，获取所述目标飞行器的类型。
具体地，获取所述目标数据中预处理后飞行信息、预处理后测量信息和预处理后探测信息各自的精度和可靠性，根据所述精度和可靠性进行相关性分析，得到预处理后飞行信息、预处理后测量信息和预处理后探测信息之间的相关性。
在本实施例中，获取所述目标数据，即预处理后飞行信息、预处理后测量信息和预处理后探测信息，对三项预处理后数据精度和可靠性进行分析，得到中三项预处理后数据各自的精度和可靠性，所述精度和所述可靠性作为具备相关性的变量元素，可以进行相关性分析来衡量这个两个变量因素的相关密切程度，根据所述精度和可靠性进行相关性分析，得到三项预处理后数据之间的相关性。
进一步地，根据所述相关性确定预处理后的飞行信息、预处理后的测量信息和预处理后的探测信息各自的权重，通过采用加权平均的方式对预处理后的飞行信息、预处理后的测量信息和预处理后的探测信息进行融合，生成定位信息。
可以理解的是，根据三项数据的相关性分析结果（即相关系数），可以为相关性系数较大的赋予较大的权重，为关性系数较小的赋予较小的权重，进而确定出预处理后的飞行信息、预处理后的测量信息和预处理后的探测信息各自的权重，然后采用加权平均（统计学术语，在加权平均中，每个数据点都有一个对应的权重值，该权重值决定了数据点在计算平均值时的重要程度。通过相乘各个数据点的值与权重值，并将结果求和后再除以所有权重值的总和，就可以得到加权平均值）的方式对三项预处理后数据进行融合，生成定位信息。
需要说明的是，除了通过加权平均的方式对数据进行融合外，还可以采用卡尔曼滤波、最小二乘等方法进行三项预处理后数据的融合，且融合时还需要考虑不同定位方法的精度、可靠性以及它们之间的相关性。
进一步地，所述基于预设的融合算法对所述目标数据进行分析和融合，生成定位信息，之后还包括：
判断所述定位信息是否在可允许的误差范围之内，若如果定位信息匹配在误差范围内，则认为定位是准确的，则将定位数据进行后续的可视化处理，得到所述定位信息的空域网格区域；若定位信息匹配超出误差范围，则认为定位不精确，可能情况为飞机伪报信息，此时应发生警报，并通知相关人员处理。
更进一步地，所述根据所述定位信息得到所述目标飞行器的网格区域，之前还包括：
根据飞行器类型，设置三种不同空域范围的默认网格区域，所述飞行器类型包括小型无人机、中型无人机、大型无人机、直升机和客机，所述默认网格区域包括第一默认网格区域、第二默认网格区域和第三默认网格区域；
其中，所述第一默认网格区域为单层分布，所述第一默认网格区域的飞行标准为仅允许类型为小型无人机的飞行器飞行；所述第二默认网格区域为单层分布，所述第二默认网格区域的飞行标准为仅允许类型为中型无人机、大型无人机或者直升机的飞行器飞行；所述第三默认网格区域为三层分布，所述第三默认网格区域的飞行标准为仅允许类型为客机的飞行器飞行。
具体而言，本方法主要基于空域网格化划分处理定位信息和可视化操作。根据不同高度层中飞行器规模和种类的不同可以做一下不同的划分：
第一默认网格区域（0-120m）：在此高度绝大部分飞行单位由小型无人机组成，垂直分布单一，因此设置为单层网格分布，单元网格较小，尺寸为100m（长）*100m（宽）*120m（高）；
第二默认网格区域（120-300m）：在此高度绝大部分飞行单位由中大型无人机和直升机组成，垂直分布单一，因此设置为单层网格分布，单元网格比W类大，尺寸为200m（长）*200m（宽）*180m（高）；
第三默认网格区域（300m-1200m）：在此高度的主要飞行单位为中大型客机，航向呈现高度差异，因此高度呈现三层网格分布，由于客机体积大，单元网格尺寸要设置为更大的300m（长）*300m（宽）*300m（高）。进一步地，可以根据实际情况调整不同高度层中的网格大小和形状，以适应不同的飞行需求。
可见，本发明通过网格化划分空域可以更好地规划和利用空域资源，明显提高空域的利用率；网格化空域划分也使得空域管理变得更加简单和清晰，降低了管理难度和复杂度；并且使得各个网格之间的信息共享变得更加容易，有利于实现空域管理的规范化、协同化和一体化；通过网格化空域划分，可以更好地监控和控制空域内的飞行活动，减少飞行冲突和事故性。
通过将空域划分为若干个网格，每个网格内的飞行器管理和控制可以更加明确和有序。这有助于减少飞行冲突，提高空域使用效率。同时，空域管制系统也需要制定相应的规则，以确保所有飞行器在网格化空域中能够有序、安全地运行。
更进一步地，根据所述定位信息得到所述目标飞行器的网格区域，获取所述目标飞行器的类型。
在本实施例中，通过所述定位信息能够得出所述目标飞行器属于上面三个默认网格区域中的哪个范围，而且确定出所述目标飞行器所处的网格区域，然后获取所述目标飞行器的类型，所述类型包括小型无人机、中型无人机、大型无人机、直升机和客机；得到所述目标飞行器的类型后，将所述类型与其对应的网格区域的飞行标准进行对比，例如所述目标飞行器的类型为中型无人机，但是检测到所述目标飞行器所处的网格区域为第一默认网格区域，第一默认网格区域并不允许中型无人机飞行，此时所述类型的飞行器就不符合所述网格区域的飞行标准。
步骤S30、若所述类型的目标飞行器符合所述网格区域的飞行标准，则获取所述网格区域的历史飞行数据，将所述历史飞行数据输入到预设的低空飞行器轨迹预测模型中进行预测，得到所述目标飞行器的未来飞行轨迹。
具体地，若所述类型的飞行器的类型符合所述网格区域的飞行标准，则获取所述网格区域的历史飞行数据，其中，所述历史飞行数据包括历史飞行器的种类、历史飞行器的飞行轨迹数据和历史飞行器的飞行速度。
将所述历史飞行数据输入到预设的低空飞行器轨迹预测模型中进行预测，所述预设的低空飞行器轨迹预测模型由BiGRU（Bidirectional Recurrent Neural Network，双向循环神经网络）模型和GNN（Graph Neural Network，图神经网络）模型构成，基于所述低空飞行器轨迹预测模型，根据现有的历史飞行数据对目标飞行器的未来飞行轨迹进行预测，实现实时运算预测，更好规避飞行器因轨迹偏移等原因带来的安全问题。
需要说明的是，这里的低空飞行器轨迹预测模型可以根据实际情况进行调整，例如所述低空飞行器轨迹预测模型也可以是LSTM（Long Short-Term Memory，长短期记忆网络）模型和CNN（Convolutional Neural Networks，卷积神经网络）模型构成，这里不做限制。
进一步地，若所述类型的目标飞行器不符合所述网格区域的飞行标准，则说明所述目标飞行器的定位处于不可飞或禁飞网格区域，此时，会向所述目标飞行器发送警告，指示飞行器修改航线并通知相关人员处理；若定位处于可飞网格区域则将历史飞行数据输入飞行轨迹预测系统。
步骤S40、获取所述网格区域内其他飞行器的飞行轨迹甬道，若所述未来飞行轨迹与其他飞行器的飞行轨迹甬道不存在冲突，则根据所述未来飞行轨迹和所述网格区域生成目标飞行轨迹甬道，控制所述目标飞行器按照所述目标飞行轨迹甬道飞行，并将所述目标飞行轨迹甬道设置为锁定状态。
具体而言，当低空空域的飞行器管制系统接收到历史飞行数据后，基于所述低空飞行器轨迹预测模型进行所述目标飞行器的未来飞行轨迹的预测，并获取述网格区域内其他飞行器的飞行轨迹甬道。
若所述未来飞行轨迹与其他飞行器轨迹甬道冲突，则警告所述目标飞行器修改航线，避免事故，如果无冲突，则在可视化网格系统中形成飞行轨迹甬道，并将甬道网格锁定，设置为其他飞行器不可飞行状态；
若所述未来飞行轨迹与其他飞行器的飞行轨迹甬道不存在冲突，则根据所述未来飞行轨迹和所述网格区域生成目标飞行轨迹甬道（在可视化网格系统中形成飞行轨迹甬道），控制所述目标飞行器按照所述目标飞行轨迹甬道飞行，并将所述目标飞行轨迹甬道设置为锁定状态，在所述目标飞行轨迹甬道处于锁定状态时，表示其他飞行器不可在所述目标飞行轨迹甬道内飞行，以确保所述目标飞行器的安全运行。
本发明的有益效果：
（1）提高空域利用率：通过网格化空域划分，可以更加精细地管理空域资源，减少无效空域的使用，从而提高空域的整体利用率。同时，结合三重定位技术，可以准确掌握飞行器的位置信息，进一步优化飞行路径，减少不必要的航程，进一步提高空域的使用效率。
（2）增强飞行安全性：三重定位技术可以提供高精度的飞行器位置信息，有助于及时发现潜在的飞行冲突，并通过飞行轨迹预测技术提前采取措施进行干预和避免。此外，空域管制系统还可以对飞行器进行实时监控，确保其在规定的空域内飞行，避免误入危险区域或与其他飞行器发生碰撞。
（3）提升管制效率：系统可以自动化地处理大量的数据和信息，实现实时的飞行监控和管制。这可以大大减少人工干预的需要，提高管制效率。同时，基于人工智能技术的应用，系统还可以实现自动化的冲突探测和解决方案生成，进一步减轻管制员的工作负担。
（4）优化资源配置：通过网格化空域划分和飞行器位置信息的精确掌握，系统可以更加合理地配置地面保障设施和其他资源。例如，可以根据飞行器的实际位置和飞行轨迹，提前调度地面车辆、设备等资源，确保其在需要时能够及时到达指定位置，为飞行器提供必要的支持和保障。
（5）支持复杂飞行任务：系统具备灵活性和实时性特点，可以支持各种复杂的飞行任务。无论是常规的航班飞行还是特殊的军事训练、救援任务等，系统都可以根据实际需求进行空域管理和飞行管制，确保任务的顺利进行。
（6）降低人为错误风险：通过自动化和智能化的技术手段，系统可以减少人为干预的需要，从而降低人为错误的风险。同时，系统还可以对飞行器的状态进行实时监控和预警，及时发现潜在的问题并进行处理，确保飞行器的安全运行。
