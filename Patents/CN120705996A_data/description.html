Description

低空飞行器运行质量评估软件
技术领域
本发明涉及航空航天技术领域，具体为低空飞行器运行质量评估软件。
背景技术
随着低空飞行产业在近年来的迅猛推进，确保飞行器运行质量的安全性与可靠性已成为亟待解决的关键问题。然而，当前低空飞行领域在运行质量监控方面仍面临诸多挑战，尤其是缺乏统一、高效的监控标准、技术手段及智能分析工具。相较于成熟规范的运输航空体系，低空飞行器因机型种类繁多、飞行环境复杂多变，导致其品质评估的难度显著增加。尽管市场上已出现一些针对特定机型的飞行品质监控系统，但这些系统普遍存在通用性不足、分析精度欠缺以及数据处理能力受限等短板，难以满足行业快速发展的实际需求。鉴于此，业界迫切需要开发一款集智能化、高适应性于一体的运行质量评估软件。
发明内容
本发明的目的在于提供低空飞行器运行质量评估软件，该软件应具备跨机型、跨飞行程序的兼容能力，能够自动生成符合专业标准的飞行品质报告，并实时提供运行风险预警，为低空飞行器的安全运行提供全方位的技术支撑。
为实现上述目的，本发明提供如下技术方案：低空飞行器运行质量评估软件，包括分院基地客户端、服务端、学院中心客户端、领导驾驶舱、飞行员端和机务端，所述分院基地客户端包括首页展示模块、数据上传模块、文件初筛模块、超限分析模块、数据关联模块、外部数据源模块、飞行数据筛选器、地图航迹还原模块、视景还原模块、仪表还原模块、飞行阶段模块、参数曲线刨面模块、安全报告模块、监控引擎研发模块、设备管理模块、用户管理模块和意见模块。

低空飞行器运行质量评估软件，包括分院基地客户端、服务端、学院中心客户端、领导驾驶舱、飞行员端和机务端；分院基地客户端包括首页展示模块、数据上传模块、文件初筛模块、超限分析模块、数据关联模块、外部数据源模块、飞行数据筛选器、地图航迹还原模块、视景还原模块、仪表还原模块、飞行阶段模块、参数曲线刨面模块、安全报告模块、监控引擎研发模块、设备管理模块、用户管理模块和意见模块。
分院基地客户端作为数据采集与初级分析的核心节点，其模块分工如下：首页展示模块动态呈现实时飞行数据统计、超限事件预警及任务进度，并集成可视化仪表盘支持快速状态感知；数据上传模块通过API接口或批量导入方式接收飞行器原始数据（如ADS-B、机载传感器日志、飞控记录仪等），兼容JSON、CSV、二进制流等多格式解析，同时支持断点续传与数据完整性校验；文件初筛模块基于预设规则（如完整性校验、异常时间戳过滤、数据段连续性检测）自动剔除无效数据，并通过数据质量评分机制标记可疑片段供人工复核；超限分析模块调用动态阈值库（如高度偏离±50米、速度超限10%Vne、过载超1.5G等机型差异化参数）实时触发多级告警，结合时间戳与空间坐标生成初步事件标签；数据关联模块通过时空对齐算法将飞行数据与外部数据源模块接入的实时气象（如风切变预警）、空域动态（如临时限制区坐标）进行融合，构建多维分析上下文；地图航迹还原模块基于GIS平台将经纬度数据映射为三维航迹，叠加空域限制图层、地形高程模型实现冲突检测，并生成航迹偏离度热力图；视景还原模块利用飞行器三维模型库与场景渲染引擎，重构带环境光影效果的飞行过程回放；仪表还原模块通过解析总线数据动态复现座舱仪表组态，支持操作序列与仪表响应的帧级同步验证；飞行阶段模块采用隐马尔可夫模型（HMM）自动识别起飞、爬升、巡航、进近等阶段，并关联各阶段标准操作程序（SOP）进行合规性比对；参数曲线剖面模块提供多参数时空矩阵分析，支持航向角-坡度-油门曲线耦合分析及剖面异常点钻取；安全报告模块整合初级分析结果，自动生成包含关键事件时间线、空间分布图谱的预处理报告；监控引擎研发模块开放规则配置界面，允许用户通过拖拽式逻辑编辑器自定义飞行阶段划分规则、超限判定条件等分析策略，并经由服务端沙箱环境验证后部署至生产环境；设备管理模块对接机载设备编码库，实现传感器校准记录、设备健康状态的履历化管理；用户管理模块采用多因子认证体系，结合飞行任务周期动态调整数据访问权限；意见模块内置结构化反馈模板，支持问题标注、截图批注与优先级标记，并通过服务端消息队列实现闭环跟踪，各模块通过高吞吐量数据总线串联，原始数据经上传、初筛、超限分析后，关联气象/空域数据增强上下文语义，再经航迹还原、视景重建等可视化处理，最终形成带多维标签的数据包推送至服务端进行深度建模分析，同时本地留存加密副本以供快速溯源。
服务端包括统一通用数据格式模块、数据格式映射模块、数据清洗模块、数据分析模块、数据校准模块、数据时间统一模块、数据空间统一模块、数据传输模块、密文传输模块、加密授权模块和驾驶舱定制模块。
服务端作为分布式计算中枢，实现全链路数据处理：统一通用数据格式模块通过协议解析引擎（支持MAVLink、ARINC429、ASTM-F3411等20+航空数据协议）将异构数据转换为带时间基准的标准化JSON时序数据流，并采用IEEE-1588精确时钟协议对齐多源时序；数据格式映射模块内置航空数据字典库，通过语义解析技术自动生成字段映射关系矩阵，支持自定义航电设备数据模板的注册与发布；数据清洗模块结合滑动窗口算法（窗口大小自适应调整）修复数据断点，运用小波变换-卡尔曼滤波联合降噪模型消除传感器噪声，对清洗后数据打上置信度标签并存入时序数据库（InfluxDB集群）；数据分析模块集成专家知识库（含3000+条飞行品质判据规则）与机器学习模型（如基于LSTM的异常模式识别），执行操纵稳定性分析（如杆力-姿态响应滞后分析）、任务完成度评分（航线吻合度＞98%、着陆精度＜3米等量化指标），并生成带置信区间的评估结论；数据校准模块采用NTP/PTP混合授时机制消除设备间时钟偏差，通过PostGIS空间引擎将多源定位数据统一至WGS84坐标系，实现厘米级时空对齐；数据传输模块基于MQTT+gRPC混合协议构建分级传输通道，针对实时监控数据（延迟＜200ms）与批量报告数据（带宽优化）实施差异化调度；密文传输模块采用国密SM4算法构建端到端加密隧道，结合量子密钥分发技术实现抗量子攻击的数据保护；加密授权模块通过RBAC+ABAC混合模型实施动态权限控制，支持生物特征（虹膜/指纹）与硬件密钥（USB-Token）多因子认证，并建立每小时动态令牌轮换机制；驾驶舱定制模块内置PowerBI级可视化引擎，支持通过拖拽方式构建跨层级KPI看板（如区域风险热力图、千架次事故率趋势预测桑基图），并基于LDAP目录服务实现多级领导视图的权限隔离；处理后的数据经时空一致性验证（Checksum校验+数字签名）后，通过Kafka消息队列分发给各终端，同时向区块链存证平台写入哈希值以确保审计溯源性，最终形成覆盖数据全生命周期的闭环处理体系，确保从原始数据接入到决策支持输出的全流程安全可控、时空精准对齐。
学院中心客户端包括首页展示模块、无线传输模块、图标定制模块、监控引擎管理模块、报告引擎管理模块、基础数据管理模块、意见反馈管理模块、数据上传模块、文件初筛模块、超限分析模块、数据关联模块、外部数据源模块、飞行数据筛选器、地图航迹还原模块、视景还原模块、仪表还原模块、飞行阶段模块、参数曲线刨面模块、安全报告模块、监控引擎研发模块、设备管理模块和用户管理模块。
学院中心客户端作为规则管理与全局监控中枢，其模块体系构建如下：首页展示模块集成多基地运行态势总览，支持自定义预警看板（如区域违规频次热力图）与关键指标仪表盘（训练达标率、数据回传完整率）；无线传输模块通过5G专网切片技术接收分院数据，采用SRT协议实现<200ms低延迟传输，支持断点续传与zstd差分压缩算法（压缩比达10:1），确保高海拔地区的弱网稳定传输；监控引擎管理模块部署动态贝叶斯网络模型，结合强化学习算法持续优化超限阈值（如根据机型气动特性自动计算过载告警值边界条件），并建立监控规则灰度发布机制（A/B测试验证）；报告引擎管理模块集成NLG引擎与CAAC标准模板库（DOC-9859/AC-91-FS-2018-03R1），通过Jinja2模板引擎动态生成含三维航迹附图、参数超标明细表的中英文双语报告，支持电子签名与区块链存证；基础数据管理模块构建Neo4j图数据库存储机型性能库（含旋翼机悬停包线、电动垂直起降飞行器电池衰减曲线等200+机型特征），通过GraphQL接口对接民航局适航数据库实现实时同步；地图航迹还原模块基于Cesium引擎构建全国空域数字沙盘，支持多基地航班流密度可视化与空域冲突预测（蒙特卡洛仿真）；视景还原模块采用Unity3D高精度建模，实现带实时气象渲染（雨雪/低能见度）的多人协同复盘推演；参数曲线剖面模块集成Plotly可视化库，支持操纵量-飞行状态参数的多维度关联分析（如总距杆位移与旋翼转速相位关系谱分析）；监控引擎研发模块提供可视化决策树编辑器，允许飞行教员拖拽构建特情处置评估逻辑（如风切变改出策略评分模型）；设备管理模块对接IETM交互式电子手册系统，实现机载设备校准数据与飞行品质分析的自动关联；用户管理模块采用虹膜识别+硬件加密狗双重认证，依据训练阶段动态授权数据访问范围（如学员仅可见本机数据）。系统通过自研工作流引擎（基于BPMN2.0标准）串联各模块，构建从5G数据接收→实时超限检测→多源数据融合→三维场景重构→智能报告生成的全自动化流水线，同时通过Hyperledger-Fabric区块链平台实现关键数据的不可篡改存证，形成覆盖训练质量评估、规则迭代优化、安全态势研判的闭环管理体系
领导驾驶舱包括首页展示模块、飞行数据筛选器、地图航迹还原模块、视景还原模块、仪表还原模块、飞行阶段模块和参数曲线刨面模块，领导驾驶舱聚焦决策支持，飞行数据筛选器支持自然语言查询（如“显示本月超限率＞5%的机型”），利用ElasticSearch实现秒级响应，地图航迹还原模块：集成Cesium引擎实现高精度三维态势展示，可回溯历史任务的空域占用热力图，参数曲线剖面模块则采用D3.js绘制多参数联动曲线（如高度-速度-舵面偏转角），支持关键帧标注与对比分析，数据看板通过微服务架构实时更新，确保决策依据的时效性；
飞行员端包括意见模块、飞行数据筛选器、地图航迹还原模块、视景还原模块、仪表还原模块、飞行阶段模块和参数曲线刨面模块，飞行员端注重个体能力评估，视景还原模块通过Unity3D引擎重建飞行场景，支持VR模式回放操纵过程，飞行阶段模块：自动划分起降、巡航等阶段，计算各阶段操纵评分（如着陆下沉率偏离值），参数曲线剖面模块突出显示个人历史数据百分位排名，生成针对性训练建议，数据与训练模拟器接口对接，实现“飞行-分析-改进”闭环；
机务端包括机型手册管理模块、设备管理模块、首页展示模块、无线传输模块、数据上传模块和意见模块，机务端聚焦装备状态管理，机型手册管理模块用于构建知识图谱关联QRH手册、SB通告，支持AR眼镜辅助排故，设备管理模块则对接机载健康管理系统（HUMS），预测部件剩余寿命（如基于LSTM网络的发动机振动趋势预测），无线传输模块通过采用LoRaWAN协议实现机库内传感器数据的低功耗采集，维护记录自动关联飞行品质数据，为故障溯源提供交叉验证依据，各终端通过服务端消息队列实现数据协同，飞行员端反馈意见触发服务端规则库迭代，机务端维护数据反向优化飞行品质评估模型，领导驾驶舱决策指令通过学院中心下发至分院执行。全系统采用数字孪生架构，实现物理飞行与虚拟分析的深度融合。
