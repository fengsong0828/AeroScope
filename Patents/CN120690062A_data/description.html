Description

基于网格的低空飞行器碰撞预警方法及系统
技术领域
本申请涉及低空飞行领域，特别是涉及一种基于网格的低空飞行器碰撞预警方法及系统。
背景技术
随着航空技术的不断发展，低空飞行器包括小型民用飞机、直升机以及数量庞大的无人机，在众多领域得到广泛应用，如航拍、农业植保、物流配送等。然而，低空飞行环境复杂多变，存在诸多不确定因素，例如城市中的高楼大厦等障碍物，给低空飞行带来了巨大的安全挑战。
相关技术中，低空飞行器将自身的飞行数据上传至服务端，服务端根据飞行数据及城市的低空三维数据，实时计算低空飞行器与障碍物的距离来进行碰撞预警，但这种预警方法所涉及的数据计算量大，其时效性较差。
发明内容
基于此，有必要针对上述技术问题，提供一种基于网格的低空飞行器碰撞预警方法、系统、计算机设备及计算机可读存储介质。
第一方面，本发明实施例提出一种基于网格的低空飞行器碰撞预警方法，所述方法包括：
获取目标区域的低空三维数据，对所述低空三维数据进行网格化划分为若干个第一网格；
利用网格编码对各所述第一网格进行标记，及利用网格类型编码对各所述第一网格的类型进行标记，所述第一网格的类型包括空域网格和障碍物网格；及利用障碍物编码对所述障碍物网格进行标记；
针对各所述空域网格，通过网格搜索方法确定在预设范围内是否存在障碍物网格；若是，则计算各所述障碍物网格与该所述空域网格的距离，并结合各所述障碍物网格对应的网格编码及障碍物编码生成该所述空域网格的障碍物信息；
基于目标低空飞行器的飞行数据确定其所在的空域网格，获取该所述空域网格对应的障碍物信息，并结合所述飞行数据判断所述目标低空飞行器是否存在障碍物碰撞风险。

在一些实施例中，所述针对各所述空域网格，通过网格搜索方法确定在预设范围内是否存在障碍物网格；若是，则计算各所述障碍物网格与该所述空域网格的距离，并结合各所述障碍物网格对应的网格编码及障碍物编码生成该所述空域网格的障碍物信息包括：
针对搜索队列中待搜索的第一个空域网格，搜索其邻面的第一网格，根据其网格类型编码确定是否为障碍物网格，若是，则判断所搜索到的障碍物编码的个数是否大于最大障碍物个数，若是，则结束搜索；若小于最大障碍物个数，则获取该障碍物网格的障碍物编码，并计算所述第一个空域网格与该第一网格之间的距离，生成所述第一个空域网格的四元组；若该第一网格为空域网格，则将该空域网格加入所述搜索队列，并继续沿该第一网格所邻面的六个方向搜索，直到搜索范围达到预设范围；根据所有的四元组生成所述第一个空域网格的障碍物信息；
重复上述网格搜索方法的步骤，直到所述搜索队列中的各空域网格完成网格搜索，生成各空域网格的障碍物信息。
在一些实施例中，所述获取该所述空域网格对应的障碍物信息，并结合所述飞行数据判断所述目标低空飞行器是否存在障碍物碰撞风险包括：

基于所述飞行数据获取所述目标低空飞行器的飞行速度，并基于所述飞行速度确定对应的预设距离；
若该所述空域网格与该所述障碍物网格的距离小于预设距离，则确定所述目标低空飞行器存在障碍物碰撞风险。

如图1所示，本发明实施例提供了一种基于网格的低空飞行器碰撞预警方法，包括以下步骤：
S102，获取目标区域的低空三维数据，对所述低空三维数据进行网格化划分为若干个第一网格。
在一些示例实施例中，例如采用GeoSOT网格、北斗网格等网格划分方式来对低空三维数据网格化划分。
S104，利用网格编码对各所述第一网格进行标记，及利用网格类型编码对各所述第一网格的类型进行标记，所述第一网格的类型包括空域网格和障碍物网格；及针对属于同一障碍物的障碍物网格利用相同的障碍物编码对所其进行标记。
在对低空三维数据网格划分之后，针对每一个第一网格采用唯一的网格编码meshCode进行标记。每个第一网格都具有其各自的位置信息f(longitude，latitude，height)，其中longitude表示经度，latitude表示纬度，height表示高度。
利用低空三维数据判断各第一网格是空域网格还是障碍物网格，利用网格类型编码type对各所述第一网格的类型进行标记。
针对属于同一障碍物的障碍物网格及利用相同的障碍物编码obstacleId对其进行标记，对于属于不同障碍物的障碍物网格采用不同的障碍物编码obstacleId对其进行标记，例如将铁塔、桥梁等障碍物标注唯一的障碍物编码obstacleId。
在一些实施例中，判断两个所述障碍物网格之间是否临接，若是，则判断所述障碍物网格属于同一障碍物，利用相同的障碍物编码对所述障碍物网格进行标记。
在一些实施例中，还针对第一网格中的地表面网格，采用地表面类型编码groundType进行标记，例如道路、湖面、山域等。
S106，针对各所述空域网格，通过网格搜索方法确定在预设范围内是否存在障碍物网格；若是，则计算各所述障碍物网格与该所述空域网格的距离，并结合各所述障碍物网格对应的网格编码及障碍物编码生成该所述空域网格的障碍物信息。
在一些实施例中，针对某一空域网格，在其预设范围内可能存在多个障碍物，这时可以选择其中距离最近的部分障碍物。
在获得各空域网格的障碍物信息之后，将障碍物信息与各空域网格的网格编码关联后进行存储。若某一些空域网格在预设范围内不存在障碍物网格，则其障碍物信息为空。
S108，基于目标低空飞行器的飞行数据确定其所在的空域网格，获取该所述空域网格对应的障碍物信息，并结合所述飞行数据判断所述目标低空飞行器是否存在障碍物碰撞风险。
所述飞行数据包括目标低空飞行器的低空飞行数据，例如位置、方向、速度、高位等信息。根据目标低空飞行器的位置信息可以确定其所在的空域网格，获取对应的网格编码通过搜索可以得到对应的障碍物信息。
在判断目标低空飞行器存在障碍物碰撞风险时，生成报警信息输出至目标低空飞行器，进行及时报警。
本实施例中，各空域网格的障碍物信息在进行障碍物碰撞风险检测之前就已经获得，因此在进行障碍物碰撞风险检测时，只需要获取目标低空飞行器的飞行数据通过搜索就可以快速的得到障碍物信息，并及时的判断是否存在障碍物碰撞风险。相比于现有技术，本技术方案的数据计算量更小，障碍物碰撞风险检测更加快速，提高了低空飞行器飞行的安全性。
在一些实施例中，所述针对各所述空域网格，通过网格搜索方法确定在预设范围内是否存在障碍物网格包括：
针对一空域网格，依次遍历搜索该所述空域网格各方向的预设范围内的网格，根据所述第一网格的网格类型编码确定是否为障碍物网格。
在一具体实施例中，在网格搜索之前需要先设定搜索条件fun(meshCode，maxDistance，obstacleCount)，其中meshCode表示待搜索的空域网格；maxDistance表示与障碍物的最大距离，由此可以确定各方向的预设范围；obstacleCount表示最大障碍物个数。
在进行网格搜索时，初始化搜索队列，先确定待搜索的第一个空域网格meshCodestart，搜索邻面的第一网格meshCodehead，根据其网格类型编码确定是否为障碍物网格，若是，则判断所搜索到的障碍物编码的个数是否大于最大障碍物个数obstacleCount，若是，则结束搜索；若小于最大障碍物个数obstacleCount，则获取该障碍物网格的障碍物编码obstacleId，并计算空域网格meshCodestart与第一网格meshCodehead之间的距离Distance，生成该空域网格的四元组（meshCodestart，obstacleId，meshCodehead，Distance）。若邻面的第一网格meshCodehead为空域网格，则将该空域网格加入搜索队列，并继续沿第一网格meshCodehead邻面的六个方向搜索，将未被处理过的邻面的第一网格逐一重复执行上述步骤，直到搜索范围达到预设范围。针对空域网格meshCodestart的网格搜索完成后，对所有的四元组进行处理得到空域网格meshCodestart的障碍物信息。针对搜索队列中的空域网格，重复上述网格搜索方法，得到所有空域网格的障碍物信息。

其中的预设距离根据目标低空飞行器的飞行速度所确定，其飞行速度越快对应的预设距离越大，当空域网格与该所述障碍物网格的距离小于预设距离，则确定目标低空飞行器是否存在障碍物碰撞风险。本实施例根据目标低空飞行器的飞行数据实现对触发障碍物碰撞风险的自适应调整。
由于对低空三维数据进行网格化划分后得到的第一网格的数量巨大，利用第一网格进行网格搜索其效率低。
为解决该技术问题，在一些实施例中，对所述低空三维数据进行网格化划分为若干个第二网格，各所述第二网格包括多个所述第一网格。第一网格可以理解为第二网格的子网格，例如第二网格包括64个第一网格。第二网格包括各第一网格的标记。
针对一空域网格，先依次遍历该空域网格所在第二网格中的第一网格，根据所述第一网格的网格类型编码确定其中的障碍物网格；再依次遍历搜索该所述空域网格各方向的预设范围内的第二网格，判断所述第二网格中是否存在障碍物网格，若存在，则依次遍历所述第二网格中的第一网格，根据所述第一网格的网格类型编码确定其中的障碍物网格。
针对第二网格可以采用与第一网格相同的网格搜索方法，在本实施例中不再赘述。
相比于利用第一网格进行网格搜索，本实施例采用的网格搜索方法采用分层搜索的方式，先搜索第二网格来快速判断是否存在障碍物网格，在存在障碍物网格的情况下，再搜索其中的第一网格实现障碍物网格的快速定位，提高了网格搜索效率。

S302，当所述低空三维数据更新时，获取更新区域对应的第一网格；
S304，利用网格编码对所述更新区域对应的第一网格进行重新标记，及利用网格类型编码对所述更新区域对应的第一网格的类型进行重新标记；及利用障碍物编码对所述障碍物网格进行标记；
S306，针对距离所述更新区域对应的第一网格在预设范围内的空域网格，通过网格搜索方法确定在预设范围内是否存在障碍物网格；若是，则计算各所述障碍物网格与该所述空域网格的距离，并结合各所述障碍物网格对应的网格编码及障碍物编码更新该所述空域网格的障碍物信息。
现有技术中，当低空三维数据更新时，需要对更新后的所有低空三维数据重新划分网格、重新标记、重新网格搜索生成障碍物信息，因此数据的处理量大，数据的处理时间长。
在本实施例中，只需要针对更新区域对应的第一网格进行重新标记，重新网格搜索生成障碍物信息，与现有技术相比，数据处理量减少，数据的处理时间大大缩短。
在一些实施例中，针对不同预设高度区间的低空三维数据，采用不同粒度的网格化划分；其中，所述预设高度区间越大，对应网格化划分的粒度越大。
在一示例实施例中，500米至1000米的高度范围内，网格的粒度为100米×100米，0至500米的高度范围内，网格的粒度为10米×10米。
可以理解的是，在相对的高空区域，存在障碍物的可能性较低，因此可以采用粒度较大的网格划分；在相对的低空区域，存在障碍物的可能性较大，因此需要采用粒度更小的网格划分。本实施例所采用的网格划分方法，大大降低了网格划分的时间。
