Description

一种低空无人飞行器航线偏离监测方法
技术领域
本发明涉及飞行器监测技术领域，具体涉及一种低空无人飞行器航线偏离监测方法。
背景技术
随着航空技术的不断进步和低空无人飞行器(无人机)的广泛应用，其在航拍、货物运输、环境监测等众多领域发挥着越来越重要的作用。然而，随着无人机飞行任务的多样化和航线复杂度的增加，航线偏离的问题也日益凸显。传统的航线偏离监测方法往往依赖于飞行员的目视观察和简单的定位设备，这种方法在实时监测和精确性方面存在明显不足。
发明内容
本申请提供一种低空无人飞行器航线偏离监测方法，用于针对解决现有技术在实时监测和精确性方面存在不足，难以满足复杂飞行环境下安全需求的技术问题。
鉴于上述问题，本申请提供了一种低空无人飞行器航线偏离监测方法。
本申请的第一个方面，提供了一种低空无人飞行器航线偏离监测方法，所述方法包括：
明确目标区域和无人飞行器，其中，所述目标区域包括飞行环境和天气环境，所述无人飞行器包括无人飞行器基础信息；对所述无人飞行器进行多源传感器布设，基于所述多源传感器获取多源定位数据，根据所述多源定位数据进行数据融合，获取所述无人飞行器的高精度定位数据；构建航线偏离识别通道，采集所述目标区域和所述无人飞行器的历史数据，所述航线偏离识别通道通过所述历史数据进行监督训练；将所述目标区域和所述高精度定位数据输入所述航线偏离识别通道进行偏离识别，获取偏离识别结果；搭建云端平台，所述云端平台包含所述目标区域、所述无人飞行器、所述航线偏离识别通道和控制中心，将所述偏离识别结果输入所述云端平台对所述偏离识别结果进行处理并展示。

实施例一
如图1所示，本申请提供一种低空无人飞行器航线偏离监测方法，所述方法包括：
步骤S100：明确目标区域和无人飞行器，其中，所述目标区域包括飞行环境和天气环境，所述无人飞行器包括无人飞行器基础信息；
在本申请实施例中，利用无人机航测技术，通过无人机搭载的高清摄像头和传感器，对目标区域进行高精度地形测绘，识别出地形特征、建筑物、电线杆等潜在障碍物。借助雷达、ADS-B等技术，实时监测目标区域内的其他飞行活动，确保无人机在飞行时能够避免与其他飞行器发生碰撞。
通过与气象站点的数据连接，获取目标区域实时的风速、风向、温度、湿度等关键气象信息。
通过详细的飞行器技术规格表，获取无人飞行器的型号、尺寸、重量、飞行速度、飞行高度、续航时间等无人飞行器基础信息。
步骤S200：对所述无人飞行器进行多源传感器布设，基于所述多源传感器获取多源定位数据，根据所述多源定位数据进行数据融合，获取所述无人飞行器的高精度定位数据；
在本申请实施例中，在无人飞行器上会安装多种类型的传感器，通过光学相机和高清摄像头获取地面的高分辨率图像，辅助视觉定位和环境感知。通过IMU提供飞行器的角速度、加速度和姿态信息。利用GPS接收器提供全球范围内的定位信息。利用激光雷达通过测量激光束的反射时间来确定距离，生成三维点云数据，用于环境建模和避障。利用超声波和红外传感器对近距离的障碍物进行检测和避碰。
这些传感器在飞行过程中会持续收集数据，如GPS提供经纬度，IMU提供飞行姿态，激光雷达扫描周围环境等。为提高定位精度和鲁棒性，采用数据融合技术对多源传感器数据进行处理，通过卡尔曼滤波或扩展卡尔曼滤波结合多个传感器的数据，通过预测和更新步骤，对飞行器的状态进行最优估计。由于不同传感器的采样频率和数据输出可能存在延迟或不同步，因此需要对数据进行时间同步和空间对齐处理。对于可能出现的传感器异常值，如GPS信号失锁或激光雷达的噪声点，进行数据清洗和剔除。
经过数据融合处理后，得到无人飞行器的高精度定位数据。
步骤S300：构建航线偏离识别通道，采集所述目标区域和所述无人飞行器的历史数据，所述航线偏离识别通道通过所述历史数据进行监督训练；
在本申请实施例中，构建航线偏离识别通道时，采用基于深度学习或其他机器学习算法的航线偏离识别模型。这些模型能够学习和识别航线偏离的模式。
通过数据库收集目标区域的地理信息、气象数据、障碍物位置等，同时收集无人飞行器的历史飞行数据，包括飞行轨迹、速度、高度、姿态等。
之后对收集到的历史数据进行清洗、标注和格式化，以便用于训练。用标注过的历史数据来训练航线偏离识别模型。通过不断调整模型参数，使其能够准确识别航线偏离的情况。使用验证数据集来评估模型的性能，并根据评估结果进行必要的调整和优化。
训练完成后，获得航线偏离识别通道。
步骤S400：将所述目标区域和所述高精度定位数据输入所述航线偏离识别通道进行偏离识别，获取偏离识别结果；
在本申请实施例中，整合目标区域的详细地理信息，包括地形、障碍物、空域限制等，以及无人飞行器的实时高精度定位数据。将整合后的数据通过数据清洗、转换和标准化等步骤格式化为航线偏离识别通道所能接受的格式。
之后将整合后的数据输入航线偏离识别通道，利用航线偏离识别通道中训练好的航线偏离识别模型，对输入的目标区域信息和高精度定位数据进行分析，获得偏离识别结果。
步骤S500：搭建云端平台，所述云端平台包含所述目标区域、所述无人飞行器、所述航线偏离识别通道和控制中心，将所述偏离识别结果输入所述云端平台对所述偏离识别结果进行处理并展示。
在本申请实施例中，根据业务需求和技术要求，选择一个可靠的云服务提供商，例如亚马逊AWS、阿里云、华为云等。在云服务上配置适当的服务器资源，包括CPU、内存、存储和网络带宽等，以满足实时数据处理和存储的需求。
通过API或SDK将目标区域的地理信息、气象数据以及无人飞行器的实时飞行数据接入云端平台。将训练好的航线偏离识别模型部署到云端平台上，确保其能够实时处理输入的数据。云端平台通过API接口接收航线偏离识别通道输出的偏离识别结果。
对接收到的结果进行进一步的处理和分析，如计算偏离的具体数值、判断偏离的严重程度等。
最后设计一个直观的用户界面，用于展示航线偏离识别结果和其他相关信息，如偏离的图形化表示、偏离的具体数值和建议的纠正措施等。同时提供交互功能，允许用户查看实时飞行数据、接收偏离识别结果。
进一步的，申请实施例提供的方法中的步骤S200还包括：
对所述无人飞行器进行多源传感器布设，并调整所述多源传感器的通信情况，确保所述多源传感器和控制中心的通信畅通；
启动所述无人飞行器并激活所述多源传感器，进行基础定位数据采集，获取多源定位数据；
根据所述多源定位数据进行数据预处理，获取处理定位数据；
确定所述多源传感器的精度，基于所述精度进行权重参数分配，对所述处理定位数据进行加权计算，确定高精度定位数据。
在本申请实施例中，根据飞行任务需求，在无人飞行器上安装多种传感器，例如光学相机、GPS接收器、IMU等，实现多维度的环境监测和定位。配置传感器与控制中心之间的通信协议，确保数据传输的稳定性和实时性。调整传输频率和带宽，以适应不同传感器数据的特点，减少传输延迟和数据丢失的可能性。
按照操作规程安全地启动无人飞行器，激活所有多源传感器，开始数据采集工作。各个传感器按照设定的频率和参数进行数据采集，如GPS位置信息、IMU的姿态数据等。通过时间戳等方式确保从不同传感器采集的数据在时间上保持同步，获取多源定位数据。
之后对多源定位数据进行清洗，去除噪声和异常值，提高数据质量。应用滤波算法，如卡尔曼滤波进一步平滑数据，减少误差。通过参考技术规格书，评估每个传感器的定位精度和稳定性。
最后根据传感器的精度评估结果，为每个传感器分配不同的权重。采用加权计算方法，将不同传感器的数据融合，生成高精度定位数据。
进一步的，申请实施例提供的方法中的步骤S300还包括：
采集所述目标区域和所述无人飞行器的历史数据，所述历史数据包括正常飞行数据和偏离飞行数据，其中，所述偏离飞行数据包括环境偏离和飞行器偏离；
构建航线偏离识别通道，所述航线偏离识别通道包括环境识别通道和飞行器识别通道；
将所述目标区域输入所述环境识别通道，判断所述目标区域环境是否适合所述无人飞行器启动；
若是，则直接判定所述无人飞行器航线偏离；
若否，启动所述无人飞行器，并基于所述无人飞行器判断航线偏离情况。
在本申请实施例中，从历史数据库收集目标区域和无人飞行器的历史数据，将收集到的数据分为正常飞行数据和偏离飞行数据，偏离数据进一步细分为由环境因素导致的偏离和飞行器自身问题导致的偏离。
利用机器学习算法和深度学习模型，如卷积神经网络CNN、长短期记忆网络LSTM构建航线偏离识别通道。航线偏离识别通道包括环境识别通道和飞行器识别通道。环境识别通道使用天气预测模型、地形分析模型和障碍物检测算法评估飞行环境的安全性。飞行器识别通道用于预测飞行器的性能和状态。
在飞行前，将目标区域的环境数据，如气象预报、地形高度图等输入到环境识别通道中进行分析。使用流数据处理技术，如Apache Flink、Storm等对实时传感器数据进行处理，以支持快速决策。
如果环境识别通道判断当前环境不适合飞行，例如，有强烈的风切变、雷暴等，则直接判定无人飞行器航线偏离。
如果环境适合飞行，启动无人飞行器，并通过飞行器识别通道实时监测飞行状态，判断航线偏离情况。
进一步的，构建航线偏离识别通道，所述方法还包括：
构建航线偏离识别模型，所述航线偏离识别模型的输入数据为所述目标区域和所述高精度定位数据，输出数据为偏离识别结果；
获取采集所述目标区域和所述无人飞行器的历史数据，以及对应的航线偏离结果；
基于所述目标区域、所述无人飞行器的历史数据和所述航线偏离结果构建样本集；
通过所述样本集对初始航线偏离识别模型进行训练和验证，获得所述航线偏离识别模型，所述航线偏离识别模型内嵌于所述航线偏离识别通道。
在本申请实施例中，首先确定目标区域，目标区域是一个具体的地理区域，比如一片特定的空域或海域。收集无人飞行器的历史数据，包括其飞行轨迹、时间戳、速度、航向等。同时获取这些历史数据对应的航线偏离结果。
将收集到的历史数据和航线偏离结果整合，构建一个样本集。每个样本包括目标区域的信息、无人飞行器的实时数据以及对应的航线偏离标签。
对于复杂的航线偏离识别任务，选择卷积神经网络模型作为初始航线偏离识别模型。使用构建的样本集对初始航线偏离识别模型进行训练。使用随机划分或分层抽样等方法将样本集分为训练集和验证集，训练集用于训练模型，验证集用于评估模型的性能并防止过拟合。
使用训练集对模型进行训练，通过反向传播算法和梯度下降优化器调整模型参数。采用批量训练、学习率衰减等技术来提高训练效率和模型性能。使用验证集评估模型的性能，计算准确率、召回率、F1分数等指标。根据验证结果调整模型参数或尝试不同的模型结构。
最后将训练好的航线偏离识别模型集成到航线偏离识别通道中。
进一步的，所述方法还包括：
获取预设数据划分比例，按照所述预设数据划分比例将所述样本集划分为训练集和验证集；
通过所述训练集对所述初始航线偏离识别模型进行监督训练，当模型输出结果趋于收敛状态时，通过所述验证集对所述初始航线偏离识别模型的输出结果进行验证；
获取预设模型验证准确率指标，当所述初始航线偏离识别模型的输出结果准确率满足所述预设模型验证准确率指标时，获得所述航线偏离识别模型。
在本申请实施例中，预设数据划分比例为根据经验、数据集的大小以及问题的复杂性确定。本申请划分比例为70:30，利用随机抽样技术将样本集划分为训练集和验证集。
与前述说明过程相同，使用训练集对初始模型进行监督训练。通过输入训练样本和对应的航线偏离标签，调整模型的参数以最小化预测误差。利用反向传播算法和梯度下降优化器等先进技术来高效训练模型，并监控训练过程中的损失函数和准确率等指标。
当模型在训练集上的性能趋于稳定后，使用验证集对模型进行验证。将验证集的样本输入到模型中，获得模型的预测结果，并与真实的航线偏离标签进行对比获取模型的准确率。之后将模型的准确率与预设模型验证准确率指标进行对比，判断初始航线偏离识别模型的输出结果准确率是否满足预设模型验证准确率指标。预设模型验证准确率指标根据实际需求和行业标准由技术专家设定。
如果模型的准确率满足或超过预设指标，则认为模型训练成功，获得航线偏离识别模型。

目标区域管理模块负责存储和管理目标区域的信息，包括地理位置、边界、限制条件等。提供接口供其他模块查询和更新目标区域信息。

搭建云端平台时，选择合适的云服务提供商，如Amazon Web Services，GoogleCloud Platform，或MicrosoftAzure等。设置基础设施，配置虚拟机、数据库、存储和网络等基础设施。将上述规划的模块开发成微服务或单体应用，并部署到云端。
开发通信接口时，定义通信协议，确定无人飞行器和云端平台之间的通信标准和数据格式。使用MQTT、WebSocket或其他实时通信协议实现数据的双向传输。出于安全性考虑，对数据进行加密和签名，以防止数据泄露和篡改。通过前述过程确保偏离识别结果能够实时传输到云端平台。

对所述高风险区域，所述云端平台提高对航线偏离的敏感度，分配相应算力处理所述偏离识别结果。
在本申请实施例中，利用大数据分析技术，对历史航线数据进行深度挖掘，识别出航线偏离频繁或事故多发的区域。通过分析飞行日志、气象数据等多源信息，发现潜在的风险模式和趋势。通过上述步骤，确定目标区域的高风险区域。
在高风险区域，云端平台提高对航线偏离的敏感度，降低航线偏离的阈值，即一旦无人机的实际航线与预定航线的偏差超过一个更小的范围，触发偏离警报。例如，对于一般区域，允许无人机航线偏离预定航线50米才触发警报，而在高风险区域，这个阈值被降低到20米或更小。
最后云端平台利用云计算的动态资源分配功能，根据高风险区域中航线偏离识别的需求，动态分配相应算力处理偏离识别结果。

基于所述目标区域选择地图，并实时接收所述无人飞行器的位置数据，更新在所述地图；
当所述航线偏离识别通道检测到偏离时，获取所述偏离识别结果，使用视觉元素在所述地图上标注偏离点；
绘制所述无人飞行器实际飞行的路径，并与预设航线进行对比，进行偏离路径展示。
在本申请实施例中，根据飞行任务的目标区域，选择适当比例和详细程度的地图，例如卫星图像、地形图或其他专用地图。通过无人机上的GPS或其他定位系统，实时获取无人机的经度、纬度和高度等数据。这些数据通过无线通信链路实时传输到地面控制站。将接收到的无人机位置数据实时叠加到所选地图上。
在飞行前，通过专业的航线规划软件为无人机设定精确的预设航线。当航线偏离识别通道检测到偏离时，生成偏离识别结果，确定偏离点的精确位置。在地图上，通过AR技术或动态图层，使用醒目的视觉元素，如闪烁的图标、警告颜色等对偏离点进行实时标注。
最后利用大数据分析技术，实时记录并处理无人机的飞行数据，生成其实际飞行路径，完成偏离路径展示。
