Description

一种低空飞行器起降平台选址优化方法
技术领域
本发明涉及无人驾驶飞行器(UAV)的管理与优化领域，具体涉及一种低空飞行器起降平台选址优化方法。
背景技术
随着无人机技术的迅猛发展，尤其是在城市空中交通(Urban Air Mobility，UAM)中的广泛应用，无人机在低空空域的使用频率和密度显著增加，但同时也带来了诸多问题，包括空域资源的高效利用、飞行安全性的保障以及多无人机协同作业的管理等。
现有的低空飞行器起降平台选址优化方法在面对高密度飞行环境和动态空域变化时，存在以下主要不足：现有系统在整合交通、气象、地理、环境等多源数据方面存在滞后，无法全面反映城市动态变化，影响选址优化的准确性和实时性；传统优化算法在处理多目标和复杂约束条件时效率低下，难以满足高效、灵活的选址需求；无法实时响应城市环境的快速变化，导致选址方案难以动态调整，降低了系统的适应性和稳定性；数字孪生体作为先进的模拟和预测工具，在低空飞行器起降平台选址优化中的应用尚不普遍，无法充分利用其提供的模拟和实时更新能力。
为解决上述问题，本发明提出一种低空飞行器起降平台选址优化方法，通过数字孪生体构建虚拟城市模型，为选址优化提供精准的模拟环境和实时反馈，以提升选址方案的准确性和可行性，进而提升无人机飞行的安全性和运营效率。
发明内容
鉴于上述问题，本发明提供一种低空飞行器起降平台选址优化方法，包括：
步骤S1，获取包括城市交通流量数据、气象数据、地形地貌数据、环境数据、POI数据的实时动态数据和静态GIS数据；
步骤S2，根据所述静态GIS数据构建城市三维数字模型，根据所述三维数字模型和所述实时动态数据构建数字孪生体模型；
步骤S3，构建所述数字孪生体模型中候选起降平台位置的环境约束条件和安全约束条件；其中，所述环境约束条件包括地形坡度、建筑物高度、障碍物分布、风向风速和噪音影响；所述安全约束条件包括安全距离、航线规划和紧急情况下的备降点；
步骤S4，通过粒子群优化算法对所述数字孪生体模型进行多目标优化，生成满足所述环境约束条件和安全约束条件的最优候选起降平台选址方案集；
步骤S5，根据实时数据反馈动态更新所述数字孪生体模型，对所述起降平台选址方案进行动态调整。
在一种可选的方式中，在所述步骤S1中：
从城市交通监控系统实时采集道路交通流量和公共交通站点数据；
从气象站或在线气象服务实时采集温度、湿度、风速和风向气象数据；
从GIS数据库获取城市地形地貌、建筑物分布和土地使用类型信息，并进行三维建模得到城市三维空间信息数据；
从环境监测站点实时采集噪声和空气质量环境数据；
从公共数据源或第三方服务获取包括城市中医院、学校、商业区和交通站点的POI数据。
在一种可选的方式中，在所述步骤S2中，所述根据所述静态GIS数据构建城市三维数字模型进一步包括：
根据所述静态GIS数据构建城市基础三维模型，通过API接口实时接入所述实时动态数据；
将所述实时动态数据映射到所述城市三维数字模型中，实时更新所述城市三维数字模型的参数，以反映城市环境的动态变化。
在一种可选的方式中，在所述步骤S4中，所述通过粒子群优化算法对所述数字孪生体模型进行多目标优化进一步包括：
根据城市人口密度、交通流量、飞行器起降需求、OD出行需求以及区域服务需求生成需求评估模型；
根据所述需求评估模型和所述环境约束条件和安全约束条件，对所述数字孪生体模型中的候选选址位置进行多目标优化，生成满足所述需求评估模型、所述环境约束条件和安全约束条件的可选起降平台选址方案。
在一种可选的方式中，所述根据城市人口密度、交通流量、飞行器起降需求、OD出行需求以及区域服务需求生成需求评估模型进一步包括：
根据人口密度和收入水平数据评估高端社区的起降需求；
根据交通流预测模型分析商务区高峰时段的需求波动；
基于历史急救数据评估医院周边紧急服务需求；
根据空域限制和交通需求优化机场交通枢纽的平台位置，以最大化服务效率；
根据地面交通枢纽数据优化地铁站附近平台的可达性。
在一种可选的方式中，所述环境约束条件进一步包括：
根据噪声传播模型和环境监测数据，动态设置噪声缓冲区；通过视域计算模型优化平台的可视性，确保飞行器安全起降；
结合地形和建筑数据，使用空间分析工具设置避障区域；
根据法律和政策数据库，排除禁飞区域内的选址点；
根据飞行高度限制数据库，自动排除禁飞区域内的选址点；
根据数字高程模型进行坡度分析，确保选址区域的物理条件。
在一种可选的方式中，所述生成满足所述环境约束条件和安全约束条件的最优候选起降平台选址方案集进一步包括：
将各个区域的需求权重与环境约束条件结合，利用加权算法对所有候选点进行评分，生成初步选址方案；
使用粒子群优化算法评估每个候选选址点的需求和环境适应性，将需求加权与环境约束条件相结合，搜索适合的选址位置并动态调整粒子的位置，找到满足所有条件的最优选址方案；其中，对不满足约束条件的候选点施加惩罚权重，避免将不合适的区域选入最终方案；
在迭代过程中动态调整搜索空间的边界，其中，当选址点不符合约束条件时，该选址点被移出搜索空间或者搜索空间的可行域被重新定义为不包含违约点的区域，筛选符合需求的候选平台位置。
在一种可选的方式中，所述方法还包括，对所述候选平台位置进行评估，具体包括：
评估每个候选平台的需求覆盖度，计算其对高需求区域的覆盖比例；
通过模拟分析评估选址方案在噪声、飞行高度、视域、避障的适应性；
结合城市的禁飞区和飞行高度限制，对选址方案进行法律合规性评估；
根据选址区域的建设成本维护成本以及预期的使用收益，评估候选平台位置的经济效益。
在一种可选的方式中，所述方法还包括：
将需求覆盖度、环境适应性、法律合规性、成本效益的评估结果进行加权计算，得出综合评分；
根据综合评分的高低，对所有选址方案进行排序，确定最佳的低空飞行器起降平台选址方案。
在一种可选的方式中，所述方法还包括：
将所述最佳的低空飞行器起降平台选址方案在所述数字孪生体模型中进行可视化呈现，并生成包含选址位置、评估结果和约束条件的选址报告。
根据本发明提供的方案，包括：步骤S1，获取包括城市交通流量数据、气象数据、地形地貌数据、环境数据、POI数据的实时动态数据和静态GIS数据；步骤S2，根据所述静态GIS数据构建城市三维数字模型，根据所述三维数字模型和所述实时动态数据构建数字孪生体模型；步骤S3，构建所述数字孪生体模型中候选起降平台位置的环境约束条件和安全约束条件；其中，所述环境约束条件包括地形坡度、建筑物高度、障碍物分布、风向风速和噪音影响；所述安全约束条件包括安全距离、航线规划和紧急情况下的备降点；步骤S4，通过粒子群优化算法对所述数字孪生体模型进行多目标优化，生成满足所述环境约束条件和安全约束条件的最优候选起降平台选址方案集；步骤S5，根据实时数据反馈动态更新所述数字孪生体模型，对所述起降平台选址方案进行动态调整。本发明通过数字孪生体构建虚拟城市模型，为选址优化提供精准的模拟环境和实时反馈，提升了选址方案的准确性和可行性。

具体实施方式
下面将参照附图更详细地描述本发明的示例性实施例。虽然附图中显示了本发明的示例性实施例，然而应当理解，可以以各种形式实现本发明而不应被这里阐述的实施例所限制。相反，提供这些实施例是为了能够更透彻地理解本发明，并且能够将本发明的范围完整的传达给本领域的技术人员。
图1示出了本发明实施例的低空飞行器起降平台选址优化方法的流程示意图。具体地，如图1所示，包括以下步骤：
步骤S1，获取包括城市交通流量数据、气象数据、地形地貌数据、环境数据、POI数据的实时动态数据和静态GIS数据。
本实施例中，通过整合实时动态数据，可以及时感知气象变化、交通状况、环境污染等，及时调整飞行计划，降低事故风险。结合地形地貌数据提前规划飞行路线，避开障碍物，确保飞行安全。根据交通状况、气象条件动态调整飞行路径。利用有利风向降低飞行能耗，提高能源利用效率。结合POI数据选择最便捷的起降地点。通过环境监测数据降低UAM运行对环境的影响，提升公众接受度。
在一种可选的方式中，在所述步骤S1中：
从城市交通监控系统实时采集道路交通流量和公共交通站点数据；
从气象站或在线气象服务实时采集温度、湿度、风速和风向气象数据；
从GIS数据库获取城市地形地貌、建筑物分布和土地使用类型信息，并进行三维建模得到城市三维空间信息数据；
从环境监测站点实时采集噪声和空气质量环境数据；
从公共数据源或第三方服务获取包括城市中医院、学校、商业区和交通站点的POI数据。
本实施例中，如图2所示，通过城市交通监控系统实时获取道路交通流量、公交车站、地铁站等交通枢纽的实时数据，并对交通数据进行多维度分析，如交通密度、平均车速、交通拥堵程度等，提供优化模型所需的交通状况数据。
其中，Dtraffic为交通密度评分，Tvehicle为单位时间内通过车辆数量，A为监测区域面积。
通过气象站或在线气象服务收集实时的温度、湿度、风速、风向等气象数据，并将气象变化趋势与历史数据进行对比，识别潜在的极端天气模式，对飞行器起降影响进行预测。
WWeather＝f(T1，H，V，D)
其中，WWeather为气象评分，T1、H、V、D分别为温度、湿度、风速、风向。
使用地理信息系统(GIS)技术获取城市的地形地貌、建筑物分布、土地使用类型等信息，结合3D建模技术，提供高精度的城市三维空间数据。
GGeography＝GIS(L，E，U)
其中，GGeography为地理评分，L为地形数据，E为建筑物分布数据，U为土地使用类型数据。
为此，进一步说明如下：
L(地形数据)：包括数字高程模型(DEM)数据，用于描述区域的坡度、海拔等地形特征。
E(建筑物分布数据)：包括建筑物的高度、密度和分布情况，通常通过矢量或点云数据获得。
U(土地使用类型)：反映不同区域的土地利用状态(如居住、商业、工业等)，一般来源于土地利用规划或GIS数据库。
GIS”函数不是单一数学函数，而是通过现有GIS软件或开发库(如ArcGIS、QGIS、GeoPandas等)，采用数据叠加和空间分析方法，对地形、建筑分布和土地利用等数据进行归一化处理和加权融合。为提高精度，可采用如下加权模型：
GGeography＝wL×fL(L)+wE×fE(E)+WU×fU(U)
其中：
fL(L)、fE(E)和fU(U)分别对原始数据进行归一化(或采用指数映射函数)处理；
wL、wE和wU分别为fL(L)、fE(E)和fU(U)的权重，依据区域实际情况(如地形起伏、建筑密度重要性等)通过专家调校或层次分析法(AHP)确定。
通过环境监测站点收集噪声、空气质量等环境数据，结合城市环境噪声地图，评估不同区域对低空飞行器起降的影响。
EEnvironment＝α×Noise+β×AirQuality
其中，α和β为权重系数，Noise(噪声)和AirQuality(空气质量)分别为环境评分指标，系数满足α+β＝1。
Noise(噪声)：代表区域环境噪声水平，数据来源于城市环境噪声监测或噪声地图。
AirQuality(空气质量)：反映区域空气质量情况，数据来源于环境监测站或相关公共数据。
α和β(权重系数)：为噪声与空气质量在整体环境评分中的相对权重，其具体取值依据实际区域特征、历史数据统计及专家评估确定。例如，在人口稠密区域中噪声的影响可能更为显著，则可设置α取值高于β，如α＝0.6，β＝0.4；在空气质量问题较为突出的区域，权重则可进行相应调整。
从公共数据源或第三方服务获取城市中医院、学校、商业区、交通站点等POI数据，建立城市关键点的空间数据库，为平台选址提供依据。
POI数据库＝{医院，学校，商业区，交通站点}
然后，如图3所示，对上述数据进行预处理，包括：
数据去重，对采集的数据进行去重处理，避免重复数据影响后续分析，使用哈希表或数据库去重算法，确保数据的唯一性。
空值填充，使用均值插补、最近邻插补或机器学习算法(如K近邻)填充缺失的数据，并根据数据特性选择适当的填充策略。
异常值检测，通过统计分析和机器学习方法(如孤立森林、支持向量机(SVM))检测并剔除异常值，确保数据的准确性与有效性。
数据标准化，采用Z-score标准化或Min-Max标准化方法，对不同数据源进行标准化处理，确保数据符合后续处理和建模的要求，统一尺度和范围，避免由于量纲不同造成的分析偏差。
如图4所示，将城市的地理信息系统(GIS)数据整合到数字孪生体中，构建城市的基础模型，包括地形、建筑物和道路网的三维数据，并将其与其他模块的数据进行融合。
通过API接口接入实时交通、气象、环境监测等动态数据，采用高效的数据流处理技术，确保数据接入的实时性和稳定性。
步骤S2，根据所述静态GIS数据构建城市三维数字模型，根据所述三维数字模型和所述实时动态数据构建数字孪生体模型。
本实施例中，静态GIS数据和三维数字模型提供高精度的城市地理信息，准确反映城市建筑和基础设施的详细信息。通过实时动态数据数字孪生模型实时反映城市的运行状态。
具体地，使用高精度的遥感设备(如LiDAR)进行城市地形和建筑物的数据采集，生成高精度的地形模型和建筑模型。使用3D建模软件将地形模型和建筑模型进行融合，生成城市三维数字模型。将三维数字模型与实时动态数据进行融合生成数字孪生模型。
在一种可选的方式中，在所述步骤S2中，所述根据所述静态GIS数据构建城市三维数字模型进一步包括：
根据所述静态GIS数据构建城市基础三维模型，通过API接口实时接入所述实时动态数据；
将所述实时动态数据映射到所述城市三维数字模型中，实时更新所述城市三维数字模型的参数，以反映城市环境的动态变化。
