Description

一种低空飞行器智能鉴别与应急反制系统
技术领域
本发明涉及低空飞行器安全管理技术领域，具体涉及一种低空飞行器智能鉴别与应急反制系统。
背景技术
目前常用的无人机等低空飞行器探测手段主要有雷达、无线电频谱、光电和声波等，这些探测手段各有各的特点，适用场景也各不相同。这些探测传感器均存在一定的检测概率和虚警概率，采用某种单一探测手段不可避免会出现目标漏报和误报虚警，其中目标漏报会带来防控区域的安全漏洞，而误报虚警又会引起执勤人员不必要的心理恐慌和可能带来的额外的警情处置成本。
另外，发现和鉴别出恶意飞行器后，需要以最快速度采取适当的反制手段来应对并消除威胁，避免或最小化可能造成的各方面损失，当前低空飞行器反制技术发展也日益成熟，主要包括导航压制干扰、导航诱骗干扰、飞行器指令干扰、射弹拦截、捕获网拦截、高功率微波毁伤、激光毁伤等处置手段。但是各种低空飞行器反制设备的装备成本和打击效费比差别很大，防控区域内不同点位的重要程度不尽相同，不同低空飞行器反制设备的部署位置和数量配置也会有所区别。
公开号为CN112598046B的中国专利，公开了一种多机协同空战中的目标战术意图识别方法，能够针对多机协同战术推理展开研究，建立多机协同空战中的目标战术意图识别模型。具体地，通过构建能够处理不同线性度及关联度特征的SVM分类器，提升传统战术意图识别方法的模型性能。并且，结合传统方法所使用参数，提出多种协同战术特征提取方法，在初步特征中提取出更高级别的特征。另外，通过空战仿真样本训练决策树模型，然后将上述全部特征输入训练好的模型中，有效的识别出协同战术意图，从而有效满足实战需求；
但是该方法是针对战时的敌对无人机进行的识别，而对于伪装成正常无人机的恶意无人机来说，该方法无法奏效；
因而，我们提出一种能够准确鉴别出恶意无人机，并能对其进行及时反制的系统。
发明内容
本发明的目的在于提供一种低空飞行器智能鉴别与应急反制系统，其能够准确鉴别出恶意无人机，并能对其进行及时反制。
本发明通过以下技术方案实现：
一种低空飞行器智能鉴别与应急反制系统，包括探测单元、鉴别单元、指挥决策平台、调度单元和反制单元；
所述探测单元用于探测防控区域内的所有低空飞行器，获得所有低空飞行器的飞行数据；
所述鉴别单元用于接收探测单元获得的飞行数据，并基于融合算法，分析判断出所有低空飞行器的威胁等级；
所述指挥决策平台用于接收各低空飞行器的威胁等级，根据防控区域的重要程度以及低空飞行器的威胁等级，向反制单元发送处置指令；
所述调度单元用于根据处置指令，结合城市全域反制设备的部署信息，通过智能优化调度算法，给出具体的反制设备调度指令；
所述反制单元根据调度指令，控制不同反制设备对相应威胁等级的低空飞行器进行精准打击。
进一步的，所述探测单元包括雷达探测设备、无线电频谱侦测设备、光电探测设备和声波探测设备，探测单元中各设备之间的物联网组网，以及探测设备部署信息数据库。
进一步的，所述融合算法包括循环神经网络模型、信息提取模块、贝叶斯网络推理模型和置信度权重计算模型。

实施例1
如附图1所示的一种低空飞行器智能鉴别与应急反制系统，包括探测单元、鉴别单元、指挥决策平台、调度单元和反制单元；
所述探测单元用于探测防控区域内的所有低空飞行器，获得所有低空飞行器的飞行数据；
特别的，所述探测单元包括雷达探测设备、无线电频谱侦测设备、光电探测设备和声波探测设备，探测单元中各设备之间的物联网组网，以及探测设备部署信息数据库；
探测单元主要实现对防控区域内飞行的各种低空飞行器的实时探测和跟踪，利用各类型传感器输出各飞行器的实时三维位置、速度、图像、声波参数（频率/幅度）等信息。
所述鉴别单元用于接收探测单元获得的飞行数据，并基于融合算法，分析判断出所有低空飞行器的威胁等级；
其中鉴别单元的硬件为一套机架式高性能服务器，配置高性能CPU、多张GPU、大量内存和快速存储，网络带宽至少10Gbps，电源具备高可靠性，散热系统强效，同时考虑数据备份、集群管理、自动化部署和状态监控等软件和系统支持。鉴别单元对恶意飞行器鉴别的总流程如图2所示。飞行器威胁等级预定义为0-9共10级，威胁等级0级代表无恶意、正常执行任务的飞行器，数字越大表示威胁程度越高。所述恶意飞行器鉴别流程主要包括循环神经网络分类、贝叶斯网络推理和综合权重计算三大步骤，循环神经网络分类和贝叶斯网络推理均可得出一个目标威胁等级值，综合权重计算步骤分别为上述两步骤得出的结果计算不同的权重并得出最终的各目标威胁等级值。探测设备获取的距离、高度、速度等飞行器数据进入恶意飞行器鉴别单元首先进行异常值筛选、剔除，经过筛选后的初始数据分别进入循环神经网络分类模块和特征匹配与高阶信息提取模块，初始数据经过循环神经网络分类得出目标威胁等级值，初始数据进入高阶信息提取模块后得出目标类型、目标是否携带武器、目标数量、目标抗毁伤能力、目标机动性等高阶信息，这些高阶目标信息再输入给贝叶斯网络推理得出各目标威胁等级值；
鉴别单元通过分析低空飞行器的实时状态信息从这些低空飞行器中鉴别出正在执行恶意行为或具有恶意行为趋势的目标，结合低空飞行器的类型、尺寸参数、结构材料、飞行状态、携带武器类型等信息评估其威胁等级。
所述指挥决策平台用于接收各低空飞行器的威胁等级，根据防控区域的重要程度以及低空飞行器的威胁等级，向反制单元发送处置指令；
其中指挥决策平台，主要包括数据接收处理工作站、网络服务器、全局态势展示投影及显示设备和执勤人员操作台等。数据接收处理工作站负责收集报警信息、恶意飞行器参数信息、视频流等实时数据。网络服务器用于内部和外部通信，包括电话系统、对讲系统、有线/无线网络互联系统等。全局态势展示投影及显示设备是指挥中心的核心显示设备，用于展示地图、监控画面、防控区安全态势、数据报表等关键信息，为执勤操作人员提供直观的视觉展示，帮助指挥决策人员快速获取关键信息。执勤人员操作台是操作人员的执行值班任务的工位，配备触摸屏、麦克风和各种控制按钮，用于信号切换、信息调取和指挥调度。
所述调度单元用于根据处置指令，结合城市全域反制设备的部署信息，通过智能优化调度算法，给出具体的反制设备调度指令；
调度单元根据待处置目标威胁等级和位置、速度等状态信息，以最高可靠性、最高执行效率和最小处置代价为优化目标，选择最合适的处置手段和处置设备。低空飞行器反制分系统根据接收的反制设备调度指令，自动选择合适时机、合适地点、配置合适毁伤参数，发挥反制设备最大效能，达到预期的恶意低空飞行器处置目标
所述反制单元根据调度指令，控制不同反制设备对相应威胁等级的低空飞行器进行精准打击。
特别的，所述反制单元包括压制干扰机、诱骗干扰机、指令干扰机、射弹拦截器、捕网拦截器、微波毁伤器和激光毁伤器，反制单元中各设备之间的物联网组网，以及反制设备部署信息数据库。

本发明所述基于蚁群算法的反制设备智能调度方法，以有向图表示反制设备智能调度方案的解空间，最后输出的反制设备调度方案为该有向图中的一组路径集合。有向图中的节点由“任务-设备对”表示，即表示使用某个反制设备完成某威胁目标处置任务。有向图中的路径代表了蚂蚁（在算法中是一种搜索代理）在节点之间的移动顺序，也就是任务分配的顺序。例如，一只蚂蚁从“威胁目标1处置任务-反制设备1”节点出发，到达“威胁目标2处置任务-反制设备3”节点，这条路径就表示了一种任务分配方式，即反制设备1执行威胁目标1处置任务、反制设备3执行威胁目标2处置任务。路径集合体现了一系列连续的调度决策。在每一步，蚂蚁根据信息素浓度和启发式信息决定走向下一个任务-设备对。在所述的有多任务和多设备的威胁目标处置场景中，蚂蚁在搜索过程中走过的路径可能包括多个任务在不同设备上的分配决策，这条路径的质量会通过上述目标函数（最大化处置成功率、最小化处置成本和最小化处置时间）来评估。
本发明所述的交叉算子，用于增强算法的搜索能力，通过组合不同个体（蚂蚁的路径）中的优良基因（好的任务-设备分配片段），产生新的更有潜力的解。它有助于跳出局部最优解，加快算法的收敛速度，使算法能够在更广泛的解空间中寻找最优或近似最优的设备调度方案。本发明采用顺序编码方式来表示蚂蚁的路径。假设有n个任务和m个设备，用一个长度为n的向量来表示蚂蚁的路径，向量中的每个元素i表示第i个任务分配到的设备编号。假设n=5,m=3，编码为[2,1,3,2,1]表示任务1分配到设备2，任务2分配到设备1，任务3分配到设备3，任务4分配到设备2，任务5分配到设备1。本发明所述的蚁群算法，根据蚂蚁所找到的解的质量来决定从蚁群中选取哪些蚂蚁（个体）进行交叉操作。优先选择质量较好的个体进行交叉，这样更有可能产生优质的后代。本发明所述交叉算子选择多个交叉点，将父代个体分割成多个部分，然后按照一定的规则交换这些部分来生成子代。例如，选择两个交叉点k1=2和k2=4，对于父代个体和，先分割为和，然后交换中间部分得到子代和。这种方法可以更灵活地组合父代个体中的优良部分，在设备调度中能产生更多样化的任务-设备分配组合，增加找到更优调度方案的概率。
本发明所述的变异算子，为了引入新的基因（调度方案中的新元素），避免算法过早收敛到局部最优解，增加种群（蚁群）的多样性。通过对个体（蚂蚁所代表的调度方案）进行小概率的变异操作，能够在解空间中探索一些未被充分发现的区域，从而有可能找到更优的设备调度策略。变异概率是一个关键参数，它决定了变异操作发生的频率。如果变异概率过高，算法会变得类似于随机搜索，失去蚁群算法基于信息素和启发式信息搜索的优势；如果变异概率过低，则很难有效地引入新的解，无法充分发挥变异算子的作用。本发明的变异概率取值范围在0.01- 0.1之间。在本发明的变异操作中，随机选择一个任务，将其从当前分配的设备重新分配到另一台设备。在当前设备调度方案中，有威胁目标T1当前分配给反制设备M1进行处置，通过变异操作，将威胁目标T1重新分配到反制设备M2。假设威胁目标T1在反制设备M1的处置时间为t1，在反制设备M2的处置时间为t2，这种变异可能会改变整个调度方案的总处置时间。
