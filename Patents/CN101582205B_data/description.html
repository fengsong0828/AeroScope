Description

低空空域飞行器飞行状态监视方法和装置
技术领域：
本发明涉及一种低空空域飞行器飞行状态监视方法和装置，属于通信技术领域。
背景技术：
随着国家经济建设不断发展、科学技术快速进步和人民生活水平逐步提高，近年来，以使用低空空域为主的通用航空飞行活动已进入国家经济建设的各个领域，并以其特有的优势在工业、农业、林业、旅游、医疗、体育等领域发挥着重要作用；热气球、滑翔伞、低空小型飞行器，已经逐步开始进入人们的生活。对低空空域飞行器实施监视，我国在此方面的研究仍处于起步阶段，目前尚无基于GPS和GPRS的低空空域飞行器飞行状态监视方法和装置。
发明内容：
针对现有技术的不足，本发明提供一种低空空域飞行器飞行状态监视方法和装置。
一种低空空域飞行器飞行状态监视装置，包括飞行器信息检测终端和监视中心信息处理终端，飞行器信息检测终端安装在飞行器上，监视中心信息处理终端安装在地面监视中心。
所述的飞行器信息检测终端包括主机功能处理和定时装置、信息采集装置、通信装置、报警装置和指示信息显示装置；信息采集装置、通信装置、报警装置和指示信息显示装置都与主机功能处理和定时装置连接；主机功能处理和定时装置为中央处理器，采用集成电路S3C44B0X，信息采集装置采用三维GPS定位，采集飞行器的三维位置、速度、时间、角度等飞行状态信息；通信装置采用GPRS通信装置MC55；报警装置由TTS(从文本到语音)和扬声器构成；指示信息显示装置由LED和LCD显示器构成。
所述的监视中心信息处理终端包括微处理器、信息接收装置、显示装置、存储装置，告警指示发送装置和报告发送装置；信息接收装置、显示装置、存储装置，告警指示发送装置和报告发送装置都与微处理器连接；微处理器是一台普通的接入因特网的PC机；信息接收装置和告警指示发送装置采用GPRS通信装置；显示装置采用普通的电脑液晶显示器；存储装置由电脑硬盘和USB标准的移动硬盘构成；报告发送装置采用Internet通信。
利用上述低空空域飞行器飞行状态监视装置监视低空空域飞行器飞行状态的方法，方法如下：
1)飞行器信息检测终端的主机功能管理和定时装置与监视中心信息处理终端的微处理器初始化；
2)飞行器信息检测终端与监视中心信息处理终端之间的通信是在UDP/IP基础上采用应答补发机制；
3)信息采集装置采集飞行器的三维位置、速度、时间、角度信息，通信装置以第1时间间隔向监视中心信息处理终端发送飞行状态信息；
4)监视中心信息处理终端的信息接收装置接收飞行状态信息，将飞行状态信息存储到存储装置，微处理器对信息进行分析处理，判断状态是否正常；若正常，则发送信息确认通知，飞行器信息检测终端在确认信息已被接受之后，再以第1时间间隔向监视中心发送新的飞行状态信息；若异常，则进行异常处理：首先判断是否超速，若不超速，再判断是否闯入三维立体围栏，若闯入三维立体围栏则进行闯入三维立体围栏处理；若速，再判断是否闯入三维立体围栏，若没有，则只进行超速处理，若有则先进行超速处理，再进行闯入三维立体围栏处理。
5)经过异常处理，当飞行器飞行状态回复正常后将时间间隔再次调整为第1时间间隔。
所述的超速处理的步骤为：首先将时间间隔调整为第2时间间隔，再由飞行器信息检测终端的通信装置以第2时间间隔向监视中心信息处理终端发送飞行状态信息，监视中心信息处理终端的信息接收装置接收飞行状态信息，将飞行状态信息存储到存储装置，并通过报告发送装置向相关管理部门报告，同时由告警指示发送装置发出速度调整指令、告警，飞行器信息检测终端的通信装置接收告警、指示，并按照指示进行速度调整，直至速度回复到正常范围。
所述的闯入三维立体围栏处理的步骤为：首先将时间间隔调整为第2时间间隔，再由飞行器信息检测终端的通信装置以第2时间间隔向监视中心信息处理终端飞行发送状态信息，监视中心信息处理终端的信息接收装置接收飞行状态信息，将飞行状态信息存储到存储装置，并通过报告发送装置向相关管理部门报告，同时由告警指示发送装置发出位置调整指令、告警，飞行器信息检测终端的通信装置接收告警、指示，并按照指示进行三维位置调整，直至其飞出三维立体围栏的范围。
所述的应答补发机制的步骤为：飞行器信息检测终端发送信息帧，监视中心信息处理终端接受信息帧，并发出应答帧，飞行器信息检测终端判断是否收到应答帧，若是，则继续发送新的帧；若不是，则打包最近的两个信息帧，再发送打包的信息帧；监视中心信息处理终端接受信息帧后，进行分析处理，发出控制帧，飞行器信息检测终端接收控制帧，按照指令进行调整并发送应答帧，监视中心信息处理终端判断是否收到应答帧，若是，继续发送新的控制帧；若不是，则打包最近的两个帧，再发送打包的帧。
所述的第1时间间隔是当飞行器飞行状态正常时，飞行器与监视中心的通信间隔。
所述的第2时间间隔是当飞行器飞行状态异常时，飞行器与监视中心的通信间隔。
所述的三维立体围栏是以要保护的目标为中心划定的立体区域。它由禁飞区和警告区两部分组成：其中禁飞区需要保护的目标的空域，它是严格禁止飞行的；警告区则是为了保护禁飞区而设定的，因为飞行器收发指令和调整飞行状态都需要一定的时间，因此当飞行器在接近禁飞区一定范围内时就需要报警，警告区的范围就是根据这段时间的划定的。在三维立体围栏之外的范围通称为安全区，飞行器可以在这个区域自由飞行。
所述的三维立体围栏的使用方法为：当飞行器在所述安全区的范围飞行时，监视中心信息处理终端只记录飞行器飞行状态，进行一般监视；当飞行器进入所述警告区范围时，监视中心信息处理终端对非法闯入飞行器发送警告和调整指令，使其飞出三维立体围栏的范围；当飞行器进入所述禁飞区范围时，监视中心信息处理终端继续对其发送警告和调整指令，警告其飞出三维立体围栏的范围，并且向相关管理部门发送飞行器信息，由相关管理部门采取强制措施。
本发明的特点在于：在高速飞行时，GPRS小区重选会造成短时间的通信中断，本专利的应用背景是飞行器信息检测终端与监视中心信息处理终端的交互通信不是满负荷通信，而且通信中断时间仅会影响一帧的通信，因此可以在UDP/IP的基础上采用应答补发机制。当前一帧没有被确认接收时，则在下一次发送时把前一帧和当前帧打包发送，一同被确认接收。这样在划定三维立体围栏范围时，需要设置一个通信中断的缓冲区，其大小为飞行器在一个第1时间间隔内的飞行距离。

实施例：
飞行器信息检测终端装置如图1所示，具备主机功能处理和定时装置1，信息采集装置2，通信装置3，报警装置4及指示信息显示装置5。主机功能处理和定时装置1采用S3C44B0X为中央处理器，进行数据处理及定时，控制信息采集装置2，通信装置3，报警装置4及指示信息显示装置5的动作；信息采集装置2采用三维GPS定位，采集飞行器的三维位置、速度、时间、角度等飞行状态信息；通信装置3采用GPRS通信装置MC55与监视中心的信息处理终端装置进行交互通信；报警装置4由TTS(从文本到语音)和扬声器构成，接收主机功能处理和定时装置1发送来的文本信息通过TTS进行语音提示；指示信息显示装置5由LED和LCD显示器构成，显示飞行器的当前飞行状态以及监视中心信息处理终端下发的控制指令。
监视中心信息处理终端装置如图2所示，具备微处理器6、信息接收装置7、显示装置8、存储装置9，告警指示发送装置10以及报告发送装置11。信息接收装置7、显示装置8、存储装置9，告警指示发送装置10和报告发送装置11都与微处理器6连接。微处理器6是一台普通的接入因特网的PC机，建立实时地理信息监视管理平台，实现高度的三维GIS管理；信息接收装置7和告警指示发送装置10采用GPRS通信装置与飞行器的信息检测终端装置进行交互通信；显示装置8采用普通的电脑液晶显示器，实时显示飞行器的飞行状态；存储装置9由电脑硬盘和USB标准的移动硬盘构成，实现信息的存储和重要信息的备份；报告发送装置11通过Internet通信，将异常飞行器信息发送给空管部门。
