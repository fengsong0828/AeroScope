Description

低空飞行器空地通信冗余组网方法、系统、装置及设备
技术领域
本发明涉及空地通信技术领域，尤其涉及低空飞行器空地通信冗余组网方法、系统、装置及设备。
背景技术
随着低空经济的发展以及各类有人、无人低空飞行器数量越来越多，稳定可靠的空地通信网络是低空经济发展的重要保障。设置搞可靠性的空地通信组网是低空经济发展的重要需求。由于空地通信既要承载飞行安全相关的飞控数据，又要承载多媒体流/高清摄像头等大带宽的视频流数据，因此空地通信网络既要考虑飞行安全敏感数据的可靠性，又要考虑承载的总流量，避免给未来运营、运维带来麻烦。但是传统的低空飞行器的地空通信只采用某一种通信技术，存在单点故障风险，当遇到设备故障、无线资源干扰、障碍物遮挡等突发因素，将导致通信终端，因此现有的低空飞行器的地空通信网络不可靠。

本发明提供的方法，可以是由交换机执行，为了方便描述，后文中称为第一交换机。本发明提供的方法中，低空飞行器上设置两种无线机载通信终端：第一机载通信终端和第二机载通信终端，二者分别对应的不同的通信技术。第一机载通信终端和第二机载通信终端可以分别与交换机建立通信隧道，例如GRE通信隧道。如图2所示，低空飞行器上设置的第一机载通信终端和第二机载通信终端在上电后，通过空口正常入网，通过基站、核心网与第一交换机建立隧道连接。
如果简单采用两种通信协议的物理隔离，进行端到端的双联路同时通信，会造成机载和地面段设备加倍，成本剧增，还会带来2个风险：
（1）如果网络设计不合理，容易导致环网风暴。由于机载飞控系统需要同时接入两种通信设备，地面飞控平台也需要同时接入两种网络的地面设备，如果网络没有合理的网络隔离设计，会导致机载设备、两种无线通信网络和地面飞控平台形成网络风暴；
（2）采用两种不同的通信技术同时传输数据，飞控数据的收发都会加倍，造成系统负荷加倍；下行多媒体数据加倍，会导致流量加倍和重复，地面端需要做数据的整合去重复，还会造成运营成本加倍。
在第一交换机与第一机载通信终端和第二机载通信终端分别建立通信隧道后，设置一个通信隧道为主用通信隧道，另一通信隧道为备用通信隧道，例如根据GRE主备协议启用第一通信隧道为主用通信隧道，第二通信隧道为备用通信隧道。
在第一通信隧道正常运行时，优先使用第一通信隧道来进行飞行器控制平台与低空飞行器之间的空地数据流传输，此时第二通信隧道断开，不进行数据传输，不产生加倍流量。飞行器控制平台用于接收低空飞行器发送的数据并进行处理，以及向低空飞行器发送控制指令等，飞行器控制平台与低空飞行器之间的数据流称为空地数据流。在第一通信隧道发生故障，例如设备故障、无线资源干扰、障碍物遮挡等突发因素导致通信终端时，第一交换机切换至备用通信隧道即第二通信隧道进行空地数据流的传输，第一交换可以自动检测并切换至备用通信隧道，业务系统无须干预，对业务完全无感。
本发明提供的方法，通过空口通信隧道+数据中心交换机通信隧道主备的方式设计高可靠性主备冗余方案，平时只有一路主隧道隧道收发数据，另一路做备份，当主隧道出现故障时交换机自动切换到备份隧道，避免了两路同时收发数据造成流量加倍。冗余方案需要机载终端和第一交换机配合工作，两者都支持GRE隧道功能，第一交换机开启GRE主备协议负责主备隧道的检测和切换。
进一步地，在一种可能的实现方式中，可以统计预设时段内第一通信隧道产生故障的频率，若该频率高于预设频率，那么可以切换第一通信隧道和第二通信隧道的优先级。优先级高的通信隧道为主用通信隧道，低的为备用通信隧道，也就是说，在优先级高的通信隧道正常运行时，只采用优先级高的通信隧道进行空地数据流的传输，只有在优先级高的通信隧道产生故障时，采用优先级低的通信隧道进行空地数据流的传输。虽然在建立通信隧道后，第一通信隧道为主用通信隧道，但是可能在后续使用中发现第一通信隧道发生故障的频率较高，那么可能会导致通信隧道的频繁切换，虽然不会对业务产生影响，但是频繁切换可能会导致通信的不稳定性，本发明提供的方法的一种可能实现方式中，每隔一个预设时段，统计主用通信隧道产生故障的频率，若产生故障的频率过高，那么将当前的主用通信隧道改为备用通信隧道，备用通信隧道改成主用通信隧道。在空地通信的过程中，根据实际运行情况实时动态地修改主备用通信隧道，可以提高空地通信的稳定性。
进一步地，本发明提供的方法中，还设置有第二交换机，第二交换机是飞行控制平台的交换机，与飞行控制平台通信连接，第二交换机是数据中心交换机。也就是说，本发明提供的方法的一个实施方式中，设置数据的中转点：第一交换机。第一交换机并不是直接接入飞行控制平台，而是而是先经由第二交换机接入飞行控制平台。进行飞行器控制平台与低空飞行器之间的空地数据流传输，包括：
通过聚合端口与第二交换机进行空地数据流的传输，第二交换机接入飞行器控制平台。
第一交换机分别和两种通信技术核心网设备通过网线或光纤正常连接，两种通信技术的空地数据流通过链路聚合端口连接到飞行器控制平台交换机，即第二交换机。
进一步地，聚合端口包括第一聚合端口和第二聚合端口，第一聚合端口与第一通信隧道对应，第二聚合端口与第二通信隧道对应。通过设置两个聚合端口，当聚合的某一链路故障，链路聚合自动中断该聚合端口，把业务切换到其他聚合端口，空地数据流的传输不受影响。这样可以提高空地数据流的传输通道的冗余，提高可靠性。
进一步地，在一种可能的实现方式中，第一聚合端口和第二聚合端口可支持负荷分担，也就是说，通过聚合端口与第二交换机进行空地数据流的传输，包括：
当空地数据流对应的数据量超出预设数据量时，同时通过第一聚合端口和第二聚合端口传输空地数据流。
可以实现设置预设数据量，该预设数据量反映单位时间内空地数据流的数据量限值，当空地数据流在单位时间内的实际数据量超出该预设数据量时，只通过第一聚合端口或只通过第二聚合端口进行数据传输，可能会导致聚合端口压力过大，产生传输卡顿、丢包等现象。本发明提供的方法，通过在空地数据流对应的数据量超出预设数据量时，同时通过第一聚合端口和第二聚合端口传输空地数据流，可以实现在高负荷时进行负荷分担，保障空地通信数据的传输流畅性和完整性。
进一步地，本发明提供的方法中，为了保证组网可靠性，如图3所示，连接两种通信技术的核心网的第一交换机采用堆叠组网，也就是说，第一交换机为堆叠交换机，其包括主用交换机和备用交换机，第一通信隧道和第二通信隧道分别对应主用交换机和备用交换机。在第一通信隧道存在故障时切换至第二通信隧道进行飞行器控制平台与低空飞行器之间的空地数据流传输，包括：
当主用交换机故障时，切换至备用交换机进行飞行器控制平台与低空飞行器之间的空地数据流传输，其中，主用交换机对应第一通信隧道，备用交换机对应第二通信隧道，主用交换机与备用交换机堆叠设置。
对于堆叠的已交换机，如果主用交换机故障，堆叠的另一台备用交换机自动切换为主用，相应的通信隧道一起切换，确保空口数据不受影响。
在地面侧组网（第一交换机与第二交换机的组网）中，连接两种核心网的数据中心交换机采用堆叠组网，两种核心网的数据接到堆叠的不同交换机上，可以避免单点/单交换机故障。同时，采用链路聚合模式连接，第一交换机参与链路聚合的端口来自堆叠的不同交换机，可以提高第一交换机与飞行器控制平台内部的第二交换机的冗余保护，进一步提高可靠性。
本发明提供的方法，针对低空飞行器空地通信，首次设计了无线网主备冗余组网和有线网多链路冗余的联动设计，有效解决了双网通信带来的资源浪费、网络风暴、负荷增加、运营成本加倍等问题。并且在低空飞行器空地通信领域首次实现GRE隧道技术、链路聚合技术和堆叠技术的结合与联动，实现了空地通信有线与无线网端到端的冗余通信，网络遇到主备切换自动进行，上层业务无感不受影响，极大提高了空地通信可靠性。进一步地利用成熟的GRE隧道技术有效实现有线网络和无线网络的解耦，增加了冗余组网方案的普适性。
