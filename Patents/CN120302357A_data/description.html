Description

一种实现低空飞行器多链路切换与冗余传输的方法和介质
技术领域
本发明涉及一种实现低空飞行器多链路切换与冗余传输的方法和介质，属于低空飞行器技术领域。
背景技术
随着低空飞行器，尤其是电动垂直起降飞行器在城市空中交通、应急救援等领域的迅速发展，对其通信系统的可靠性和实时性提出了更高要求。为了确保飞行安全及任务连续性，低空飞行器通常配备多条通信链路，用于实现飞行数据、系统状态及环境感知信息的稳定回传。
现有技术中多链路通信策略主要分为两类：一类是采用纯冗余传输机制，即将相同的数据同时通过多条链路发送至地面端；另一类是采用链路切换机制，即在主链路失效或性能下降至一定阈值时切换至备用链路。纯冗余机制虽然能够提高数据传输的可靠性，但会导致带宽资源浪费、能耗上升以及系统处理开销增加。纯切换机制能够优化资源利用，但在链路质量逐步劣化过程中存在数据丢失风险，且切换延迟可能影响飞行控制数据的及时性，带来安全隐患。
部分技术方案基于链路性能权重动态分配数据流，虽能实现一定程度的资源优化，但通常缺乏明确的链路状态分级标准，无法针对不同质量状况采取有针对性的传输策略。同时现有多链路管理方案普遍忽略了对数据包重要性的区分处理，所有数据均采用统一的传输策略，导致在资源受限或链路劣化环境下，关键控制数据的传输可靠性无法得到有效保障。
综上所述，现有低空飞行器多链路通信技术在链路状态动态感知、策略切换精度、关键数据保障及系统资源利用效率方面仍存在不足。亟需一种能够根据实时链路质量变化进行精细化状态划分，并据此灵活调整传输策略，同时兼顾通信可靠性和资源优化利用的多链路管理方法，以提高低空飞行器在复杂通信环境下的稳定性与飞行安全性。
发明内容
本发明的目的在于克服现有技术中的不足，提供一种实现低空飞行器多链路切换与冗余传输的方法，解决了低空飞行器在多链路通信中缺乏智能切换与差异化传输机制的问题，提升其在复杂网络环境下的通信可靠性与资源利用效率。
为达到上述目的，本发明是采用下述技术方案实现的：
第一方面，本发明提供了一种基于低空飞行器的低时延数据转换方法，包括：
构建低空飞行器与地面控制中心之间的通信链路，动态检测各通信链路的质量指标；
根据检测到的质量指标计算通信链路综合质量评价值；
根据所述通信链路综合质量评价值确定通信链路状态；
根据通信链路状态进行多链路切换与冗余传输。

进一步地，根据所述通信链路状态进行多链路切换与冗余传输包括：
当通信链路质量良好时，直接通过当前通信链路传输数据；
当通信链路轻度劣化时，启动关键数据包的冗余传输策略；
当通信链路严重劣化时，立即触发备用链路预建立连接，完成数据流切换至备用链路。
进一步地，当通信链路轻度劣化时，启动关键数据包的冗余传输策略包括：
选定重要性高的数据包作为关键数据包，标注关键属性；
在当前主链路上正常发送数据包时，在备用链路上对所述关键数据包进行同步冗余发送；
所述关键数据包中携带数据序列号和时间戳信息，用于接收端对所述关键数据包的数据去重和重组。
进一步地，当通信链路严重劣化时，立即触发备用链路预建立连接，完成数据流切换至备用链路包括：
提前在备用链路上完成连接预建立，保持备用链路处于就绪状态；
在触发切换条件时，将数据流快速迁移至备用链路，同时在切换窗口期内在主链路和备用链路上双路发送数据以确保数据不中断；
在备用链路确认数据接收无误后，断开原主链路连接。
进一步地，接收端接收来自不同通信链路传输的多个数据包；
根据所述数据包携带的序列号信息，采用哈希表去重算法去除重复数据包；
根据所述序列号大小顺序，重组连续数据流，确保数据顺序一致性；
将去重并重组后的数据流提供给应用层进行业务处理。

地面端接收模块支持多链路输入，所有接收到的数据包均携带序列号信息。系统构建哈希表，以序列号作为索引字段，执行冗余数据识别与去重操作：当某数据包序列号首次出现时，写入哈希表并保留数据；若再次出现相同序列号的数据包，则视为重复包，直接丢弃。为避免哈希表无限增长，系统设置清理机制，定期清除过期记录。
完成去重后的数据包根据序列号顺序进行重新组合，形成连续数据流。所述数据流经缓冲处理后，输入至飞行器应用层系统，供飞控计算模块、遥测显示模块、任务调度模块等后续处理使用，确保数据完整性、顺序性和实时性。
