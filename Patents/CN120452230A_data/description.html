Description

低空飞行器与车辆协同的智慧高速管控系统
技术领域
本发明涉及智能交通技术领域，具体来说，涉及低空飞行器与车辆协同的智慧高速管控系统。
背景技术
随着经济社会的快速发展和交通运输需求的持续增长，高速公路作为国家重要的交通基础设施，其管理和控制面临着前所未有的挑战。传统的高速公路管控系统主要依赖地面固定的传感器和监控设备开展工作，这种单一的管控模式已难以满足日益复杂的交通管理需求；近年来，低空飞行器凭借其机动灵活、视野开阔、快速响应等优势，在交通管理领域展现出巨大的应用潜力，为构建新一代智慧高速管控系统提供了新的技术手段和发展方向。
目前，高速公路管控系统主要在通信与数据交互、感知与定位、协同决策与控制等方面开展技术革新。在通信与数据交互方面，研究重点集中在异构网络融合、低延迟高带宽通信以及数据安全与隐私保护等领域；在感知与定位方面，主要致力于解决高精度定位融合和环境感知与目标识别等技术难题；在协同决策与控制方面，着力实现交通流量协同优化和应急事件协同处理等功能。虽然已有一些基于无人机和车辆协同的调度管理系统应用于物资配送领域，实现了物资管理、路线规划和配送管理等功能，但这些系统主要针对物流配送场景，难以满足高速公路复杂多变的管控需求。
然而，现有技术在实际应用中仍存在诸多局限性。首先，在通信技术方面，由于高速公路环境复杂，传统民用通信网络面临信号频繁切换、衰减和干扰等问题，且低空飞行器与车辆的通信频段不一致，缺乏统一协调机制；其次，在感知定位方面，现有技术过度依赖单一定位手段，在山区、隧道等特殊地形环境下容易出现定位精度大幅下降的情况；同时，不同传感器各自存在局限性，如毫米波雷达在恶劣天气下探测性能下降，而车辆和低空飞行器的感知系统又缺乏有效的协同机制；最后，在协同决策控制方面，现有技术缺乏精准的交通流量协同调控模型和完善的应急事件协同处理机制，难以实现交通流量的最优调配和突发事件的快速响应，制约了智慧高速管控系统的整体效能。
针对相关技术中的问题，目前尚未提出有效的解决方案。
发明内容
（一）解决的技术问题
针对现有技术的不足，本发明提供了低空飞行器与车辆协同的智慧高速管控系统，具备异构网络融合、多源感知定位和协同决策控制的优点，进而解决现有技术中通信网络不统一、定位感知不精准以及协同管控不高效的问题。
（二）技术方案
为实现上述异构网络融合、多源感知定位和协同决策控制的优点，本发明采用的具体技术方案如下：
低空飞行器与车辆协同的智慧高速管控系统，该低空飞行器与车辆协同的智慧高速管控系统包括：
异构网络融合单元，用于基于飞行器与车辆的通信网络需求，建立统一通信协议框架，并制定通用通信规则和数据格式，实现不同网络之间的协议转换和适配；
定位感知识别单元，用于集成多种定位技术，融合不同传感器原始数据，通过分层式不确定性建模与传递机制，识别目标行为属性，并建立目标跟踪算法，实时跟踪飞行器和车辆的位置和运动状态；
协同决策控制单元，用于融合飞行器和车辆的交通数据，规划飞行器和车辆的路径，并通过传感器和数据融合技术，快速检测突发事件，设计应急事件协同处理机制。
进一步地，异构网络融合单元包括智能网关硬件模块与智能网关软件模块；
智能网关软件模块，用于连接不同类型的网络，并进行协议转换，将飞行器的通信协议转换为车辆或高速公路基础设施能够接收的协议；
其中，智能网关软件模块包括：
协议转换模块，用于采用自适应通信协议与通信协议栈优化技术，通过构建通信性能评估模型，实现不同网络之间的协议转换和适配；
数据过滤模块，用于过滤和筛选通信数据进行过滤和筛选，并根据不同的应用需求将数据发送至正确目的地；
路由模块，用于选择最优的通信路径，提高数据传输效率；
安全防护模块，用于保护网络安全和隐私，防止数据被窃取或篡改。
进一步地，协议转换模块在采用自适应通信协议与通信协议栈优化技术，通过构建通信性能评估模型，实现不同网络之间的协议转换和适配时，包括：
通过网络探测工具与接收端反馈的信号指标，实时评估网络状态；
根据网络状态和通信需求，制定自适应协议参数调整策略；
基于网络、应用和用户体验层面，构建多维度性能评估指标体系；
利用层次分析法确定多维度性能评估指标体系中各层次评价指标的权重，并通过隶属度函数将实测数据映射到不同评价等级，构建通信性能评估模型；
基于历史数据训练和优化通信性能评估模型，通过调整参数提升协议转换的准确性和效率。
进一步地，利用层次分析法确定多维度性能评估指标体系中各层次评价指标的权重，并通过隶属度函数将实测数据映射到不同评价等级，构建性能评估模型包括：
设定评价指标，并根据实际应用场景和需求，确定评价等级；
根据评价指标的特点和实际情况，构建各指标对应的隶属度函数，并计算各评价指标对不同评价等级的隶属度向量；
利用各评价指标对不同评价等级的隶属度向量，构建模糊关系矩阵；
通过层次分析法确定权重向量，并与模糊关系矩阵进行合成运算，得到综合评价结果向量，采用最大隶属度原则确定最终评价等级。

进一步地，定位感知识别单元包括：
传感器融合模块，用于利用车辆、低空飞行器及固定传感器，初步分析采集的数据，并根据传输带宽，将得到的初步分析结果与部分关键原始数据发往中央处理模块；
中央处理模块，用于融合不同传感器原始数据和初步分析结果，通过分层式不确定性建模与传递机制，识别目标行为属性；
目标跟踪模块，用于基于目标跟踪算法，通过三维协方差矩阵和多因子协方差预测方程，实时跟踪飞行器和车辆的位置和运动状态。
进一步地，传感器融合模块包括：
网络监测模块，用于通过网络监测工具，获取当前网络可用的传输带宽与带宽占用率信息；
初步分析模块，用于基于车辆、低空飞行器及固定传感器，提取并融合简单特征，确定关键原始数据的数据量和传输优先级；
分配传输模块，用于根据传输带宽状况和数据特征，分配传输资源；
动态调整模块，用于持续监测带宽变化和数据传输进度，并动态调整传输策略；
验证优化模块，用于检查数据完整性和准确性，根据检查结果优化传输策略和数据处理方法。
进一步地，中央处理模块包括：
数据采集模块，用于分析传感器精度和环境噪声对采集数据的影响，根据不同传感器特点，选择相应的概率分布，并建立采集数据的概率密度函数；
特征提取模块，用于根据算法局限性和数据模糊性的量化结果，构建特征提取的不确定性模型；
信息融合模块，用于基于不确定性模型，采用加权平均信息融合算法，融合不同传感器的采集数据。
进一步地，目标跟踪模块包括：
状态初始化模块，用于根据先验知识与初始测量值，确定目标的初始状态向量和三维协方差矩阵；
状态预测模块，用于基于不同传感器获取目标的测量值，建立状态预测方程，并利用状态转移矩阵根据上一时刻的状态估计预测当前时刻的目标状态；
增益计算模块，用于利用观测残差建立多因子协方差预测方程，动态调整噪声协方差，并根据预测协方差和测量噪声协方差计算卡尔曼增益；
状态更新模块，用于基于状态更新方程，结合预测值和测量值，通过卡尔曼增益对状态估计进行更新，并更新协方差矩阵，根据实际需求判断是否继续对目标进行跟踪。

（三）有益效果
与现有技术相比，本发明提供了低空飞行器与车辆协同的智慧高速管控系统，具备以下有益效果：
（1）本发明通过异构网络融合单元，设计异构网络融合及智能网关系统架构，建立了统一通信协议框架，能够实现不同通信网络的无缝融合，利用智能网关设备进行协议转换、数据过滤和路由，适应飞行器和车辆不同需求；同时，采用自适应通信协议及优化技术，根据不同的通信场景和需求自动调整通信协议，对通信协议栈进行优化，确保飞行器与车辆之间、它们与交通管理中心之间的通信质量，实现低延迟、高带宽的数据传输，并通过可靠的安全加密和身份认证体系，全方位保护数据交互过程中的安全与隐私，有效保障了数据传输的高效性和可靠性，为实现高效的协同管控提供了坚实的技术支持。
（2）本发明通过定位感知识别单元，运用高精度定位融合算法集成卫星定位、惯性导航等多种定位技术，通过滤波等技术融合处理信息，无论在何种地形和环境下，都能精确确定车辆和低空飞行器的位置；同时，综合利用飞行器和车辆上的雷达、摄像头、激光雷达等传感器，结合道路和空域的固定传感器，如气象传感器和交通流量监测传感器等，实现对环境的全面感知，包括彼此的位置、速度、方向和天气状况等，对不同传感器的原始数据进行预处理后传递到中央处理模块进行融合处理，并采用主成分分析等方法将不同传感器提取的特征融合，提高目标检测和识别的准确性，提前发现潜在危险；此外，本发明的目标跟踪算法通过状态预测和测量更新两个阶段，有效利用目标的先验运动知识和实时测量数据，不断修正跟踪结果，即使在存在噪声和不确定性的情况下，也能对目标的位置和运动状态进行较为准确的实时跟踪，保障交通安全。
（3）本发明通过协同决策控制单元，融合飞行器和车辆运行数据，利用模拟退火算法协同规划飞行路径和行驶路线，实现最小化总行驶时间和减少交通拥堵等目标，实施动态交通管制措施，根据实时状况调整流量，提高道路和空域使用效率，实现交通流量的科学优化；同时建立应急事件监测系统，利用传感器和数据融合技术快速检测突发事件并分类分级，通过设计的协同处理机制和应急响应流程，合理调度资源，一旦出现突发状况，能快速协调飞行器和车辆等资源，引导救援车辆等迅速抵达事故现场，实现飞行器和车辆的协同作战，有效提升了整个高速场景应对紧急事件的协同处理能力，保障了高速交通的安全与顺畅，减少因通信不畅导致的事故和延误。

具体地，异构网络融合单元1采用异构网络融合及智能网关系统架构设计。
具体地，当前高速公路上的通信网络种类繁多，包括飞行器使用的特定通信网络、车辆的车载通信网络以及高速公路基础设施自身的通信网络等。建立统一的通信协议框架是实现异构网络融合的基础。通过制定通用的通信规则和数据格式，使得不同网络之间能够相互理解和交互信息。在本实施例中，对于飞行器通信网络，需要考虑其高移动性和对通信距离的要求；而车辆通信网络则更注重实时性和可靠性。通过统一的协议框架，将这些不同的需求整合起来，实现信息的无缝传输。
具体地，异构网络融合实现的关键是智能网关设备。它能够连接不同类型的网络，并进行协议转换，将来自一种网络的信息转换为另一种网络能够理解的格式。本实施例中，当飞行器发送的信息需要传输到车辆或高速公路基础设施时，智能网关可以将飞行器的通信协议转换为车辆或高速公路基础设施能够接收的协议。智能网关还可以实现数据的过滤和路由功能，根据不同的需求将信息发送到正确的目的地。本实施例中，在应急情况下，智能网关可以优先将救援信息发送到相关的救援车辆和指挥中心。智能网关设备包含高性能的处理器和通信模块，具备强大的计算能力和数据处理能力。它可以支持多种通信协议，如Wi-Fi、蓝牙、ZigBee、4G/5G等，并能够根据需要进行协议转换和数据路由。
具体地，采用自适应的通信协议及通信协议栈优化技术，通过优化数据压缩算法、减少数据包的大小、提高数据传输的可靠性等方式来优化通信协议栈，减少数据传输的开销，同时，根据网络状况和通信需求自动调整协议参数，以实现最佳的通信效果。

具体地，目标大致位置、运动方向等简单特征融合，通过时间同步与空间对齐统一多传感器数据基准，基于摄像头、雷达等边缘设备的计算能力，分别提取目标位置、速度、轮廓、类别等特征，本实施例中，利用加权平均、卡尔曼滤波或贝叶斯网络等算法对单传感器数据进行时空一致性处理，生成包含位置、运动趋势及置信度的简化特征向量，同时保留关键原始数据片段，最终在本地完成低复杂度融合后将特征包传输至中央模块，实现分布式实时处理与数据降维。
③分析数据传输模型根据传输带宽状况和数据特征。分配传输资源，以确保重要数据能够及时、准确地传输到中央处理模块。将关键原始数据和重要的初步处理结果赋予较高的优先级，优先进行传输。
④动态调整传输策略。在数据传输过程中持续监测带宽的变化情况以及数据的传输进度。如果发现带宽出现波动，或者数据传输出现延迟或丢包等情况，及时根据传输模型和当前的实际情况，动态调整传输策略。本实施例中，当带宽变窄（传输速度突然降至128kbps以下时）时适当降低一些非关键数据的传输速率（降低20%-50%），以保证关键数据的传输质量。
⑤验证与优化。对传输到中央处理模块的数据进行验证，检查数据的完整性和准确性。根据验证结果，对传输策略和数据处理方法进行优化，不断提高数据传输的效率和质量。本实施例中，通过对比接收数据与原始发送数据的校验和，或者检查数据是否符合特定的格式和范围，来判断数据是否完整准确。如果发现问题，分析原因并调整传输参数或数据处理方式。

综上所述，借助于本发明的上述技术方案，通过异构网络融合单元1，设计异构网络融合及智能网关系统架构，建立统一通信协议框架，实现不同通信网络的无缝融合，智能网关设备进行协议转换、数据过滤和路由，适应飞行器和车辆不同需求；采用自适应通信协议及优化技术，根据不同的通信场景和需求自动调整通信协议，对通信协议栈进行优化，确保飞行器与车辆之间、它们与交通管理中心之间的通信质量，实现低延迟、高带宽的数据传输，同时通过可靠的安全加密和身份认证体系，全方位保护数据交互过程中的安全与隐私，有效保障了数据传输的高效性和可靠性，为实现高效的协同管控提供了坚实的技术支持。本发明通过定位感知识别单元2，运用高精度定位融合算法集成卫星定位、惯性导航等多种定位技术，通过滤波等技术融合处理信息，无论在何种地形和环境下，都能精确确定车辆和低空飞行器的位置；同时，综合利用飞行器和车辆上的雷达、摄像头、激光雷达等传感器，结合道路和空域的固定传感器，如气象传感器和交通流量监测传感器等，实现对环境的全面感知，包括彼此的位置、速度、方向和天气状况等，对不同传感器的原始数据进行预处理后传递到中央处理模块进行融合处理，采用主成分分析等方法将不同传感器提取的特征融合，提高目标检测和识别的准确性，提前发现潜在危险；基于卡尔曼滤波的目标跟踪算法通过状态预测和测量更新两个阶段，有效利用目标的先验运动知识和实时测量数据，不断修正跟踪结果，即使在存在噪声和不确定性的情况下，也能对目标的位置和运动状态进行较为准确的实时跟踪，保障交通安全。本发明通过协同决策控制单元3，融合飞行器和车辆运行数据，利用模拟退火算法协同规划飞行路径和行驶路线，实现最小化总行驶时间和减少交通拥堵等目标，实施动态交通管制措施，根据实时状况调整流量，提高道路和空域使用效率，实现交通流量的科学优化；同时建立应急事件监测系统，利用传感器和数据融合技术快速检测突发事件并分类分级，通过设计的协同处理机制和应急响应流程，合理调度资源，一旦出现突发状况，能快速协调飞行器和车辆等资源，引导救援车辆等迅速抵达事故现场，实现飞行器和车辆的协同作战，有效提升了整个高速场景应对紧急事件的协同处理能力，保障了高速交通的安全与顺畅，减少因通信不畅导致的事故和延误。
