Description

一种面向低空飞行器的数据安全共享方法及系统
技术领域
本申请属于低空经济数据安全技术领域，涉及 技术，具体是一种面向低空飞行器的数据安全共享方法及系统。
背景技术
随着低空经济快速发展，低空飞行器在物流、农业、应急等领域的应用日益广泛，其运行过程中产生的海量数据成为支撑安全管控与高效运营的核心要素，例如飞行轨迹、气象信息、地理信息等。然而，当前低空数据共享仍面临挑战，数据采集、传输与存储过程中存在安全隐患，如飞行器与地面站间的通信链路易受窃取或篡改，威胁隐私保护与空域安全。为此，亟需构建数据共享方法，实现低空飞行全链条数据的可信共享与合规应用，为低空经济规模化发展提供基础支撑。
现有技术在低空飞行器的数据安全共享方法中，往往忽略了飞行器数据的重要性会随着飞行器的状态发生不同程度的改变，使得存在一些数据未能及时采取措施导致数据在传输过程中被篡改或窃取，导致数据安全共享的效率和准确度较低；因为，对于面向低空飞行器的数据安全共享方法仍需进一步的改进。
发明内容
本申请旨在至少解决现有技术中存在的技术问题之一；为此，本申请提出了一种面向低空飞行器的数据安全共享方法及系统，用于解决现有技术忽略了飞行器数据的重要性会随着飞行器的状态发生不同程度的改变，使得存在一些数据未能及时采取措施导致数据在传输过程中被篡改或窃取，导致数据安全共享的效率和准确度较低的技术问题。
为实现上述目的，本申请的第一方面提供了一种面向低空飞行器的数据安全共享方法，包括：
实时获取飞行器数据；所述飞行器数据是指低空飞行器在整个工作过程中出现的数据；
根据飞行器数据生成数据敏感度；
根据数据敏感度对飞行器数据进行加密得到密文；
获取加密时的密钥，根据加密时的密钥生成若干加密密钥分片；
根据飞行器数据对密文和加密密钥分片进行传输；
根据密文和加密密钥分片进行解密操作得到解密后的飞行器数据。
本申请通过上述步骤，依据飞行器数据的变化动态调整各数据参数的重要性，并自适应地挑选最适合的加密算法来加密这些参数；同时，在验证加密数据时设置相应的验证机制，确保数据安全共享的完整性和准确性，提升了数据安全共享方法的效率和准确度。

获取飞行器数据和敏感标签；所述飞行器数据包括若干参数；所述敏感标签包括固定标签和动态标签；
提取若干参数中的固定敏感参数，以及提取若干参数中的动态敏感参数及其对应的若干影响参数；
将固定标签及其对应的固定敏感参数输入至敏感判别模型中得到固定标签对应的固定敏感度；
将动态标签及其对应的动态敏感参数和若干影响参数输入至敏感判别模型中得到动态标签对应的动态敏感度；

所述敏感判别模型通过人工智能模型进行构建。

选择一个人工智能模型作为动态标签对应的基础模型；
通过动态标签对应的训练集训练动态标签对应的基础模型，并在动态标签对应的验证集上调整学习率和其他超参数得到动态标签对应的预训练模型；
通过对动态标签对应的预训练模型在动态标签对应的测试集上进行验证，最终得到输入为动态标签及其对应的动态敏感参数和若干影响参数、输出为动态标签对应的动态敏感度的敏感判别模型。

获取当前时刻飞行器数据中参数对应的数据敏感度；

否，选择当前时刻飞行器数据中参数对应的敏感度加密算法对应的密钥对飞行器数据中参数进行加密得到密文。
本申请通过对参数的数据敏感度进行自适应的选择加密算法，并且在参数数据敏感度发生变化时自动选择合适的加密算法进行加密，同时动态调整所有加密算法对应的密钥更新周期；同时，在加密时，判断密钥加密时的密钥是否在密钥更新周期范围内，动态权重合适的密钥进行加密，提高飞行器数据的安全性。

进一步的，所述根据加密后的密钥生成若干加密密钥分片，包括：
获取共享方数据和加密时的密钥；所述共享方数据是指飞行器数据与之共享的接收方的数据；
通过门限秘密共享方法对加密时的密钥生成N个密钥分片，并定义M/N规则，M/N规则是指至少有M个密钥分片合成密钥；其中，M和N均为正整数，且M<N；
提取N个共享方数据对应的N个分片公钥；
将密钥分片与分片公钥一一对应进行密钥分片加密操作得到N个加密密钥分片。
进一步的，所述根据飞行器数据对密文和加密密钥分片进行传输，包括：
获取飞行器数据、共享方数据、密文以及密文对应的若干加密密钥分片；

本申请通过对飞行器数据中与数据传输相关参数的分析，将各参数的数据敏感度作为确定传输优先级的依据，在传输条件不佳的情况下，能够优先保障高敏感度、高重要性数据的传输，从而有效提升了飞行器数据在传输过程中的效率与可靠性。

判断FS是否大于等于M；
是，对若干密钥分片进行门限秘密共享方法得到重组后的密钥；
通过密钥对密文进行解密操作得到解密后的飞行器数据；
否，生成解密条件不符告警信号。

本发明的另一方面提供了一种面向低空飞行器的数据安全共享系统，包括：数据采集模块、数据分析模块和预警模块；所述数据采集模块与数据分析模块呈电性和/或通信连接；所述数据分析模块与预警模块呈电性和/或通信连接；

2、本申请通过预先训练的敏感判别模型对飞行器数据中的各个参数进行敏感度判别，尤其是一些参数随着其他参数的变化影响导致该参数的数据敏感度也会随着变化，在这种情况下，需要通过敏感判别模型准确判别各个受影响参数的敏感度，为后续参数的加密算法的选择以及密钥更新周期的自适应提供准确的数据支持。

4、本申请通过分析飞行器数据中有关于数据传输的相关参数，以各个参数对应的数据敏感度作为传输优先级的判定标准，能够在传输效果不佳时尽可能保证重要数据的优先传输，提高了飞行器数据在数据传输时的效率。

本实施例中的根据数据敏感度对飞行器数据进行加密得到密文，包括：
获取数据敏感度和飞行器数据；数据敏感度包括固定敏感度和动态敏感度；飞行器数据包括若干参数；
根据数据敏感度在加密算法表中选择数据敏感度对应的敏感度加密算法；加密算法表由专家根据加密算法对数据敏感度对应的飞行器数据进行加密时产生的效果进行构建；本实施例中将数据敏感度分为3个等级，分别是L3、L2和L1等级，L3等级表示为数据敏感度最高等级；本实施例对于L3等级的参数采用SM9标识加密算法，对于L2等级的参数采用AES-256-GCM加密算法，对于L1等级的参数采用PRESENT-80加密算法；
根据敏感度加密算法和飞行器数据生成敏感度加密算法对应的密钥更新周期；密钥更新周期是指一个加密算法的密钥在什么时间内需要更新的时长；
获取当前时刻飞行器数据中参数对应的数据敏感度；
获取上一时刻飞行器数据中参数对应的上时刻数据敏感度；
判断数据敏感度与上时刻数据敏感度是否一致；判断敏感度是否保持一致就能知道参数对应的加密算法有没有进行更换，一旦参数的数据敏感度发生变化，则需要更换加密算法对参数进行重新加密；
是，获取当前时刻与最近一次密钥更新时刻之间的时间差值；判断时间差值是否小于数据敏感度对应的密钥更新周期；是，采用上一时刻飞行器数据中参数使用的密钥对飞行器数据中参数进行加密得到密文；否，采用飞行器数据中参数更新后的密钥对飞行器数据中参数进行加密得到密文；即在数据敏感度未发生变化时，判断此刻数据进行加密时密钥需不需要使用更新后的密钥，如果不用更新后的密钥则用上一时刻加密的密钥进行加密；
否，选择当前时刻飞行器数据中参数对应的敏感度加密算法对应的密钥对飞行器数据中参数进行加密得到密文。
本实施例根据参数的数据敏感度，自适应地选择合适的加密算法，并在参数敏感度发生变化时，自动切换至匹配的加密算法进行加密处理；同时，动态调整所有加密算法对应的密钥更新周期，在加密过程中，还会判断当前所用密钥是否处于有效的更新周期范围内，并根据权重动态选取适当的密钥进行加密，从而全面提升飞行器数据的安全性与防护水平。
本实施例中的根据敏感度加密算法和飞行器数据生成敏感度加密算法对应的密钥更新周期，包括：
获取敏感度加密算法和飞行器数据；
根据敏感度加密算法选择其对应的初始密钥更新周期CMGZ；本实施例中每个敏感度加密算法均对应初始密钥更新周期，例如，将L3等级使用的SM9标识加密算法的初始密钥更新周期设置为1小时，在其他实施例中可以设置为2小时甚至是24小时，将L1等级使用的PRESENT-80加密算法的初始密钥更新周期设置为10小时；
提取飞行器数据中动态敏感参数对应的动态敏感度属于高敏感度范围的参数数量变化量CSB；高敏感度范围根据经验设置，本实施例中采用了L3、L2和L1三个等级，将L2和L3等级作为高敏感度范围；参数数据变化量是指在飞行器工作过程中高敏感度对应的参数的增加量，例如，飞行器理论的高敏感度的参数为2个，在工作过程中，某一时刻，高敏感度的参数达到了5个，那么参数数量变化量为3个；
提取参数数量变化量对应的参数数量变化率CSL；参数数量变化率是指飞行器工作过程中高敏感度对应的参数数量的变化率，例如飞行器理论的高敏感度的参数为2个，在工作过程中，某一时刻，高敏感度的参数达到了5个，那么参数数量变化率为1.5；同理，飞行器理论的高敏感度的参数为1个，在工作过程中，某一时刻，高敏感度的参数达到了4个，那么参数数量变化率为3；在相同的参数数量变化量的基础上，参数数量变化率较大时各个数据的安全性会比参数数量变化率较小时更为重要；

本实施例中的根据加密后的密钥生成若干加密密钥分片，包括：
获取共享方数据和加密时的密钥；共享方数据是指飞行器数据与之共享的接收方的数据；
通过门限秘密共享方法对加密时的密钥生成N个密钥分片，并定义M/N规则，M/N规则是指至少有M个密钥分片合成密钥；其中，M和N均为正整数，且M<N；M和N的具体取值根据经验设置，本实施例中共设置有3个共享方，将N设置为3，M设置为2，即至少存在2个密钥分片即可得到完整的密钥用于对密文进行解密；
提取N个共享方数据对应的N个分片公钥；分片公钥用于对密钥分片进行加密；
将密钥分片与分片公钥一一对应进行密钥分片加密操作得到N个加密密钥分片。

对数据敏感度按照降序操作得到密文传输队列；降序操作是为了按照数据敏感度的大小进行传输，即将高敏感度对应的密文和若干加密密钥分片进行传输；

按照密文传输队列的先后顺序将密文和密文对应的若干加密密钥分片选择多个传输路径进行传输。

当共享方ID发出请求解密信号时，获取其他共享方ID提供的密钥分片；本实施例中设置了3个共享方，在任一共享方需要对密文进行解密时，需要对另外的共享方发出请求解密信号，在其他共享方收到信号后会考虑是否把相应的密钥分片发送给发出信号的共享方，如果共享方收到的密钥分片达不到解密的最低标准，则无法进行解密；
