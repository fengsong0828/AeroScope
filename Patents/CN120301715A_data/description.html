Description

一种面向低空飞行器的身份认证与访问控制方法及装置
技术领域
本发明属于低空安全管控技术领域，具体是一种面向低空飞行器的身份认证与访问控制方法。
背景技术
随着低空飞行器的广泛应用，其身份认证与访问控制已成为空域安全管理的关键环节。当前主流认证系统多基于静态多因子认证机制，如固定组合的密钥认证、数字证书或地理围栏技术。然而，现有方案的身份认证策略缺乏动态适应性，无法根据实时环境参数与设备状态智能调整认证强度，导致高风险场景下安全防护不足，容易出现非法接入漏判，导致存在单点攻破风险。此外，传统系统往往以延长认证耗时换取安全性，难以兼顾高密度低空交通场景下的效率需求。
发明内容
本申请提供一种面向低空飞行器的身份认证与访问控制方法及装置，解决了现有技术中低空飞行器身份认证安全性不足、无法动态适应复杂空域环境的技术问题。
为达到上述目的，本申请采用如下技术方案：
第一方面，提供一种面向低空飞行器的身份认证与访问控制方法，包括：
获取低空飞行器的环境数据、设备状态数据以及历史认证日志，得到采集数据；
将采集数据输入AI决策引擎，通过预训练的深度强化学习模型生成认证因子组合，得到主因子和备用因子；其中，所述备用因子的认证方式的认证精度高于所述主因子的认证方式；
利用主因子进行飞行器的身份认证，当主因子身份认证的匹配度处于预设区间，则触发备用因子进行二次身份认证，得到认证结果；
根据认证结果和风险评估模型生成访问令牌，飞行器根据访问令牌进行访问控制；其中，所述访问令牌用于描述权限等级与认证因子和环境数据之间的关系，所述风险评估模型内置在AI决策引擎。
基于上述技术方案，本申请提供的面向低空飞行器的身份认证与访问控制方法，通过动态多因子认证机制，结合AI驱动的智能决策引擎，能够灵活生成因子认证组合，主动识别伪造或异常设备，有效拦截非法接入行为。环境风险模型可实时评估空域复杂条件，动态调整权限等级与访问范围，确保在信号干扰、定位偏差等高风险场景下自动实施严格管控，能够显著提升空域安全性。
结合上述第一方面，在一种可能的实现方式中，所述采集数据包括：

包含飞行器的电池剩余容量、CPU或GPU负载、存储剩余空间以及飞行的高度、速度、航向的设备状态数据；

AI决策引擎模块，用于对采集数据进行处理，通过预训练的深度强化学习模型生成认证因子组合，同时根据环境数据利用风险评估模型计算环境风险系数。
身份认证模块，用于利用主因子进行飞行器的身份认证，在主因子身份认证匹配度处于预设区间时触发备用因子进行二次认证并得到认证结果，并根据认证结果和环境风险系数生成访问令牌，飞行器根据访问令牌进行访问控制。
为解决现有技术中低空飞行器身份认证安全性不足、无法动态适应复杂空域环境的技术问题，本申请实施例提供了一种面向低空飞行器的身份认证与访问控制方法，该方法包括：获取多维度采集数据、智能生成认证因子组合、分级执行身份认证、动态生成访问令牌，基于此实现低空飞行器的智能安全管控。
如图2所示，本申请实施例提供的一种面向低空飞行器的身份认证与访问控制方法，包括：
S1、获取低空飞行器的环境数据、设备状态数据以及历史认证日志，得到采集数据。
其中，采集数据是认证决策的基础信息集合，用于反映飞行器运行环境、设备状态及历史认证行为特征。
在一些实现方式中，环境数据可包含信号强度、频谱干扰值、水平分量精度因子等空域环境参数，设备状态数据可包括电池容量、飞行高度、速度等设备运行指标，历史认证日志可记录认证结果、耗时、因子组合等历史信息。
需要指出的是，采集数据的类型可根据实际应用场景扩展，例如增加天气数据或空域管制信息等。
示例性的，环境数据中的信号强度（RSSI）可通过传感器实时采集，设备状态数据中的飞行高度可通过飞行器内置GPS模块获取。
S2、将采集数据输入AI决策引擎，通过预训练的深度强化学习模型生成认证因子组合，得到主因子和备用因子；其中主因子和备用因子表示不同认证方式。
其中，AI决策引擎是实现智能认证的核心模块，深度强化学习模型用于从采集数据中提取特征并生成最优认证策略。
在一些实现方式中，认证因子组合可包含多种主因子（如射频指纹认证、密钥认证）和备用因子（如生物特征认证）的组合策略。
需要指出的是，模型训练过程需基于历史认证日志构建状态空间，通过奖励函数优化因子组合的安全性与效率。
示例性的，主因子可优先选择多频段射频指纹认证，备用因子可配置为指纹认证，适用于常规空域环境。
S3、利用主因子进行飞行器的身份认证，当主因子身份认证的匹配度处于预设区间，则触发备用因子进行二次身份认证，得到认证结果。
其中，主因子认证匹配度用于衡量认证信息与注册模板的一致性，预设区间可根据风险等级动态调整。且，注册模板表示预先存储的、用于验证飞行器身份的基准数据或规则集合，不同认证方式对应不同形式的注册模板。
在一些实现方式中，主因子认证可包含多频段射频指纹相似度计算、密钥解密验证等方式，匹配度可综合各认证方式结果生成；例如，对于密钥解密验证、数字签名校验类认证方式，当认证通过，如密钥哈希值与注册模板完全一致、签名验签成功时，则设定匹配度为1；认证失败时，设定匹配度为0；对于多频段射频指纹、传感器特征向量类认证方式（基于连续型特征的相似度匹配），可通过计算认证数据与注册模板的余弦相似度、欧氏距离或动态时间规整（DTW）距离生成匹配度；对于动态令牌时间同步、响应时延类认证方式，可通过计算认证成功时的时间差值绝对值与预设阈值的比较生成匹配度。
需要指出的是，预设区间通常为“认证成功但存在一定风险”的阈值范围，例如匹配度为60%-80%时触发二次认证。
示例性的，当飞行器进入区域管控区时，向认证系统发送身份认证请求，此时系统会先向飞行器发送数据收集请求，获取设备状态数据，并根据飞行器ID调取历史认证日志，同时从环境监测传感器或飞行器内置传感器获取环境数据，得到所有的采集数据。接着，认证系统对采集数据进行归一化处理后，输入AI决策引擎，通过预训练的深度强化学习模型生成包含主因子和备用因子的认证因子组合。系统向飞行器下发主因子认证指令，如执行多频段射频指纹认证或密钥认证，飞行器完成认证操作后将结果返回系统，系统计算主因子身份认证的匹配度。若匹配度达到第一预设阈值，则表明认证成功。若匹配度小于第二预设阈值，则表明认证失败，此时向飞行器发送禁止访问的信号，并通过认证系统内置的风险管控模块强制执行权限拦截，拒绝飞行器的所有操作请求。若匹配度处于[第一预设阈值,第二预设阈值]的预设区间内，系统会触发备用因子进行二次身份认证，如生物特征认证或区块链身份溯源认证，飞行器再次执行认证并反馈结果。
需要说明的是，若认证系统检测到有新飞行器接入区域管控区的网络，但没有发送身份认证请求时，系统会主动向飞行器发送数据收集请求，若飞行器拒绝该收集请求，则直接通过认证系统内置的风险管控模块强制执行权限拦截，并拒绝飞行器的所有操作请求。
S4、根据认证结果和风险评估模型生成访问令牌，飞行器根据访问令牌进行访问控制。
其中，访问令牌用于描述权限等级、有效期与认证因子、环境风险的关联关系，是访问控制的核心凭证。
在一些实现方式中，风险评估模型可基于环境数据计算风险系数，权限等级可划分为基础、中级、高级等层级，不同等级对应不同操作权限与令牌有效期。
需要指出的是，认证结果为失败时不生成访问令牌，且高风险环境下可能限制或禁止访问。
示例性的，若环境风险系数较高（如强信号干扰），即使认证成功，也仅生成基础权限令牌，限制飞行器的飞行高度与速度。
基于上述技术方案，本申请通过获取低空飞行器的环境数据、设备状态数据及历史认证日志，为认证决策提供多维度数据支撑；借助AI决策引擎与预训练的深度强化学习模型动态生成主因子和备用因子组合，构建了灵活的多因子认证机制，可主动识别异常设备并拦截非法接入；通过主因子认证匹配度动态触发备用因子进行二次认证，提升了认证过程的可靠性；结合认证结果与内置的风险评估模型生成访问令牌，实现了根据环境风险动态调整权限等级与访问范围的智能管控，有效提升了低空飞行器在复杂环境中的认证安全性、灵活性和对空域风险的自适应能力。
